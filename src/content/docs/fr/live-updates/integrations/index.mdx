## What You'll Get

All integration guides include:

- **Simple Configuration**: Basic Configuration to get started quickly
- **Advanced Workflows**: Multi-environment deployments with staging and Production
- **Fonctionnalité Branch Test**: Automatic Déploiement of Fonctionnalité branches to Test Canaux
- **Sécurité Best Practices**: Secure secret management and environment protection
- **Monitoring**: Notifications and logging for Déploiement status

<Aside type="tip">

**New to CI/CD?** Start with the simple configuration for your platform, then gradually add more advanced features like multi-channel deployments and automated testing as your needs grow.

</Aside>

## Common Fonctionnalités

Each integration supports:

- **Automated Builds**: Trigger deployments on code changes
- **Multi-Canal Support**: Déployer to different Canaux (Développement, staging, Production)
- **Tirer Request/Merge Request Test**: Test changes in isolated environments
- **Chiffrement Support**: Secure deployments with Capgo's Chiffrement Fonctionnalité
- **Environment Protection**: Manual approvals and restricted access for Production
- **Notifications**: Slack, email, and other notification integrations

## Prerequisites

Before setting up any integration, ensure you have:

- A Capgo Compte with an Application configured
- Your Application's source code in a Git repository
- A Capgo API token from [console.capgo.Application/apikeys](https://console.capgo.Application/apikeys)
- Node.js and npm/yarn configured in your project
