## Exigences

Avant de commencer avec Capgo, assurez-vous d'avoir :
- Une application Capacitor installée et configurée. [Découvrez comment configurer Capacitor](https://capacitorjs.com/docs/getting-started/)
- Node.js 20 ou version ultérieure installé
- Un des environnements de développement suivants :
  - **macOS** avec Xcode (pour le développement iOS) et/ou Android Studio (pour le développement Android)
  - **Linux** avec Android Studio (pour le développement Android)
  - **Windows** avec Android Studio (pour le développement Android)

## Les mises à jour en direct sont à 3 étapes


### Configuration guidée

<Steps>

1. Créez votre compte sur [https://capgo.app/register](https://capgo.app/register).
![capture d'écran d'inscription](/signup.webp "capture d'écran d'inscription")

2. Utilisez les commandes Init pour commencer <Code code={`npx @capgo/cli@latest init [APIKEY]`} lang="bash" /> Une série de questions vous sera présentée. Fournissez les réponses nécessaires pour terminer la configuration automatisée.

3. Déployez une mise à jour en direct

</Steps>

<Aside type="tip">En suivant ces étapes, vous serez opérationnel en un rien de temps. Si vous avez besoin d'aide supplémentaire pendant le processus, notre équipe d'assistance est [là pour vous aider](https://support.capgo.app). Bonne intégration !</Aside>

### Manual setup

Si la commande init ne fonctionne pas pour vous, vous pouvez ajouter manuellement une application.

<Steps>

1. Connectez le CLI à votre compte : <Code code={`npx @capgo/cli@latest login [APIKEY]`} lang="bash" />

2. Ajoutez l'application à votre compte avec cette commande : <Code code={`npx @capgo/cli@latest app add [APP_NAME]`} lang="bash" />

3. Installez le plugin dans votre application : <Code code={`npm i @capgo/capacitor-updater`} lang="bash" />

4. Configurez le plugin dans votre `capacitor.config` <Code code={`{\n  "plugins": {\n    CapacitorUpdater: {\n      "appId": "Your appID",\n      "autoUpdate": true,\n      "version": "1.0.0"\n    }\n  }\n}`} lang="json" />
  [Voir toutes les options disponibles](/docs/plugin/settings/). Ces informations seront déduites si elles ne sont pas fournies.

5. Appelez la méthode init le plus tôt possible dans votre application : <Code code={`import { CapacitorUpdater } from '@capgo/capacitor-updater';\nCapacitorUpdater.notifyAppReady();`} lang="typescript" />

6. Déployez une mise à jour en direct

</Steps>

<Aside type="note" title="Installation pour les anciennes versions de Capacitor">
La commande ci-dessus (étape 3) installe la dernière version (v8.x) pour Capacitor 8. Pour les anciennes versions de Capacitor, utilisez la balise npm appropriée :

```bash
# Capacitor 7
npm i @capgo/capacitor-updater@lts-v7

# Capacitor 6
npm i @capgo/capacitor-updater@lts-v6

# Capacitor 5
npm i @capgo/capacitor-updater@lts-v5
```

Chaque version majeure du plugin correspond à la version majeure de Capacitor (v8 → Capacitor 8, v7 → Capacitor 7, v6 → Capacitor 6, v5 → Capacitor 5). Les versions mineures partagent le même ensemble de fonctionnalités dans toutes les versions majeures (par exemple, 5.34.0, 6.34.0, 7.34.0 et 8.34.0 incluent toutes les mêmes fonctionnalités).
</Aside>
