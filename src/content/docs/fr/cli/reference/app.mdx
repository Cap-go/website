📱 Gérez les paramètres et configurations de votre application Capgo dans Capgo Cloud.

### <a id="app-add"></a> ➕ **Ajouter**

**Alias :** `a`

```bash
npx @capgo/cli@latest app add
```

➕ Ajoutez une nouvelle application au Capgo Cloud avec un identifiant d'application unique au format com.test.app.
Toutes les options peuvent être devinées à partir de la configuration si elles ne sont pas fournies.

**Exemple :**

```bash
npx @capgo/cli@latest app add com.example.app --name "My App" --icon ./icon.png
```

**Options :**

| Paramètres | Tapez | Descriptif |
| ---------- | ------------- | -------------------- |
| **-n,** | <code>string</code> | Nom de l'application à afficher dans Capgo Cloud |
| **-je,** | <code>string</code> | Chemin de l'icône de l'application à afficher dans Capgo Cloud |
| **-a,** | <code>string</code> | Clé API pour créer un lien vers votre compte |
| **--supa-hôte** | <code>string</code> | URL d'hôte Supabase personnalisée (pour l'auto-hébergement ou le développement Capgo) |
| **--supa-anon** | <code>string</code> | Clé anonyme Supabase personnalisée (pour l'auto-hébergement) |

### <a id="app-delete"></a> 🗑️ **Supprimer**

```bash
npx @capgo/cli@latest app delete
```

🗑️ Supprimez une application du Capgo Cloud, en spécifiant éventuellement une version pour supprimer uniquement ce bundle.

**Exemple :**

```bash
npx @capgo/cli@latest app delete com.example.app
```

**Options :**

| Paramètres | Tapez | Descriptif |

| **--supa-hôte** | <code>string</code> | URL d'hôte Supabase personnalisée (pour l'auto-hébergement ou le développement Capgo) |

### <a id="app-debug"></a> 🐞 **Débogage**

```bash
npx @capgo/cli@latest app debug
```

🐞 Écoutez les événements de mise à jour en direct dans Capgo Cloud pour déboguer votre application.
Ciblez éventuellement un appareil spécifique pour des diagnostics détaillés.

**Exemple :**

```bash
npx @capgo/cli@latest app debug com.example.app --device DEVICE_ID
```

**Options :**| Paramètres | Tapez | Descriptif |
| ---------- | ------------- | -------------------- |
| **-a,** | <code>string</code> | Clé API pour créer un lien vers votre compte |
| **-d,** | <code>string</code> | L'ID de périphérique spécifique à déboguer |
| **--supa-hôte** | <code>string</code> | URL d'hôte Supabase personnalisée (pour l'auto-hébergement ou le développement Capgo) |
| **--supa-anon** | <code>string</code> | Clé anonyme Supabase personnalisée (pour l'auto-hébergement) |

### <a id="app-setting"></a> ⚙️ **Réglage**

```bash
npx @capgo/cli@latest app setting
```

⚙️ Modifiez la configuration Capacitor par programme.
Spécifiez le chemin de configuration (par exemple, plugins.CapacitorUpdater.defaultChannel) avec --string ou --bool.

**Exemple :**

```bash
npx @capgo/cli@latest app setting plugins.CapacitorUpdater.defaultChannel --string "Production"
```

**Options :**

| Paramètres | Tapez | Descriptif |
| ---------- | ------------- | -------------------- |
| **--bool** | <code>string</code> | Une valeur pour le paramètre à modifier sous forme booléenne, ex : --bool true |
| **--chaîne** | <code>string</code> | Une valeur pour le paramètre à modifier sous forme de chaîne, ex : --string "Production" |

### <a id="app-set"></a> ⚙️ **Ensemble**

**Alias :** `s`

```bash
npx @capgo/cli@latest app set
```

⚙️ Mettez à jour les paramètres d'une application existante dans Capgo Cloud, tels que le nom, l'icône ou la période de conservation des offres groupées.
La rétention de 0 signifie un stockage infini.

**Exemple :**

```bash
npx @capgo/cli@latest app set com.example.app --name "Updated App" --retention 30
```

**Options :**

| Paramètres | Tapez | Descriptif |
| ---------- | ------------- | -------------------- |
| **-n,** | <code>string</code> | Nom de l'application à afficher dans Capgo Cloud |
| **-je,** | <code>string</code> | Chemin de l'icône de l'application à afficher dans Capgo Cloud |
| **-a,** | <code>string</code> | Clé API pour créer un lien vers votre compte |
| **-r,** | <code>string</code> | Jours de conservation des anciens bundles (0 = infini, par défaut : 0) |
| **--expose-metadonnées** | <code>string</code> | Exposer les métadonnées du bundle (lien et commentaire) au plugin (vrai/faux, par défaut : faux) |
| **--supa-hôte** | <code>string</code> | URL d'hôte Supabase personnalisée (pour l'auto-hébergement ou le développement Capgo) |
| **--supa-anon** | <code>string</code> | Clé anonyme Supabase personnalisée (pour l'auto-hébergement) |
