Ceci est la documentation de l'API publique de Capgo cloud. L'API vous permet de gérer par programmation vos ressources Capgo, y compris les organisations, les appareils, les canaux et les paquets. Elle est conçue pour être RESTful et utilise des méthodes HTTP standard.

## Authentification

Tous les endpoints de l'API nécessitent une authentification. Pour authentifier vos requêtes, ajoutez votre clé API dans l'en-tête `authorization`.

Exemple :
```bash
curl -H "authorization: your-api-key" https://api.capgo.app/organization/
```

## Limitation du débit

L'API implémente une limitation du débit pour garantir une utilisation équitable. Les limites actuelles sont :
- 100 requêtes par minute pour les comptes standard
- 1000 requêtes par minute pour les comptes entreprise

Si vous dépassez ces limites, vous recevrez une réponse 429 (Too Many Requests).

## Format de réponse

Toutes les réponses sont au format JSON. Les réponses réussies incluent généralement soit un objet `data`, soit un champ `status`. Les réponses d'erreur incluent un champ `error` avec une description de ce qui s'est mal passé.

Exemple de réponse réussie :
```json
{
  "status": "ok",
  "data": { ... }
}
```

Exemple de réponse d'erreur :
```json
{
  "error": "Invalid API key",
  "status": "KO"
}
```

## Meilleures pratiques

1. **Gestion des erreurs** : Vérifiez toujours les réponses d'erreur et traitez-les de manière appropriée
2. **Limitation du débit** : Implémentez un backoff exponentiel lors de l'atteinte des limites de débit
3. **Mise en cache** : Mettez en cache les réponses lorsque cela est approprié pour réduire les appels API
4. **Versionnage** : Suivez les modifications de l'API via notre journal des modifications
