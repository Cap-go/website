## Philosophie

L'équipe Ionic maintient Capacitor avec ses propres priorités et calendrier de publication. Les contributions communautaires peuvent attendre des mois, voire des années, avant d'être fusionnées. Capacitor+ adopte une approche différente :

1. **Fusionner les PR des forks** - Les PR précieuses bloquées dans la file d'attente en amont sont activement fusionnées
2. **Synchronisation continue** - Chaque changement en amont est automatiquement récupéré et testé
3. **Versions rapides** - Les changements sont publiés sur npm dès qu'ils passent le CI
4. **Communauté d'abord** - Vos contributions comptent et sont priorisées
5. **Transparence** - Toute l'automatisation est open source et visible
