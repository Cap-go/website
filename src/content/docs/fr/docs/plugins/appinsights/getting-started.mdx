## Utilisation

Importez le plugin et initialisez-le avec vos identifiants AppInsights :

```typescript
import { CapacitorAppinsights } from '@capgo/capacitor-appinsights';

// Initialize the SDK
const initializeAnalytics = async () => {
  await CapacitorAppinsights.init({
    partnerId: 'your-partner-id',
    partnerKey: 'your-partner-key'
  });
};

// Set user ID for tracking
const setUser = async (userId: string) => {
  await CapacitorAppinsights.setUserId({ userId });
};

// Get SDK state
const checkSDKState = async () => {
  const state = await CapacitorAppinsights.getState();
  console.log('SDK State:', state);
};

// Get plugin version
const checkVersion = async () => {
  const { version } = await CapacitorAppinsights.getPluginVersion();
  console.log('Plugin version:', version);
};
```

## Référence API

### init(options)

Initialiser le SDK AppInsights avec vos identifiants.

```typescript
await CapacitorAppinsights.init({
  partnerId: 'your-partner-id',
  partnerKey: 'your-partner-key'
});
```

**Paramètres :**
- `partnerId` (string) : Votre ID de partenaire AppInsights
- `partnerKey` (string) : Votre clé de partenaire AppInsights

### setUserId(options)

Définir ou mettre à jour l'ID utilisateur après l'initialisation.

```typescript
await CapacitorAppinsights.setUserId({
  userId: 'user-123'
});
```

**Paramètres :**
- `userId` (string) : L'ID utilisateur à définir pour le suivi

### getState()

Obtenir l'état actuel du SDK.

```typescript
const state = await CapacitorAppinsights.getState();
console.log(state);
// {
//   initCompleted: true,
//   jobScheduled: true,
//   permissionAcquired: true
// }
```

**Retourne :**
- `initCompleted` (boolean) : Si l'initialisation du SDK est terminée
- `jobScheduled` (boolean) : Si le travail de collecte de données est planifié
- `permissionAcquired` (boolean) : Si les permissions nécessaires sont acquises

### getPluginVersion()

Obtenir la version native du plugin Capacitor.

```typescript
const { version } = await CapacitorAppinsights.getPluginVersion();
console.log('Version:', version);
```

## Exemple complet

```typescript
import { CapacitorAppinsights } from '@capgo/capacitor-appinsights';

export class AnalyticsService {
  private initialized = false;

  async initialize(partnerId: string, partnerKey: string) {
    try {
      await CapacitorAppinsights.init({
        partnerId,
        partnerKey
      });

      const state = await CapacitorAppinsights.getState();
      this.initialized = state.initCompleted;

      console.log('AppInsights initialized:', state);
    } catch (error) {
      console.error('Failed to initialize AppInsights:', error);
    }
  }

  async identifyUser(userId: string) {
    if (!this.initialized) {
      throw new Error('AppInsights not initialized');
    }

    await CapacitorAppinsights.setUserId({ userId });
    console.log('User identified:', userId);
  }

  async getSDKStatus() {
    const state = await CapacitorAppinsights.getState();
    return {
      ready: state.initCompleted && state.permissionAcquired,
      tracking: state.jobScheduled,
      state
    };
  }
}
```

## Bonnes pratiques

1. **Initialiser tôt**
   Initialisez le SDK dès que votre application démarre pour assurer un suivi précis :
   ```typescript
   // In your app initialization
   await CapacitorAppinsights.init({
     partnerId: process.env.APPINSIGHTS_PARTNER_ID,
     partnerKey: process.env.APPINSIGHTS_PARTNER_KEY
   });
   ```

2. **Définir l'ID utilisateur lors de la connexion**
   Identifiez les utilisateurs après l'authentification :
   ```typescript
   async function onUserLogin(user) {
     await CapacitorAppinsights.setUserId({
       userId: user.id
     });
   }
   ```

3. **Vérifier l'état du SDK**
   Vérifiez que le SDK est prêt avant les opérations critiques :
   ```typescript
   const state = await CapacitorAppinsights.getState();
   if (!state.initCompleted) {
     console.warn('AppInsights not ready');
   }
   ```

4. **Gérer les erreurs avec élégance**
   ```typescript
   try {
     await CapacitorAppinsights.init(config);
   } catch (error) {
     console.error('Analytics initialization failed:', error);
     // Continue app operation even if analytics fails
   }
   ```

## Notes de plateforme

### iOS
- Nécessite iOS 10.0+
- Assurez-vous d'avoir les permissions nécessaires dans Info.plist pour le suivi

### Android
- Nécessite Android 5.0 (API 21)+
- Peut nécessiter la permission de statistiques d'utilisation selon les fonctionnalités de suivi

### Web
- Non supporté sur la plateforme web
