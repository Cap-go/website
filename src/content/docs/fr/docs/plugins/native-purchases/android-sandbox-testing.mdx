Tester les achats intégrés nécessite une configuration appropriée dans Google Play Console. Ce guide vous guidera à travers la configuration des tests sandbox pour votre application Android.

## Prérequis

Vous avez besoin d'un compte Google Play Console actif avec un abonnement en cours (renouvelé annuellement).

## Processus de configuration

<Steps>

1. **Ajouter un compte de test**

   Accédez à votre Google Play Console, sélectionnez votre application et allez dans **Setup > License testing**.

   Ajoutez le compte Google principal associé à l'appareil Android que vous utilisez pour les tests.

   ![Ajouter un compte de test](/native-purchases/android/sandbox-testing/add-testing-account.webp)

2. **Créer une piste de test**

   Allez dans **Testing > Closed testing** ou **Testing > Internal testing** et créez une nouvelle piste.

   Vous devez télécharger au moins une version signée de votre application sur cette piste.

   ![Créer une piste de test](/native-purchases/android/sandbox-testing/create-testing-track.webp)

3. **Créer une liste de testeurs**

   Après avoir configuré la piste de test, créez une liste d'e-mails et ajoutez l'adresse e-mail du compte de l'appareil de test.

   ![Créer une liste de testeurs](/native-purchases/android/sandbox-testing/create-tester-list.webp)

4. **Rejoindre le programme de test**

   Ouvrez l'URL d'inscription depuis votre appareil de test et cliquez sur le bouton **"Become a tester"** pour vous inscrire.

   ![Rejoindre le test](/native-purchases/android/sandbox-testing/join-testing-program.webp)

5. **Télécharger un APK signé**

   Générez un APK signé ou utilisez Android App Bundle pour télécharger votre application sur la piste de test.

   Le déploiement n'est pas nécessaire - il suffit de télécharger et d'attendre l'approbation.

   ![Télécharger un APK](/native-purchases/android/sandbox-testing/upload-signed-apk.webp)

6. **Compiler et tester**

   Exécutez votre application sur l'appareil de test. Lors d'une tentative d'achat, vous devriez voir un message :

   > "This is a test order; you will not be charged."

   ![Achat de test](/native-purchases/android/sandbox-testing/test-purchase-confirmation.webp)

</Steps>

## Notes importantes

- Les comptes de test ne seront pas facturés pour les achats
- Les achats de test utilisent le même flux que les achats en production
- Vous pouvez tester toutes les fonctionnalités d'abonnement, y compris les essais et les offres d'introduction
- Les abonnements de test ont des périodes de renouvellement accélérées pour des tests plus rapides

## Dépannage

**Les produits ne s'affichent pas en mode test :**
- Assurez-vous que votre application est téléchargée sur une piste de test
- Vérifiez que le compte de test est ajouté à License testing
- Vérifiez que les produits sont actifs dans Google Play Console

**Erreur "Item not available" :**
- Attendez 2-3 heures après la création des produits pour qu'ils deviennent disponibles
- Assurez-vous que le nom de package de votre application correspond à celui de Play Console
- Vérifiez que vous êtes connecté avec un compte de test

**Les achats de test apparaissent comme des frais réels :**
- Vérifiez que le compte est ajouté à License testing
- Assurez-vous que vous utilisez la version de la piste de test
- Vérifiez que la bannière de test apparaît lors de l'achat

## Ressources supplémentaires

Pour plus de détails, consultez la [documentation officielle de Google Play](https://developer.android.com/google/play/billing/test) sur les tests d'achats intégrés.
