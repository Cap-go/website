## Que couvre ce tutoriel ?

Ce tutoriel montrera comment démarrer supabase à partir de zéro ainsi que démarrer les fonctions edge

## Prérequis
1. [capgo](https://github.com/Cap-go/capgo/) cloné
3. [supabase](https://supabasecom/)

## Pour commencer

Pour commencer, exécutez
```bash
supabase start
```

Ensuite, vous devriez voir quelque chose comme ceci :
```js
Started supabase local development setup

         API URL: http://localhost:54321
     GraphQL URL: http://localhost:54321/graphql/v1
          DB URL: postgresql://postgres:postgres@localhost:54322/postgres
      Studio URL: http://localhost:54323
    Inbucket URL: http://localhost:54324
      JWT secret: [truncated]
        anon key: supa_key_anon
service_role key: supa_key_admin
```

Ensuite, ouvrez `configsjson` et définissez les valeurs suivantes :
```json
{
  "base_domain": {
    "prod": "webcapgoapp",
    "development": "developmentwebcapgoapp",
    "local": "localhost:3332"
  },
  "supa_anon": {
    "prod": "supa_key_anon",
    "development": "supa_key_anon",
    "local": "supa_key_anon"
  },
  "supa_url": {
    "prod": "http://localhost:54321",
    "development": "http://localhost:54321",
    "local": "http://localhost:54321"
  }
}
```

où `supa_key_anon` est la valeur de l'étape précédente

:::danger
⚠️ Ne committez pas `configsjson` dans le dépôt distant
:::

Ensuite, vérifiez que vous pouvez accéder à [localhost:54323](http://localhost:54323/projects/) et que la table `users` ressemble à ceci

<figure><img src="/supabasewebp" alt="supabase" /><figcaption></figcaption></figure>

Si c'est le cas, démarrez les fonctions edge en exécutant :

```bash
supabase functions serve
```

et démarrez le frontend en exécutant :
```bash
bun run serve
```
