Ces intégrations vous permettent de déployer automatiquement les mises à jour d'application à chaque fois que vous poussez des modifications de code, de tester des branches de fonctionnalités et de gérer plusieurs environnements de déploiement.

## Ce que vous obtiendrez

Tous les guides d'intégration incluent :

- **Configuration simple** : Configuration de base pour démarrer rapidement
- **Workflows avancés** : Déploiements multi-environnements avec staging et production
- **Tests de branches de fonctionnalités** : Déploiement automatique des branches de fonctionnalités vers des canaux de test
- **Meilleures pratiques de sécurité** : Gestion sécurisée des secrets et protection des environnements
- **Surveillance** : Notifications et journalisation pour le statut de déploiement

<Aside type="tip">

**Nouveau dans CI/CD ?** Commencez avec la configuration simple pour votre plateforme, puis ajoutez progressivement des fonctionnalités plus avancées comme les déploiements multi-canaux et les tests automatisés à mesure que vos besoins évoluent.

</Aside>

## Fonctionnalités communes

Chaque intégration prend en charge :

- **Builds automatisés** : Déclencher des déploiements lors de modifications de code
- **Support multi-canaux** : Déployer vers différents canaux (développement, staging, production)
- **Tests de Pull Request/Merge Request** : Tester les modifications dans des environnements isolés
- **Support du chiffrement** : Déploiements sécurisés avec la fonctionnalité de chiffrement de Capgo
- **Protection des environnements** : Approbations manuelles et accès restreint pour la production
- **Notifications** : Intégrations Slack, e-mail et autres notifications

## Prérequis

Avant de configurer une intégration, assurez-vous d'avoir :

- Un compte Capgo avec une application configurée
- Le code source de votre application dans un référentiel Git
- Un jeton API Capgo depuis [console.capgo.app/apikeys](https://console.capgo.app/apikeys)
- Node.js et npm/yarn configurés dans votre projet
