## Parité des fonctionnalités

Si vous choisissez d'utiliser votre propre serveur, vous perdrez le flux d'installation en 5 minutes\
Vous devez implémenter toutes ces fonctionnalités vous-même


| Fonctionnalités          | Capgo | Auto-hébergé |
| ------------------------ | ----- | ------------ |
| Mises à jour            | ✅     | 🚧          |
| Retour auto             | ✅     | 🚧          |
| Alertes email en cas d'échec | ✅     | 🚧          |
| Canaux                  | ✅     | 🚧          |
| Remplacement de canaux  | ✅     | 🚧          |
| Remplacement d'appareil | ✅     | 🚧          |
| Paramètres des canaux   | ✅     | 🚧          |
| Paramètres d'appareil   | ✅     | 🚧          |
| ID personnalisé         | ✅     | 🚧          |
| Configuration auto des canaux | ✅     | 🚧          |
| API des canaux          | ✅     | 🚧          |
| Statistiques des mises à jour | ✅     | 🚧          |

:::danger
Si vous envoyez une mauvaise mise à jour à vos utilisateurs, vous pouvez et allez casser leur application
:::

> Gardez à l'esprit que vous ne pouvez pas utiliser le cloud Capgo et votre serveur en même temps

:::danger
La fonction de mise à jour en direct (OTA) ne s'applique qu'aux modifications apportées aux fichiers HTML, CSS et JavaScript

Si vous apportez des modifications au code natif, comme des mises à jour des plugins Capacitor, il est obligatoire de soumettre à nouveau l'application à l'app store pour approbation
:::

## Choisir entre Auto et Manuel
En mode auto, une partie de la logique est gérée par le code natif, les mises à jour sont décidées côté serveur, c'est plus sécurisé et permet des mises à jour précises, un déploiement partiel sur un appareil ou un groupe et plus

En mode manuel, toute la logique est gérée par le JS

## Installer Capacitor updater

```bash title="Installer le Capacitor updater"
npm install @capgo/capacitor-updater
npx cap sync
```

## Préparer votre bundle

Pour envoyer des mises à jour à votre application, vous devez la compresser
La meilleure façon d'être certain que votre zip est bon est d'utiliser le CLI Capgo pour la compression
```bash title="Créer un bundle avec vos fichiers à servir depuis votre serveur"
npx @capgo/cli@latest bundle zip
```

Vous devrez servir ce zip depuis votre serveur par vos propres moyens
