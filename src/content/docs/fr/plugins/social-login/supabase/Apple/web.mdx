## Prérequis

Ce guide vous aidera à intégrer la connexion Apple avec l'authentification Supabase sur le Web. On suppose que vous avez déjà complété :
- la [Apple Configuration Web de connexion](/docs/plugins/social-login/apple/web/)
- le [Supabase Apple Connexion - Configuration générale](./general/).

## Implémentation

L’implémentation complète est disponible dans le fichier [`supabaseAuthUtils.ts`](https://github.com/Cap-go/capacitor-social-login/blob/main/example-app/src/supabaseAuthUtils.ts) de l’application exemple. Ce guide explique les concepts clés et comment l'utiliser.

### Utilisation de l'assistant d'authentification

La fonction `authenticateWithAppleSupabase` gère l'intégralité du flux d'authentification :

```typescript
import { authenticateWithAppleSupabase } from './supabaseAuthUtils';

const result = await authenticateWithAppleSupabase();
if (result.success) {
  console.log('Signed in:', result.user);
  // Navigate to your authenticated area
} else {
  console.error('Error:', result.error);
}
```

## Comment ça marche

Sur le Web, la connexion Apple utilise un flux contextuel OAuth :

1. **Initialisation** : le plugin est initialisé avec votre ID de service et l'URL de la page actuelle comme URL de redirection.
2. **OAuth Popup** : Ouvre une fenêtre contextuelle avec la page OAuth de Apple.
3. **Authentification utilisateur** : l'utilisateur s'authentifie avec Apple dans la fenêtre contextuelle
4. **Jeton d'identité** : Apple renvoie un jeton d'identité (JWT) contenant des informations utilisateur
5. **Supabase Authentification** : le jeton d'identité est envoyé à Supabase à l'aide de `signInWithIdToken()`

La fonction d'assistance détecte automatiquement la plateforme Web et configure tout en conséquence.

## Remarques importantes

### Configuration de l'ID de service

- Le Web nécessite votre ID de service Apple (identique à Android)
- L'ID de service doit être configuré dans le portail de développeur Apple avec les URL de retour correctes.
- Assurez-vous que votre domaine est ajouté aux domaines autorisés dans le portail des développeurs Apple.

### URL de redirection

- Sur le Web, l'URL de redirection est automatiquement définie sur `window.location.href` (URL de la page actuelle)
- Cela doit correspondre à l'une des URL de retour configurées dans le portail de développeur Apple
- Assurez-vous que l'URL avec et sans barre oblique finale est configurée dans le portail des développeurs Apple.

### Supabase ID client

Dans Supabase, configurez votre fournisseur Apple avec :
- **ID client** : votre identifiant de service Apple (par exemple, `com.example.app.service`)

Si vous utilisez également iOS, vous devrez fournir à la fois l'ID d'application et l'ID de service dans le champ ID client de Supabase (séparés par des virgules).

### Mettre à jour la fonction d'assistance

Lorsque vous utilisez la fonction d'assistance `authenticateWithAppleSupabase`, vous **devez** mettre à jour le `clientId` pour qu'il corresponde à votre ID de service Apple :

```typescript
await SocialLogin.initialize({
  apple: {
    clientId: isIOS
      ? undefined // iOS uses bundle ID automatically
      : 'your.service.id.here', // Your Apple Service ID for Web and Android
    redirectUrl: redirectUrl,
  },
});
```

:::attention[Important]
Remplacez `'your.service.id.here'` par votre ID de service Apple réel (la même valeur que vous avez configurée dans le portail de développeur Apple).
:::

## Dépannage

Si l'authentification échoue :- **Incompatibilité d'ID de service** : vérifiez que votre ID de service correspond à la fois dans le portail de développeur Apple et dans votre code.
- **URL de retour non configurée** : assurez-vous que l'URL de votre page actuelle (avec et sans barre oblique finale) est configurée dans le portail des développeurs Apple.
- **Popup bloqué** : Vérifiez les paramètres du navigateur - certains navigateurs bloquent les popups par défaut
- **Domaine non autorisé** : vérifiez que votre domaine est ajouté aux domaines autorisés dans le portail de développeur Apple.
- **Configuration Supabase** : vérifiez que votre ID de service est correctement configuré dans les paramètres du fournisseur Supabase Apple
- Consultez l'[exemple de code d'application](https://github.com/Cap-go/capacitor-social-login/blob/main/example-app/src/supabaseAuthUtils.ts) pour référence.
