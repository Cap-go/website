## Différences entre l'accès en ligne et l'accès hors ligne

Il existe plusieurs façons d'utiliser Google Connectez-vous avec Capacitor. Voici un tableau qui résume les différences entre les deux :

|                         | Accès en ligne | Accès hors ligne |
|:-------------:|:-------------:|:--------------:|
| Nécessite un back-end | ❌ | ✅ |
| Jeton d'accès de longue durée | ❌ | ✅ |
| Installation facile | ✅ | ❌ |

:::note
**Le mode hors ligne** signifie que votre backend peut accéder aux API de Google même lorsque l'utilisateur n'utilise pas activement votre application (c'est-à-dire lorsque l'utilisateur est « hors ligne » du point de vue de votre application). Les jetons d'accès de longue durée activent cette fonctionnalité en permettant au backend d'appeler les API Google au nom de l'utilisateur à tout moment.
:::

:::caution
**Le mode hors ligne EXIGE un serveur backend.** Lors de l'utilisation du mode hors ligne, le frontend reçoit un minimum d'informations (principalement juste un code d'authentification du serveur). Sans backend pour échanger ce code contre des jetons et des informations utilisateur, le mode hors ligne ne fournit aucune donnée utile à votre application frontend.
:::

Si vous ne savez toujours pas lequel choisir, veuillez considérer les scénarios suivants :

1. Vous souhaitez que l'utilisateur se connecte immédiatement après que vous allez lui émettre un JWT personnalisé. Votre application n'appellera PAS les API Google
   
   Dans ce cas, choisissez l'accès en ligne.

2. Votre application appellera certaines API Google depuis le client, mais jamais depuis le backend
   
   Dans ce cas, choisissez l'accès en ligne

3. Votre application appellera certaines API Google depuis le backend, mais uniquement lorsque l'utilisateur utilise activement l'application.
   
   Dans ce cas, choisissez l'accès en ligne

4. Votre application vérifiera périodiquement le calendrier de l'utilisateur, même s'il n'utilise pas activement l'application.
   
   Dans ce cas, choisissez l'accès hors ligne

## Un exemple de backend pour l'accès en ligne

Dans cette partie du tutoriel, je vais vous montrer comment valider l'utilisateur sur votre backend. 

Cet exemple sera très simple et il sera basé sur les technologies suivantes :

- [Dactylographié](https://www.typescriptlang.org/)

- [Hono](https://hono.dev/)

- [Récupération Javascript API](https://developer.mozilla.org/en-US/docs/Web/API/Window/fetch)

Vous pouvez trouver le code de cet exemple [ici](https://github.com/WcaleNieWolny/capgo-social-login-backend-demo/blob/141c01d93a85240e31a0d488a89df13c842708b1/index.ts#L135-L153)

Comme vous pouvez le constater :

<img src="/social-login-assets/vscode_auth_google.png" alt="Code VS affichant le code d'authentification Google qui vérifie les jetons" />L'idée est plutôt simple. Vous envoyez une simple requête `GET` à `https://www.googleapis.com/oauth2/v3/tokeninfo` et cela vous renvoie si le token est valide ou non et si c'est le cas, il vous donne l'email de l'utilisateur. Il vous donne également d'autres informations sur le jeton utilisateur
<img src="/social-login-assets/google_auth_playground_token_info.png" alt="Google OAuth Aire de jeu affichant la réponse aux informations de jeton avec les détails de l'utilisateur" />

À partir de là, vous pouvez attribuer à l'utilisateur votre propre JWT ou émettre une sorte de cookie de session. Les possibilités sont infinies pour l'implémentation finale de l'authentification.

Si vous souhaitez appeler celui de Google API, je vous recommande fortement de consulter le terrain de jeu OAuth 2.0 de [Google](https://developers.google.com/oauthplayground). De là, vous pouvez facilement voir quelles API vous pouvez appeler.

## Utiliser l'accès hors ligne avec votre propre backend

Pour utiliser l'accès en ligne, vous aurez besoin des éléments suivants :

- Un serveur HTTP

Dans cet exemple, j'utiliserai les technologies suivantes pour fournir l'accès hors ligne dans mon application :

- [Hono](https://hono.dev/)

- [Validateur Hono Zod](https://hono.dev/docs/guides/validation#with-zod)

- [Zod](https://zod.dev/)

- [Hono JWT](https://hono.dev/docs/helpers/jwt#jwt-authentication-helper)

- [LowDb](https://www.npmjs.com/package/lowdb) (une simple base de données)

Le code de cet exemple peut être trouvé [ici](https://github.com/WcaleNieWolny/capgo-social-login-backend-demo/blob/aac7a8c909f650a8c2cd7f88c97f5f3c594ce9ba/index.ts#L139-L287)

Quant au code client, il ressemble à ceci :

```dactylographié
import { Capacitor } from '@capacitor/core';
import { GoogleLoginOfflineResponse, SocialLogin } from '@capgo/capacitor-social-login';
import { usePopoutStore } from '@/popoutStore'; // <-- specific to my app

const baseURL = "[expurgé]";

fonction asynchrone fullLogin() {
  attendre SocialLogin.initialize({
    google : {
      webClientId : '[expurgé]',  
      iOSClientId : '[expurgé]',
      iOSServerClientId: 'La même valeur que webClientId',
      mode : 'hors ligne' // <-- important
    } 
  })
  réponse const = attendre SocialLogin.login ({
    fournisseur : 'google',
    options : {
      forceRefreshToken : true // <-- important
    }
  })

  if (response.provider === 'google') {
    const result = réponse.result comme GoogleLoginOfflineResponse
    const res = attendre fetch(`${baseURL}/auth/google_offline`, {
      en-têtes : {
        "Content-Type": "application/json"
      },
      corps : JSON.stringify({
        serverAuthCode : résultat.serverAuthCode,
        plateforme : Capacitor.getPlatform()
      }),
      méthode : "POST"
    })

    si (res.status !== 200) {
      popoutStore.popout("Échec de la connexion complète à Google", "vérifier la console");
      retour
    }

    const { jwt } = attendre res.json();
    const userinfo = attendre fetch(`${baseURL}/auth/get_google_user`, {
      en-têtes : {
        Autorisation : `Bearer ${jwt}`
      }
    })
    si (userinfo.status !== 200) {
      popoutStore.popout("Échec de la connexion complète à Google (informations utilisateur), "vérifier la console");
      retour
    }
    popoutStore.popout("userinfo res", wait userinfo.text());
  }
}
