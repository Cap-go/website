**Configuration de base (mode en ligne - recommandé pour la plupart des applications) :**
      ```typescript
      // onMounted is Vue specific
      onMounted(() => {
        SocialLogin.initialize({
          google: {
            iOSClientId: '673324426943-redacted.apps.googleusercontent.com',
            mode: 'online' // Default mode
          }
        })
      })
      ```

      **Configuration avancée avec des ID client supplémentaires :**
      ```typescript
      onMounted(() => {
        SocialLogin.initialize({
          google: {
            webClientId: 'YOUR_WEB_CLIENT_ID',           // Optional: for web platform support
            iOSClientId: 'YOUR_IOS_CLIENT_ID',           // Required: from step 1
            iOSServerClientId: 'YOUR_WEB_CLIENT_ID',     // Optional: same as webClientId, needed for some advanced features
            mode: 'online'                               // 'online' or 'offline'
          }
        })
      })
      ```
     
      :::remarque
      **Exigences en matière d'identification client :**
      - `iOSClientId` : **Obligatoire** - Doit se terminer par `googleusercontent.com` (à partir de l'étape 1 ci-dessus)
      - `webClientId` : **Facultatif** - Nécessaire uniquement si vous prenez également en charge la plate-forme Web ou si vous avez besoin de fonctionnalités avancées.
      - `iOSServerClientId` : **Facultatif** - Doit avoir la même valeur que `webClientId` lorsqu'il est fourni
      
      Pour une configuration avancée avec `webClientId` et `iOSServerClientId`, consultez le [guide de configuration Web](/docs/plugins/social-login/google/web/) pour créer ces informations d'identification.
      :::

      :::attention
      **À propos du mode hors ligne :** Lors de l'utilisation de `mode: 'offline'`, la réponse de connexion ne contiendra pas directement les données utilisateur. Au lieu de cela, vous recevrez un code d'authentification du serveur qui doit être échangé contre des informations utilisateur via votre serveur backend. Consultez le [guide de configuration général](/docs/plugins/social-login/google/general/#using-offline-access-with-your-own-backend) pour plus de détails sur la mise en œuvre.
      :::
   
   3. Implémentez la fonction de connexion. Créez un bouton et exécutez le code suivant en cliquant
     
      **Pour le mode en ligne :**
      ```typescript
      const res = await SocialLogin.login({
        provider: 'google',
        options: {}
      })
      // handle the response - contains user data
      console.log(JSON.stringify(res))
      ```

      **Pour le mode hors ligne :**
      ```typescript
      const res = await SocialLogin.login({
        provider: 'google',
        options: {
          forceRefreshToken: true // Recommended for offline mode
        }
      })
      // res contains serverAuthCode, not user data
      // Send serverAuthCode to your backend to get user information
      console.log('Server auth code:', res.result.serverAuthCode)
      ```

5. Testez votre application
   
   1. Créez votre application et exécutez « cap sync »
   
   2. Si vous avez tout fait correctement, vous devriez voir le flux de connexion Google fonctionner correctement.

## Problèmes connus

### Incompatibilité du plugin d'écran de confidentialité

Le plugin de connexion Google est incompatible avec [@capacitor/privacy-screen](https://github.com/ionic-team/capacitor-privacy-screen). Lors de l'utilisation des deux plugins ensemble, la vue Web de connexion Google sera interrompue par l'écran de confidentialité.

**Solution :** Appelez `await PrivacyScreen.disable();` avant d'appeler la fonction de connexion :

```typescript
import { PrivacyScreen } from '@capacitor/privacy-screen';
import { SocialLogin } from '@capgo/capacitor-social-login';

await PrivacyScreen.disable();
await SocialLogin.login({
  provider: 'google',
  options: {}
});
```
