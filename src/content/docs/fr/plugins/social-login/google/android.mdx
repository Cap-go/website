##Présentation

Dans ce guide, vous apprendrez comment configurer la connexion Google avec la connexion sociale Capgo pour Android.
Je suppose que vous avez déjà lu le [guide de configuration général](/docs/plugins/social-login/google/general/).

## En utilisant Google, connectez-vous sur Android

Dans cette partie, vous apprendrez comment configurer la connexion Google dans Android.

:::caution
Le certificat Android SHA1 est plus que douloureux et je ne souhaiterais à personne d'avoir à le configurer. Les étapes suivantes supposent le scénario le plus simple d’une application qui n’est pas publiée sur Google Play Store et qui est utilisée uniquement par le simulateur local ou le périphérique matériel de développement. 

Si vous avez déployé votre application sur Google Play Store, vous **DEVEZ** ajouter un ID client Android supplémentaire qui contient le SHA1 de la console Google Play pour les versions de production. Vous pouvez trouver le hachage SHA1 que Google Play utilise pour signer votre bundle de versions sous `Test and release > Setup > App Signing`.

Enfin, il est important de mentionner que si vous vous trompez, l'erreur ne sera PAS évidente. Cela peut être très difficile à déboguer. Si vous rencontrez des difficultés avec la configuration, veuillez consulter les [problèmes Github](https://github.com/Cap-go/capacitor-social-login/issues).

De plus, vous pouvez consulter la section de dépannage de la [configuration de connexion Google pour Android](#troubleshooting) pour plus d'informations.
:::

:::note
Vous pouvez créer plusieurs ID client Android. Ceci est requis si vous disposez de plusieurs certificats SHA1.
:::

<Steps>
1. Créez un ID client Android.
   1. Cliquez sur la barre de recherche
      <img src="/social-login-assets/google_cons_search.png" alt="" />
   
   2. Recherchez `credentials` et cliquez sur celui « API et services » (numéro 2 sur la capture d'écran)
      <img src="/social-login-assets/google_cons_cred_search.png" alt="Résultats de recherche affichant l'option d'informations d'identification avec les API et les services mis en évidence" />
   
   3. Cliquez sur « créer des informations d'identification »
      <img src="/social-login-assets/google_cons_create_cred.png" alt="Bouton Créer des informations d'identification dans la console Google" />
   
   4. Sélectionnez `OAuth client ID`
      <img src="/social-login-assets/google_cons_cred_oauth.png" alt="Option d'ID client OAuth dans le menu de création d'informations d'identification" />

   5. Sélectionnez le type d'application `Android`
      <img src="/social-login-assets/google_cons_app_type_android.png" alt="Sélection du type d'application avec l'option Android mise en évidence" />
   
   6. Ouvrez Android Studio
   
   7. Tout en bas du navigateur, recherchez les « Gradle Scripts »
      <img src="/social-login-assets/studio_gradle_scripts.png" alt="Section Gradle Scripts dans le navigateur de projet Android Studio" />
   
   8. Recherchez `build.gradle` pour le module `app`
      <img src="/social-login-assets/studio_build_gradle.png" alt="Fichier build.gradle (Module : app) dans la section Gradle Scripts" />
   
   9. Copiez le `android.defaultConfig.applicationId`. Ce sera votre « nom de package » dans la console Google
      <img src="/social-login-assets/studio_build_gradle_app_id.png" alt="Fichier Build.gradle montrant la configuration de l'ID d'application" />
   
   10. Maintenant, ouvrez le terminal. Assurez-vous que vous êtes dans le dossier `android` de votre application et exécutez `./gradlew signInReport`
      <img src="/social-login-assets/term_sign_report.png" alt="Terminal affichant la commande Gradlew SignInReport" />
   
   11. Faites défiler vers le haut de cette commande. Vous devriez voir ce qui suit. Copiez le `SHA1`.
      <img src="/social-login-assets/term_sign_report_res.png" alt="Sortie du terminal affichant l'empreinte digitale du certificat SHA1" />
   
   12. Maintenant, revenez à la console Google. Entrez votre `applicationId` comme « Nom du package » et votre SHA1 dans le champ du certificat et cliquez sur `create`.
      <img src="/social-login-assets/google_cons_creat_android_client.png" alt="Formulaire de création de client Android avec nom du package et champs SHA1 renseignés" />
   
2. Créez un client Web (ceci est requis pour Android)
   
   1. Accédez à la page « Créer des informations d'identification » dans la console Google.
   
   2. Définissez le type d'application sur `Web`
      <img src="/social-login-assets/google_cons_app_type_web.png" alt="Sélection du type d'application avec l'option Web en surbrillance" />
   
   3. Cliquez sur `Create`
      <img src="/social-login-assets/google_cons_web_app_create.png" alt="Formulaire de création de client Web avec le bouton Créer en bas" />
   
   4. Copiez l'ID client, vous l'utiliserez comme `webClientId` dans votre code JS/TS
      <img src="/social-login-assets/google_cons_copy_web_client_id.png" alt="Détails de l'ID client affichant l'ID client Web à copier" />

3. Modifiez votre `MainActivity`1. Veuillez ouvrir votre application dans Android Studio. Vous pouvez exécuter `cap open android`
   
   2. Trouvez `MainActivity.java`
     
      1. Ouvrez le dossier `app`
         <img src="/social-login-assets/studio_app_folder.png" alt="Dossier d'application dans le navigateur de projet Android Studio" />
     
      2. Trouvez `java`
         <img src="/social-login-assets/studio_app_java.png" alt="Dossier Java dans la structure du projet Android Studio" />
     
      3. Trouvez votre `MainActivity.java` et cliquez dessus
         <img src="/social-login-assets/studio_app_java_activity_main.png" alt="Fichier MainActivity.java dans la structure du package" />
   
   3. Modifiez `MainActivity.java`. Veuillez ajouter le code suivant
     
      ```java
      import ee.forgr.capacitor.social.login.GoogleProvider;
      import ee.forgr.capacitor.social.login.SocialLoginPlugin;
      import ee.forgr.capacitor.social.login.ModifiedMainActivityForSocialLoginPlugin;
      import com.getcapacitor.PluginHandle;
      import com.getcapacitor.Plugin;
      import android.content.Intent;
      import android.util.Log;
      import com.getcapacitor.BridgeActivity;
      
      // ModifiedMainActivityForSocialLoginPlugin is VERY VERY important !!!!!!    
      public class MainActivity extends BridgeActivity implements ModifiedMainActivityForSocialLoginPlugin {
      
            @Override
            public void onActivityResult(int requestCode, int resultCode, Intent data) {
              super.onActivityResult(requestCode, resultCode, data);
      
              if (requestCode >= GoogleProvider.REQUEST_AUTHORIZE_GOOGLE_MIN && requestCode < GoogleProvider.REQUEST_AUTHORIZE_GOOGLE_MAX) {
                PluginHandle pluginHandle = getBridge().getPlugin("SocialLogin");
                if (pluginHandle == null) {
                  Log.i("Google Activity Result", "SocialLogin login handle is null");
                  return;
                }
                Plugin plugin = pluginHandle.getInstance();
                if (!(plugin instanceof SocialLoginPlugin)) {
                  Log.i("Google Activity Result", "SocialLogin plugin instance is not SocialLoginPlugin");
                  return;
                }
                ((SocialLoginPlugin) plugin).handleGoogleLoginIntent(requestCode, data);
              }
            }
      
            // This function will never be called, leave it empty
            @Override
            public void IHaveModifiedTheMainActivityForTheUseWithSocialLoginPlugin() {}
      }
      ```

   4. Enregistrez le fichier

4. Utilisez Google Connectez-vous dans votre candidature
   
   1. Tout d’abord, importez `SocialLogin`
     
      ```typescript
      import { SocialLogin } from '@capgo/capacitor-social-login';
      ```
   
   2. Appelez initialiser. Cela ne devrait être appelé qu'une seule fois.
     
      ```typescript
      // onMounted is Vue specific
      // webClientId is the client ID you got in the web client creation step not the android client ID.
      onMounted(() => {
        SocialLogin.initialize({
          google: {
            webClientId: '673324426943-avl4v9ubdas7a0u7igf7in03pdj1dkmg.apps.googleusercontent.com',
          }
        })
      })
      ```
     
   3. Appelez `SocialLogin.login`. Créez un bouton et exécutez le code suivant en cliquant.
     
      ```typescript
      const res = await SocialLogin.login({
        provider: 'google',
        options: {}
      })
      // handle the response
      console.log(JSON.stringify(res))
      ```

   4. Cliquez sur `next`

   6. Cliquez sur suivant

## Dépannage

Si vous rencontrez des problèmes, veuillez consulter les [problèmes Github](https://github.com/Cap-go/capacitor-social-login/issues).

Les problèmes avec la connexion Google sont **TOUJOURS** liés au certificat SHA1.

Si vous ne parvenez pas à obtenir le certificat de développement SHA1, essayez d'utiliser un magasin de clés personnalisé. [Ici](https://github.com/Cap-go/capacitor-social-login/issues/147#issuecomment-2849742574) est un commentaire expliquant comment ajouter un magasin de clés à votre projet.
