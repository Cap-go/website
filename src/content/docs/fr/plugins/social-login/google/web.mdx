##Présentation

Dans ce guide, vous apprendrez comment configurer la connexion Google avec la connexion sociale Capgo pour les applications Web.
Je suppose que vous avez déjà lu le [guide de configuration général](/docs/plugins/social-login/google/general/).

## Utilisation de la connexion Google sur le Web

Utiliser la connexion Google sur le Web est plutôt simple.
Pour l'utiliser, vous devez procéder comme suit :

<Steps>
1. Créez un client Web dans la console Google
   
   :::remarque
   Si vous avez déjà configuré la connexion Google pour Android, vous pouvez ignorer cette étape car vous avez déjà créé un client Web. Vous pouvez passer directement à l'étape 2.
   :::
   
   1. Cliquez sur la barre de recherche
      <img src="/social-login-assets/google_cons_search.png" alt="Google Barre de recherche de la console" />
   
   2. Recherchez `credentials` et cliquez sur l'option « API et services » (numéro 2 sur la capture d'écran)
      <img src="/social-login-assets/google_cons_cred_search.png" alt="Résultats de recherche affichant l'option d'informations d'identification avec les API et les services mis en évidence" />
   
   3. Cliquez sur « créer des informations d'identification »
      <img src="/social-login-assets/google_cons_create_cred.png" alt="Bouton Créer des informations d'identification dans la console Google" />
   
   4. Sélectionnez `OAuth client ID`
      <img src="/social-login-assets/google_cons_cred_oauth.png" alt="Option d'ID client OAuth dans le menu de création d'informations d'identification" />
   
   5. Sélectionnez le « Type d'application » comme « Application Web »
      <img src="/social-login-assets/google_cons_app_type_web.png" alt="Sélection du type d'application avec l'option Web en surbrillance" />
   
   6. Nommez votre client et cliquez sur `Create`
      <img src="/social-login-assets/google_cons_web_app_create.png" alt="Formulaire de création de client Web avec le champ de nom en surbrillance" />
   
   7. Copiez l'ID client, vous l'utiliserez comme `webClientId` dans votre application
      <img src="/social-login-assets/google_cons_copy_web_client_id.png" alt="Détails de l'ID client affichant l'ID client Web à copier" />

2. Configurez le client Web dans la console Google

   1. Veuillez ouvrir la [page d'informations d'identification](https://console.cloud.google.com/apis/credentials) et cliquer sur votre client Web.
      <img src="/social-login-assets/google_cons_open_web_client_id.png" alt="Liste des informations d'identification montrant le client Web sur lequel cliquer" />

   2. Maintenant, veuillez ajouter le `Authorized JavaScript origins`. Cela doit inclure toutes les adresses que vous pourriez utiliser pour votre application. Dans certains cas, j'utiliserai **UNIQUEMENT** localhost, mais comme j'utilise un port personnalisé, je dois ajouter à la fois `http://localhost` et `http://localhost:5173`.
    
      1. Veuillez cliquer sur « ajouter un URI »
         <img src="/social-login-assets/google_cons_authorized_js_add_btn.png" alt="Section d'origines JavaScript autorisée avec bouton AJOUTER URI" />
    
      2. Please type your URL
         <img src="/social-login-assets/google_cons_authorized_js_typed_url.png" alt="Boîte de dialogue AJOUTER URI avec l'URL de l'hôte local saisie" />
    
      3. Veuillez répéter jusqu'à ce que vous ayez ajouté toutes les URL
    
      4. Lorsque vous avez terminé, votre écran devrait ressembler à ceci
         <img src="/social-login-assets/google_cons_authorized_js_final.png" alt="Origines JavaScript autorisées avec plusieurs URL d'hôte local ajoutées" />

   3. Maintenant, veuillez ajouter quelques « URI de redirection autorisés ». Cela dépendra de la page sur laquelle vous comptez utiliser le plugin CapacitorSocialLogin. Dans mon cas, je vais l'utiliser le `http://localhost:5173/auth` 
    
      1. Veuillez cliquer sur « AJOUTER URI »
         <img src="/social-login-assets/google_cons_web_add_redirect_url_1.png" alt="Section URI de redirection autorisée avec bouton AJOUTER URI" />
    
      2. Entrez votre URL et cliquez à nouveau sur « AJOUTER une URL »
         <img src="/social-login-assets/google_cons_web_add_redirect_url_2.png" alt="Boîte de dialogue AJOUTER URI avec l'URL de redirection saisie" />

   4. Click `save`
      <img src="/social-login-assets/google_cons_web_app_save.png" alt="Bouton Enregistrer en bas de la configuration du client Web" />

3. Maintenant, vous devriez être prêt à appeler `login` depuis JavaScript comme ceci :

   1. Tout d’abord, importez `SocialLogin`
  
      ```typescript
      import { SocialLogin } from '@capgo/capacitor-social-login';
      ```

   2. Ensuite, appelez initialize. Cela ne devrait être appelé qu'une seule fois.
  
      ```typescript
      // onMounted is Vue specific
      // webClientId is the client ID you got in the web client creation step not the android client ID.
      onMounted(() => {
       SocialLogin.initialize({
         google: {
           webClientId: '673324426943-avl4v9ubdas7a0u7igf7in03pdj1dkmg.apps.googleusercontent.com',
         }
       })
      })
      ```

   :::note[Gestion des redirections Web]
   Lorsque vous utilisez la connexion Google sur le Web, vous ** DEVEZ ** appeler n'importe quelle fonction du plugin lorsque la redirection se produit pour initialiser le plugin afin qu'il puisse gérer la redirection et fermer la fenêtre contextuelle. Vous pouvez appeler soit `isLoggedIn()` OU `initialize()` - les deux déclencheront la gestion de la redirection :

   ```typescript
   // Option 1: Call isLoggedIn when the redirect page loads
   SocialLogin.isLoggedIn({ provider: 'google' }).catch(() => {
     // Ignore the result, this is just to initialize the plugin
   });

   // Option 2: Call initialize when the redirect page loads
   SocialLogin.initialize({
     google: {
       webClientId: 'YOUR_WEB_CLIENT_ID.apps.googleusercontent.com',
     }
   }).catch(() => {
     // Ignore any errors, this is just to handle the redirect
   });
   ```
   :::

   3. Créez un bouton de connexion qui appelle `SocialLogin.login` lorsque vous cliquez dessus
  
      ```typescript
      const res = await SocialLogin.login({
       provider: 'google',
       options: {}
      })
      // Handle the response
      console.log(JSON.stringify(res));
      ```
</Steps>
