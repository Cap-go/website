##Présentation

Dans ce guide, vous apprendrez comment configurer la connexion Facebook avec la connexion sociale Capgo. Vous aurez besoin des éléments suivants :

- Un compte développeur Facebook
- Le nom/ID du package de votre application
- Accès à un terminal de génération de hachages de clés (Android)

## Configuration générale

Si vous n'avez pas encore créé d'application Facebook, procédez comme suit :

<Steps>
1. Créez une application Facebook
   
   Suivez le didacticiel pour [Créer une application](https://developers.facebook.com/docs/development/create-an-app/)

2. Ajoutez la connexion Facebook à votre application
   
   Dans votre tableau de bord de développeur Facebook, ajoutez le produit de connexion Facebook à votre application.

3. Avant de pouvoir rendre votre application publique, suivez ce [tutoriel](https://developers.facebook.com/docs/development/release/) pour la publier.
</Steps>

## Android Configuration

<Steps>
1. Ajoutez une autorisation Internet à votre `AndroidManifest.xml`

   Assurez-vous que cette ligne est présente :
   ```xml
   <uses-permission android:name="android.permission.INTERNET"/>
   ```

2. Générez votre hachage de clé Android
   
   Il s'agit d'une étape de sécurité cruciale requise par Facebook. Ouvrez votre terminal et exécutez :

   ```bash
   keytool -exportcert -alias androiddebugkey -keystore ~/.android/debug.keystore | openssl sha1 -binary | openssl base64 -A
   ```

   Lorsque vous êtes invité à saisir un mot de passe, utilisez : `android`

   :::remarque
   Pour les versions de version, vous devrez utiliser votre magasin de clés de version :
   ```bash
   keytool -exportcert -alias your-key-name -keystore your-keystore-path | openssl sha1 -binary | openssl base64 -A
   ```
   :::

3. Ajoutez le hachage de clé à votre application Facebook
   
   1. Accédez au tableau de bord de votre application sur Facebook Developers
   2. Accédez à Paramètres > De base
   3. Faites défiler jusqu'à la section "Android".
   4. Cliquez sur "Ajouter une plateforme" si Android n'est pas encore ajouté et remplissez les détails.
   5. Ajoutez le hachage de clé que vous avez généré
   6. Pour la production, ajoutez les hachages de clé de débogage et de publication

4. Mettez à jour votre `AndroidManifest.xml` pour inclure :

   ```xml
   <application>
       ...
       <activity android:name="com.facebook.FacebookActivity"
           android:configChanges="keyboard|keyboardHidden|screenLayout|screenSize|orientation"
           android:label="@string/app_name" />
       
       <activity
           android:name="com.facebook.CustomTabActivity"
           android:exported="true">
           <intent-filter>
               <action android:name="android.intent.action.VIEW" />
               <category android:name="android.intent.category.DEFAULT" />
               <category android:name="android.intent.category.BROWSABLE" />
               <data android:scheme="FB[APP_ID]" />
           </intent-filter>
       </activity>
   </application>
   ```

   :::attention
   Assurez-vous de remplacer `[APP_ID]` par votre ID d'application Facebook réel dans l'attribut `android:scheme`.
   :::
</Steps>

## iOS Configuration

<Steps>
1. Ajoutez la plateforme iOS dans la console développeur Facebook
   
   1. Accédez au tableau de bord de votre application sur Facebook Developers
   2. Accédez à Paramètres > De base
   3. Faites défiler vers le bas de la page et cliquez sur « Ajouter une plateforme »
   4. Sélectionnez iOS et remplissez les détails requis

2. Ouvrez votre projet Xcode et accédez à Info.plist

3. Ajoutez les entrées suivantes à votre Info.plist :

   ```xml
   <key>FacebookAppID</key>
   <string>[APP-ID]</string>
   <key>FacebookClientToken</key>
   <string>[CLIENT-TOKEN]</string>
   <key>FacebookDisplayName</key>
   <string>[APP-NAME]</string>
   <key>LSApplicationQueriesSchemes</key>
   <array>
       <string>fbapi</string>
       <string>fb-messenger-share-api</string>
   </array>
   <key>CFBundleURLTypes</key>
   <array>
       <dict>
           <key>CFBundleURLSchemes</key>
           <array>
               <string>fb[APP-ID]</string>
           </array>
       </dict>
   </array>
   ```

   :::attention
   Remplacez les valeurs suivantes :
   - `[APP-ID]` avec votre identifiant d'application Facebook
   - `[CLIENT-TOKEN]` avec votre token client
   - `[APP-NAME]` avec le nom de votre application
   :::

4. Modifiez le `AppDelegate.swift`

   ```swift
   import FBSDKCoreKit
   @UIApplicationMain
   class AppDelegate: UIResponder, UIApplicationDelegate {
         func application(_ application: UIApplication, didFinishLaunchingWithOptions launchOptions: [UIApplication.LaunchOptionsKey: Any]?) -> Bool {
           // Override point for customization after application launch.
           
           // Initialize Facebook SDK
           FBSDKCoreKit.ApplicationDelegate.shared.application(
               application,
               didFinishLaunchingWithOptions: launchOptions
           )
           
           return true
       }

       func application(_ app: UIApplication, open url: URL, options: [UIApplication.OpenURLOptionsKey: Any] = [:]) -> Bool {
           // Called when the app was launched with a url. Feel free to add additional processing here,
           // but if you want the App API to support tracking app url opens, make sure to keep this call
           
           if (FBSDKCoreKit.ApplicationDelegate.shared.application(
               app,
               open: url,
               sourceApplication: options[UIApplication.OpenURLOptionsKey.sourceApplication] as? String,
               annotation: options[UIApplication.OpenURLOptionsKey.annotation]
           )) {
               return true;
           } else {
               return ApplicationDelegateProxy.shared.application(app, open: url, options: options)
           }
       }
   }
   ```
</Steps>

## Utilisation de Facebook Connectez-vous à votre application:::caution
**Avant de commencer** : n'oubliez pas qu'avec le nouveau Facebook SDK, le type de jeton que vous recevez dépend entièrement du choix de suivi des applications de l'utilisateur, et non de la configuration de votre code. Implémentez toujours la gestion du jeton d'accès et du jeton JWT dans votre backend pour garantir que l'authentification fonctionne pour tous les utilisateurs.
:::

<Steps>
1. Initialisez la connexion Facebook dans votre application

   ```typescript
   import { SocialLogin } from '@capgo/capacitor-social-login';

   // Initialize during app startup
   await SocialLogin.initialize({
     facebook: {
       appId: 'APP_ID',
       clientToken: 'CLIENT_TOKEN',
     }
   })
   ```

2. Implémentez la fonction de connexion

   ```typescript
   async function loginWithFacebook() {
     try {
       const result = await SocialLogin.login({
         provider: 'facebook',
        options: {
          permissions: ['email', 'public_profile'],
          limitedLogin: false // See Limited Login section below for important details
        }
       });
       console.log('Facebook login result:', result);
       // Handle successful login
     } catch (error) {
       console.error('Facebook login error:', error);
       // Handle error
     }
   }
   ```

   :::remarque
   **Connexion limitée (iOS uniquement)** : définissez `limitedLogin` sur true si vous souhaitez utiliser la fonctionnalité de connexion limitée de Facebook. Il s'agit d'une fonctionnalité réservée à iOS qui offre une confidentialité améliorée en limitant les données partagées lors de la connexion.

   **Limites importantes :**
   - **iOS uniquement** : la connexion limitée n'affecte que les appareils iOS et n'a aucun impact sur Android.
   - **ATT Override** : même si vous définissez `limitedLogin: false`, Facebook le forcera automatiquement à `true` si l'utilisateur n'a pas accordé l'autorisation App Tracking Transparency (ATT)
   - **Toujours gérer les deux cas** : votre application doit toujours être prête à gérer des scénarios de connexion limités et complets.

   **Checking ATT Status:**
   ```typescript
   // Check if user has granted tracking permission
   const trackingStatus = await SocialLogin.providerSpecificCall({
     call: 'facebook#requestTracking',
     options: {}
   });
   console.log('Tracking status:', trackingStatus.status); // 'authorized', 'denied', 'notDetermined', or 'restricted'
   ```

   **Mise en œuvre recommandée si access_token est préféré :**
   ```typescript
   async function loginWithFacebook() {
     try {
       // Check ATT status first
       const trackingStatus = await SocialLogin.providerSpecificCall({
         call: 'facebook#requestTracking',
         options: {}
       });
       
       const result = await SocialLogin.login({
         provider: 'facebook',
         options: {
           permissions: ['email', 'public_profile'],
           limitedLogin: trackingStatus.status === 'denied' // Auto-adjust based on ATT
         }
       });
       
       // Handle different response types based on limited login
       if (result.result.accessToken) {
         // Your app logic should work with both limited and full login
         console.log('Login successful:', result);
       }
     } catch (error) {
       console.error('Facebook login error:', error);
     }
   }
   ```

   **Que se passe-t-il en cas de connexion limitée :**
   - **Accès réduit aux données** : certaines données utilisateur peuvent ne pas être disponibles
   - **Différents types de jetons** : les jetons d'accès peuvent avoir différentes capacités
   - **Conformité en matière de confidentialité** : permet de se conformer aux exigences de confidentialité de iOS

   **Important** : Testez toujours votre application avec des scénarios de connexion limitée et complète pour vous assurer que votre application fonctionne correctement dans les deux cas. Vous pouvez en savoir plus sur la connexion limitée [ici] (https://developers.facebook.com/docs/facebook-login/limited-login/).
   :::

3. **Obtenir les données du profil utilisateur**

   Après une connexion réussie, vous pouvez récupérer des informations de profil supplémentaires :

   ```typescript
   async function getFacebookProfile() {
     try {
       const profileResponse = await SocialLogin.providerSpecificCall({
         call: 'facebook#getProfile',
         options: {
           fields: ['id', 'name', 'email', 'first_name', 'last_name', 'picture']
         }
       });

       console.log('Facebook profile:', profileResponse.profile);
       return profileResponse.profile;
     } catch (error) {
       console.error('Failed to get Facebook profile:', error);
       return null;
     }
   }

   // Example usage after login
   async function loginAndGetProfile() {
     const loginResult = await loginWithFacebook();
     if (loginResult) {
       const profile = await getFacebookProfile();
       if (profile) {
         console.log('User ID:', profile.id);
         console.log('Name:', profile.name);
         console.log('Email:', profile.email);
         console.log('Profile Picture:', profile.picture?.data?.url);
       }
     }
   }
   ```

   :::tip
   **Champs de profil disponibles** : vous pouvez demander tous les champs disponibles dans le graphique API de Facebook. Les champs courants incluent : `id`, `name`, `email`, `first_name`, `last_name`, `picture`, `birthday`, `gender`, `location`, `hometown`. Notez que certains champs peuvent nécessiter des autorisations supplémentaires.
   :::

   :::warning
   **Limitation du type de jeton** : l'appel `getProfile` ne fonctionne que lorsque vous disposez d'un **jeton d'accès** (connexion standard avec suivi autorisé). Si l'utilisateur a refusé le suivi ou si vous utilisez une connexion limitée (jeton JWT uniquement), cet appel échouera. Dans ce cas, utilisez les données de profil fournies dans la réponse de connexion initiale.
   :::
</Steps>

## ⚠️ Critique : gestion des jetons backend

:::danger
**COMPORTEMENT CRITIQUE iOS** : la connexion limitée et la transparence du suivi des applications (ATT) sont des fonctionnalités **iOS-ONLY**. Le Android, vous recevrez toujours des jetons d'accès quel que soit le paramètre `limitedLogin`.:::tip
**Essentiel pour iOS** : Vous DEVEZ implémenter les deux méthodes de gestion des jetons. De nombreux développeurs iOS supposent qu'ils obtiendront toujours des jetons d'accès et que leurs applications seront interrompues lorsque les utilisateurs refuseront le suivi.
:::

## Exigences relatives au contexte sécurisé (Web/Capacitor)

### Crypto API Limites

Le flux de connexion Facebook mis à jour nécessite le **Web Crypto API** pour la génération de noms occasionnels, qui n'est disponible que dans des **contextes sécurisés** :

```typescript
// This requires secure context (HTTPS or localhost)
async function sha256(message: string) {
  const msgBuffer = new TextEncoder().encode(message);
  const hashBuffer = await crypto.subtle.digest("SHA-256", msgBuffer); // ❌ Fails in insecure context
  // ...
}
```

### Problèmes d'environnement de développement

**Problème courant** : `ionic serve` avec les URL HTTP interrompt l'authentification Facebook

| Environnement | Crypto API disponible | Facebook Connexion Fonctionne |
|-------------|-----------|-------------------------|
| `http://localhost:3000` | ✅ Oui | ✅ Oui |
| `http://127.0.0.1:3000` | ✅ Oui | ✅ Oui |
| `http://192.168.1.100:3000` | ❌ Non | ❌ Non |
| `https://any-domain.com` | ✅ Oui | ✅ Oui |

### Solutions pour le développement Capacitor

<Steps>
1. **Utilisez localhost pour les tests Web**
   ```bash
   # Instead of ionic serve --host=0.0.0.0
   ionic serve --host=localhost
   ```

2. **Activez HTTPS dans Ionic**
   ```bash
   ionic serve --ssl
   ```

3. **Test sur des appareils réels**
   ```bash
   # Capacitor apps run in secure context on devices
   ionic cap run ios
   ionic cap run android
   ```

4. **Génération occasionnelle alternative pour le développement**
   ```typescript
   async function generateNonce() {
     if (typeof crypto !== 'undefined' && crypto.subtle) {
       // Secure context - use crypto.subtle
       return await sha256(Math.random().toString(36).substring(2, 10));
     } else {
       // Fallback for development (not secure for production)
       console.warn('Using fallback nonce - not secure for production');
       return btoa(Math.random().toString(36).substring(2, 10));
     }
   }
   ```
</Steps>

### Firebase Note d'intégration

La documentation récente Firebase nécessite des jetons JWT avec des noms occasionnels pour l'authentification Facebook, quels que soient les paramètres de connexion. Cette approche fonctionne à la fois avec `limitedLogin: true` et `limitedLogin: false` :

```typescript
   // Both modes can return JWT tokens depending on user choice
   const loginResult = await SocialLogin.login({
     provider: 'facebook',
     options: {
       permissions: ['email', 'public_profile'],
       limitedLogin: false, // true = always JWT, false = depends on user tracking choice
       nonce: nonce
     }
   });
```

:::warning
**Limitation de développement** : si vous utilisez « ionic serve » sur une adresse IP réseau (et non sur un hôte local), la connexion Facebook échouera en raison des restrictions cryptographiques API. Utilisez localhost ou HTTPS pour les tests Web.
:::

:::tip
**Sécurité de la production** : les applications Capacitor sur iOS/Android s'exécutent toujours dans des contextes sécurisés. Cette limitation n'affecte donc que les environnements de développement Web.
:::

## Dépannage

### Problèmes courants et solutions

1. **Erreurs de hachage de clé sur Android**
   - Vérifiez à nouveau que vous avez ajouté le hachage de clé correct au tableau de bord Facebook
   - Pour les versions release, assurez-vous d'avoir ajouté les hachages de clé de débogage et de release
   - Vérifiez que vous utilisez le bon magasin de clés lors de la génération du hachage

2. **Le bouton de connexion Facebook n'apparaît pas**
   - Vérifiez que toutes les entrées du manifeste sont correctes
   - Vérifiez que votre identifiant d'application Facebook et votre jeton client sont corrects.
   - Assurez-vous d'avoir correctement initialisé le SDK

3. **Problèmes courants de iOS**
   - Assurez-vous que toutes les entrées Info.plist sont correctes
   - Vérifiez que les schémas d'URL sont correctement configurés
   - Vérifiez que votre identifiant de forfait correspond à ce qui est enregistré dans le tableau de bord Facebook

### Tests

1. **Avant de tester, ajoutez des utilisateurs de test dans la console du développeur Facebook**
   - Allez dans Rôles > Utilisateurs test
   - Créer un utilisateur test
   - Utilisez ces informations d'identification pour les tests

2. **Testez les versions de débogage et de version**
   - Construction de débogage avec hachage de clé de débogage
   - Version de version avec hachage de clé de version
   - Test sur émulateur et appareils physiquesN'oubliez pas de tester le flux de connexion complet, notamment :
- Connexion réussie
- Annulation de connexion
- Gestion des erreurs
- Fonctionnalité de déconnexion
