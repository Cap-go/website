##Présentation

Ce guide vous aidera à intégrer la connexion Google avec l'authentification Firebase sur Android. Je suppose que vous avez déjà terminé la [configuration générale Firebase Google](./general/) 

:::note
Je suppose que vous n'avez pas encore créé votre application Android dans la console Firebase. Si c’est le cas, vos étapes seront légèrement différentes.
:::

## Étapes de configuration

<Steps>
1. Accédez à la présentation de votre projet sur [console.cloud.google.com](https://console.cloud.google.com/)
    <img src="/social-login-assets/firebase_project_overview.webp" alt="Firebase Aperçu du projet" />

2. Cliquez sur le bouton « Ajouter une application »
    <img src="/social-login-assets/firebase_project_add_app.webp" alt="Firebase Ajouter un bouton d'application" />

    :::remarque
    Il est possible que vous deviez chercher ce bouton ici. Cela s'applique uniquement si vous avez déjà créé une autre application dans la console Firebase.
    :::

    <img src="/social-login-assets/firebase_project_add_app_2.webp" alt="Firebase Ajouter un bouton d'application" />

3. Sélectionnez `Android`
    <img src="/social-login-assets/firebase_project_add_app_android.webp" alt="Firebase Ajouter une application Android Bouton" />

4. Remplissez la première partie du formulaire
   1. Remplissez le `Android package name`
      1. Ouvrez Android Studio sur votre application en utilisant `npx cap open android`
      2. Tout en bas du navigateur, recherchez les « Gradle Scripts »
         <img src="/social-login-assets/studio_gradle_scripts.png" alt="Section Gradle Scripts dans le navigateur de projet Android Studio" />
      3. Recherchez `build.gradle` pour le module `app`
         <img src="/social-login-assets/studio_build_gradle.png" alt="Fichier build.gradle (Module : app) dans la section Gradle Scripts" />
      4. Copiez le `android.defaultConfig.applicationId`. Ce sera votre « nom de package » dans la console Firebase
         <img src="/social-login-assets/studio_build_gradle_app_id.png" alt="Fichier Build.gradle montrant la configuration de l'ID d'application" />
      :::remarque
      L'identifiant affiché ici sera différent de celui que j'utiliserai pour le reste du guide. J'utiliserai `app.capgo.plugin.SocialLogin` pour le reste du guide.
      :::
      5. Collez-le dans la console Firebase
          <img src="/social-login-assets/firebase_project_add_app_android_package_name.webp" alt="Firebase Ajouter une application Android Champ de nom du package" />
    2. Cliquez sur le bouton « Enregistrer l'application »
        <img src="/social-login-assets/firebase_project_add_app_android_register.webp" alt="Firebase Ajouter une application Android Bouton d'enregistrement" />
5. Ignorez l'étape « Télécharger puis ajouter le fichier de configuration »
    <img src="/social-login-assets/firebase_project_add_app_android_skip_download.webp" alt="Firebase Ajouter une application Android Ignorer « Télécharger puis ajouter le fichier de configuration »" />
6. Ignorez l'étape `Add firebase SDK`
   <img src="/social-login-assets/firebase_project_add_app_android_skip_download_firebase_sdk.webp" alt="Firebase Ajouter une application Android Ignorer « Ajouter Firebase SDK »" />
7. Cliquez sur le bouton « Continuer vers la console »
   <img src="/social-login-assets/firebase_project_add_app_android_continue_to_console.webp" alt="Firebase Ajouter une application Android Bouton Continuer vers la console" />
8. Si vous n'êtes pas automatiquement authentifié, accédez à `settings` -> `general` -> `vos applications` -> recherchez votre application Android et cliquez dessus.
   <img src="/social-login-assets/firebase_project_add_app_android_settings_general_your_apps.webp" alt="Firebase Ajouter une application Android Paramètres Général Bouton Vos applications" />
9. Obtenez votre empreinte digitale SHA1
   
   Suivez les étapes 10 à 11 du [Google Login Android guide de configuration](/docs/plugins/social-login/google/android/#using-google-login-on-android) :
   
   1. Maintenant, ouvrez le terminal. Assurez-vous que vous êtes dans le dossier `android` de votre application et exécutez `./gradlew signInReport`
      <img src="/social-login-assets/term_sign_report.png" alt="Terminal affichant la commande Gradlew SignInReport" />
   
   2. Faites défiler vers le haut de cette commande. Vous devriez voir ce qui suit. Copiez le `SHA1`.
      <img src="/social-login-assets/term_sign_report_res.png" alt="Sortie du terminal affichant l'empreinte digitale du certificat SHA1" />

   :::remarque
   J'utiliserai un SHA1 légèrement différent pour le reste du guide, car j'ai modifié les calculs depuis la rédaction du guide de configuration Android de connexion original Google.
   :::

   :::attention
   Le SHA1 est très important pour réussir. Si vous vous trompez, l'authentification échouera de manière étrange.
   Veuillez __**[LIRE LE GUIDE DE CONFIGURATION ANDROID DE CONNEXION GOOGLE](/docs/plugins/social-login/google/android/#using-google-login-on-android)**__ pour bien faire les choses.
   :::
   
10. Ajoutez le SHA1 au projet Firebase
      1. Cliquez sur le bouton « Ajouter une empreinte digitale »
         <img src="/social-login-assets/firebase_project_add_app_android_add_fingerprint.webp" alt="Firebase Ajouter une application Android Ajouter un bouton d'empreinte digitale" />
      2. Collez le SHA1 que vous avez copié à l'étape précédente
         <img src="/social-login-assets/firebase_project_add_app_android_add_fingerprint_sha1.webp" alt="Firebase Ajouter une application Android Ajouter un champ SHA1 d'empreinte digitale" />
      3. Cliquez sur le bouton `Save`
         <img src="/social-login-assets/firebase_project_add_app_android_add_fingerprint_add.webp" alt="Firebase Ajouter une application Android Ajouter un bouton d'enregistrement d'empreintes digitales" />11. Obtenez votre identifiant client Web
      1. Accédez à `Build` -> `Authentication`
         <img src="/social-login-assets/firebase_select_authentication.webp" alt="Firebase Menu d'authentification" />
      2. Cliquez sur le bouton « Méthode de connexion »
         <img src="/social-login-assets/firebase_select_authentication_sign_in_method.webp" alt="Firebase Bouton de méthode de connexion d'authentification" />
      3. Cliquez sur le fournisseur `Google`
         <img src="/social-login-assets/firebase_select_authentication_sign_in_method_google.webp" alt="Firebase Méthode de connexion par authentification Google Fournisseur" />
      4. Cliquez sur le bouton `Web SDK configuration`
         <img src="/social-login-assets/firebase_select_authentication_sign_in_method_web_sdk_configuration.webp" alt="Firebase Méthode de connexion par authentification Web SDK Bouton de configuration" />
      5. Copiez l'« ID client Web ». Ce sera votre `webClientId` dans la méthode `initialize` du plugin.
        <img src="/social-login-assets/firebase_select_authentication_sign_in_method_web_sdk_configuration_web_client_id.webp" alt="Firebase Méthode de connexion par authentification Web SDK Configuration ID client Web" />
12. Utilisez l'ID du client Web dans JS.
      :::remarque
      Je recommande d'utiliser la fonction d'assistance `authenticateWithGoogle` disponible dans le fichier [authUtils.ts](https://github.com/Cap-go/capacitor-social-login/blob/main/example-app/src/authUtils.ts) de l'exemple d'application.
      :::
</Steps>

À ce stade, vous êtes **__TECHNIQUEMENT__** prêt à utiliser Google Connectez-vous avec Firebase Authentification sur Android. Cependant, je recommanderais de revérifier la configuration dans la console Cloud Google, comme expliqué à l'étape suivante.

## Vérifiez à nouveau la configuration dans Google Cloud Console

Afin de vous assurer que la configuration est correcte, vous devez revérifier la configuration dans la console Cloud Google.
<Steps>
   1. Accédez à [console.cloud.google.com](https://console.cloud.google.com/)
   2. Trouvez votre projet
      1. Cliquez sur le sélecteur de projet
         <img src="/social-login-assets/firebase_double_check_gc_project_select.webp" alt="Google Sélecteur de projets Cloud Console" />
      2. Recherchez votre projet par le nom exact de votre projet Firebase et cliquez dessus. Dans mon cas, il s'agit de `sociallogin-tutorial-app`.
         <img src="/social-login-assets/firebase_gc_project_select_project.webp" alt="Firebase Sélecteur de projet Projet" />
   3. Ouvrez la barre de recherche et ouvrez `credentials`
      1. Ouvrez la barre de recherche
         <img src="/social-login-assets/firebase_double_check_gc_search_bar.webp" alt="Google Barre de recherche Cloud Console" />
      2. Recherchez `credentials` et cliquez sur celui « API et services » (numéro 2 sur la capture d'écran)
         <img src="/social-login-assets/firebase_gc_credentials_search.webp" alt="Google Recherche d'identifiants Cloud Console" />
   4. Vérifiez que vous voyez à la fois les ID Android et client Web dans la liste.
      <img src="/social-login-assets/firebase_gc_credentials_list.webp" alt="Google Liste des informations d'identification de Cloud Console" />

      :::attention
      Si vous ne voyez pas les identifiants Android et client Web dans la liste, vous avez commis une erreur lors de la configuration. Veuillez revenir en arrière et vérifier vos étapes.

      Il est également possible, et cela m'est arrivé, que vous ayez déjà ajouté le hachage Android SHA1 avec le même ID d'application dans un projet différent. Cela empêchera Firbase de créer un ID client Android. Dans ce cas, vous devrez supprimer le SHA1 de l'autre projet ainsi que sur Firebase (en utilisant la console Firebase pour supprimer l'application android) et le recréer sur Firbase
      :::
   5. Vérifiez que l'ID client Android est correctement configuré dans la console Firebase.
      1. Cliquez sur l'application `Android`
         <img src="/social-login-assets/firebase_gc_android_app_select.webp" alt="Google Cloud Console Android Sélection d'application" />
      2. Confirmez que le hachage SHA1 est correctement configuré et qu'il correspond à celui que vous avez copié lors des étapes précédentes.
         <img src="/social-login-assets/firebase_gc_android_app_sha1_configured.webp" alt="Google Cloud Console Android Application SHA1 configurée" />
   6. Vérifiez que l'ID client Web est correctement configuré dans la console Firebase.
      1. Cliquez sur l'application `Web`
         <img src="/social-login-assets/firebase_gc_web_app_select.webp" alt="Google Sélection de l'application Web Cloud Console" />
      2. Confirmez que l'ID client correspond à celui que vous avez copié lors des étapes précédentes.
         <img src="/social-login-assets/firebase_gc_web_app_client_id_configured.webp" alt="" />
      :::remarque
      Veuillez ignorer le reste des paramètres du client Web. Nous en discuterons dans le [guide de configuration Web](/docs/plugins/social-login/firebase/google/web/).
</Steps>Voilà ! Vous êtes maintenant prêt à utiliser la connexion Google avec l'authentification Firebase sur Android.

## Dépannage

Si l'authentification se bloque ou échoue :

- Vérifiez que l'audience `idToken` correspond à votre identifiant de client Web Firebase.
- Vérifiez que la connexion Google est activée dans la console Firebase.
- Assurez-vous que l'empreinte digitale SHA-1 est correctement configurée
- Consultez l'[exemple de code d'application](https://github.com/Cap-go/capacitor-social-login/tree/main/example-app/src/authUtils.ts) pour référence.
