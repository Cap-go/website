La connexion Apple sur Android est piratée. Apple n'a pas de support officiel pour `Sign in with Apple` sur Android, la solution est donc légèrement hackée.

Android utilise actuellement des onglets Chrome pour afficher un site Web OAuth2. Cette approche présente les défis suivants :

- Configuration difficile
- Un backend est requis

## Comprendre le flux sur Android.

Permettez-moi d'utiliser un diagramme pour expliquer le flux sur Android :

<MermaidGraph graph={appleLoginFlowGraph} ariaLabel="Apple Login flow diagram on Android" fullWidth={false} />

Maintenant que vous êtes conscient des défis et du déroulement, commençons la configuration.

## Création de l'ID du service

<Steps>
1. Connectez-vous au [Apple Portail des développeurs](https://developer.apple.com). 

2. Cliquez sur `Identifiers`.

    <img src="/social-login-assets/apple_dev_portal_iden.png" alt="Apple Section Identifiants du portail des développeurs" />
    
    Vous devriez voir un écran qui ressemble à ceci :

    <img src="/social-login-assets/apple_dev_portal_iden_2.png" alt="Apple Écran des identifiants du portail de développeur" />

    1. Assurez-vous que ce champ indique <code>App IDs</code>
    2. Assurez-vous que vous pouvez trouver votre identifiant d'application.
        :::remarque
        Si vous n'avez pas configuré la connexion Apple pour IOS, vous devrez en créer une. Pour ma part, j'en ai déjà un créé. L'ID d'application que j'utiliserai est `me.wcaleniewolny.test.ionic.vue`. Si vous n'en avez pas, veuillez en créer un à l'aide de l'[étape de création d'application](#creating-the-app).
        :::
3. Assurez-vous que la fonctionnalité `Sign in with Apple` est activée pour votre application
    1. Cliquez sur votre application
        <img src="/social-login-assets/apple_dev_click_on_app.png" alt="Sélection de votre application dans la liste" />
    2. Assurez-vous que la fonctionnalité `Sign in with Apple` est activée
        <img src="/social-login-assets/apple_dev_sign_in_with_apple_enabled.png" alt="Connectez-vous avec la case à cocher la fonctionnalité Apple activée." />
    3. S'il n'est pas activé, activez-le.
4. Revenez à tous les « Tous les identifiants » 
    <img src="/social-login-assets/apple_dev_go_back_iden.png" alt="Bouton de navigation Tous les identifiants" />
5. Cliquez sur « App Ids » et accédez à « Services IDs »
   <img src="/social-login-assets/apple_dev_go_to_services_id.png" alt="Navigation vers la section ID de services" />
6. Créez un nouvel identifiant
    1. Cliquez sur le bouton plus
        <img src="/social-login-assets/apple_dev_iden_add.png" alt="Bouton Ajouter un nouveau numéro de service" />
    
    2. Sélectionnez « ID de service » et cliquez sur `Continue`.
        <img src="/social-login-assets/apple_dev_service_and_cont.png" alt="" />
    
    3. Entrez une description et un identifiant et cliquez sur `Continuie`.

        <img src="/social-login-assets/apple_dev_reg_service_2.png" alt="Saisie des détails de l'ID de service" />

        :::remarque
        Ce `identifiers` deviendra le `clientId` que vous passerez dans la fonction `initialize` ET `ANDROID_SERVICE_ID` pour le backend. 
        
        **S'il vous plaît, enregistrez-le !!!**
        :::

        :::remarque
        L'ID de service ne doit pas nécessairement correspondre à l'ID de l'application, mais je recommande de définir l'ID de service sur `YOUR_APP_ID.service` . Pour rappel, j'utilise `me.wcaleniewolny.test.ionic.vue` pour mon identifiant d'application mais j'utilise `ee.forgr.io.ionic.service2` comme identifiant de service.
        :::
    4. Veuillez vérifier les détails et cliquer sur `Register`
        <img src="/social-login-assets/apple_dev_service_ref_fin.png" alt="Confirmation de l'enregistrement de l'ID de service" />

        <img src="/social-login-assets/apple_dev_open_serv.png" alt="Sélection de l'ID de service nouvellement créé" />
    
    6. Activez l'option `Sign in with Apple`
        <img src="/social-login-assets/apple_dev_serv_enable_sign_with_apple.png" alt="Activation de la connexion avec Apple pour l'ID de service" />
    
    7. Configurez le `Sign In with Apple`
        <img src="/social-login-assets/apple_dev_conf_serv_sign_with_apple.png" alt="Configurer le bouton pour se connecter avec Apple" />
    
    8. Assurez-vous que « l'ID d'application principal » est défini sur l'ID d'application configuré à l'étape précédente.

Pour les besoins de ce tutoriel, j'utiliserai `https://xyz.wcaleniewolny.me/login/callback` pour l'URL de retour et `xyz.wcaleniewolny.me` le domaine. Appuyez sur suivant.

## Configurer la redirection de l'application

Comme vous l'avez vu dans le diagramme, le backend effectue une étape appelée « Rediriger vers l'application ». Cela nécessite des modifications manuelles de votre application.

<Steps>
    1. Modifiez le `AndroidManifest.xml`
        1. Ouvrez le fichier, j'utiliserai `AndroidStudio`
            <img src="/social-login-assets/studio_android_manifest_file.png" alt="Fichier AndroidManifest.xml dans Android Studio" />
        2. Recherchez le `MainActivity` et ajoutez le filtre d'intention suivant
            <img src="/social-login-assets/studio_manifest_code_to_add.png" alt="Code de filtre d'intention à ajouter dans MainActivity" />

            ```xml
            <intent-filter>
                <action android:name="android.intent.action.VIEW" />
                <category android:name="android.intent.category.DEFAULT" />
                <category android:name="android.intent.category.BROWSABLE" />
                <data android:scheme="capgo-demo-app" android:host="path" />
            </intent-filter>
            ```
    2. Modifiez le `MainActivity`
        1. Veuillez ouvrir le `MainActivity`
            <img src="/social-login-assets/studio_main_activ_file.png" alt="Fichier MainActivity.java dans Android Studio" />

        2. Ajoutez le code suivant :
            <img src="/social-login-assets/studio_main_actv_new_code.png" alt="Code à ajouter à MainActivity pour gérer les liens profonds" />

            ```java
            @Override
            protected void onNewIntent(Intent intent) {
                String action = intent.getAction();
                Uri data = intent.getData();
            
                if (Intent.ACTION_VIEW.equals(action) && data != null) {
                    PluginHandle pluginHandle = getBridge().getPlugin("SocialLogin");
                    if (pluginHandle == null) {
                        Log.i("Apple Login Intent", "SocialLogin login handle is null");
                        return;
                    }
                    Plugin plugin = pluginHandle.getInstance();
                    if (!(plugin instanceof SocialLoginPlugin)) {
                        Log.i("Apple Login Intent", "SocialLogin plugin instance is not SocialLoginPlugin");
                        return;
                    }
                    ((SocialLoginPlugin) plugin).handleAppleLoginIntent(intent);
                    return;
                }
                super.onNewIntent(intent);
            }
            ```

            :::attention
            Cet exemple suppose qu’aucun lien profond n’est configuré. Si c'est le cas, veuillez ajuster le code
            :::
</Steps>

:::note
Vous venez d'ajouter un nouveau lien profond dans votre application. Le lien profond ressemblera à ceci : `capgo-demo-app://path`. Vous pouvez modifier le `android:scheme` et le `android:host` pour modifier l'apparence de ce lien profond.
**Important :** Dans la configuration backend, ce lien profond deviendra `BASE_REDIRECT_URL`
:::


## Configuration du back-end

Un backend est requis pour Android, mais la configuration d'un backend aura également un impact sur IOS. Un exemple de backend est fourni [ici](https://github.com/WcaleNieWolny/capgo-social-login-backend-demo/blob/main/index.ts)Cet exemple fournit les éléments suivants :
- Une simple base de données JSON
- Un moyen de demander le JWT aux serveurs de Apple
- Une simple vérification JWT

:::note
J'utilise `PM2` afin d'héberger cet exemple. Un exemple `ecosystem.config.js` peut être trouvé [ici](https://github.com/WcaleNieWolny/capgo-social-login-backend-demo/blob/main/ecosystem.config.js.example)
:::

Compte tenu de tout ce que j'ai dit dans ce tutoriel, voici à quoi ressemblerait la section `env` :

- `ANDROID_SERVICE_ID` = ID de service
- `IOS_SERVICE_ID` = identifiant de l'application

```js
env: {
  PRIVATE_KEY_FILE: "AuthKey_U93M8LBQK3.p8",
  KEY_ID: "U93M8LBQK3",
  TEAM_ID: "UVTJ336J2D",
  ANDROID_SERVICE_ID: "ee.forgr.io.ionic.starter.service2",
  IOS_SERVICE_ID: "me.wcaleniewolny.test.ionic.vue",
  PORT: 3000,
  REDIRECT_URI: "https://xyz.wcaleniewolny.me/login/callback",
  BASE_REDIRECT_URL: "capgo-demo-app://path"
}
```

#### Utilisation du plugin

L'utilisation de la fonction `login` ne change pas, c'est la même que celle d'IOS. Veuillez consulter cette section pour plus d'informations. **CEPENDANT**, la méthode `initialize` change un peu.

```typescript
await SocialLogin.initialize({
  apple: {
    clientId: 'ee.forgr.io.ionic.starter.service2',
    redirectUrl: 'https://appleloginvps.wcaleniewolny.me/login/callback'
  }
})
```

:::danger
    Notez que l'ajout de `redirectUrl` **EST** affectera IOS !!!!!
:::
