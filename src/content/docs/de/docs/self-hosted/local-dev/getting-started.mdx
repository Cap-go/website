## Was behandelt dieses Tutorial?

Dieses Tutorial zeigt, wie man Supabase von Grund auf startet und Edge-Funktionen einrichtet

## Voraussetzungen
 1 Geklontes [capgo](https://githubcom/Cap-go/capgo/)
 3 [supabase](https://supabasecom/)

## Erste Schritte

Zum Starten führen Sie aus:
```bash
supabase start
```

Als Nächstes sollten Sie etwa Folgendes sehen:
```js
Started supabase local development setup

         API URL: http://localhost:54321
     GraphQL URL: http://localhost:54321/graphql/v1
          DB URL: postgresql://postgres:postgres@localhost:54322/postgres
      Studio URL: http://localhost:54323
    Inbucket URL: http://localhost:54324
      JWT secret: [truncated]
        anon key: supa_key_anon
service_role key: supa_key_admin
```

Öffnen Sie als Nächstes `configsjson` und setzen Sie die folgenden Werte:
```json
{
  "base_domain": {
    "prod": "webcapgoapp",
    "development": "developmentwebcapgoapp",
    "local": "localhost:3332"
  },
  "supa_anon": {
    "prod": "supa_key_anon",
    "development": "supa_key_anon",
    "local": "supa_key_anon"
  },
  "supa_url": {
    "prod": "http://localhost:54321",
    "development": "http://localhost:54321",
    "local": "http://localhost:54321"
  }
}
```

wobei `supa_key_anon` der Wert aus dem vorherigen Schritt ist

:::danger
⚠️ Committen Sie `configsjson` nicht in das Remote-Repository
:::

Überprüfen Sie als Nächstes, ob Sie auf [localhost:54323](http://localhost:54323/projects/) zugreifen können und ob die Tabelle `users` etwa so aussieht

<figure><img src="/supabasewebp" alt="supabase" /><figcaption></figcaption></figure>

Wenn ja, starten Sie Edge-Funktionen mit:

```bash
supabase functions serve
```

und starten Sie das Frontend mit:
```bash
bun run serve
```
