## Warum beitragen?

Zunächst einmal vielen Dank, dass Sie einen Beitrag zu capgo Open-Source-Projekten in Erwägung ziehen! Es sind Menschen wie Sie, die capgo Open-Source-Projekte zu solch großartigen Werkzeugen machen.

Hier sind einige Gründe, warum Sie beitragen sollten:

* Die Mitarbeit an capgo Open-Source-Projekten ist eine großartige Möglichkeit, Geld durch die [zahlreichen Prämien](https://consolealgoraio/org/Capgo/) zu verdienen, die vom capgo-Team angeboten werden
* Die Mitarbeit an capgo Open-Source-Projekten ist eine großartige Möglichkeit, eine Funktion hinzuzufügen, die Sie gerne sehen würden
* Die Mitarbeit an capgo Open-Source-Projekten ist eine großartige Möglichkeit, einen Bug zu beheben, auf den Sie gestoßen sind
* Die [Hauptlizenz von Capgo](https://githubcom/Cap-go/capgo/blob/main/LICENSE/) verlangt, dass Sie alle Änderungen, die Sie daran vornehmen, als Open Source veröffentlichen. Durch Ihren Codebeitrag können Sie Ihre Änderungen als Open Source beibehalten und andere davon profitieren lassen

## Wie man beiträgt

* Zunächst müssen Sie das Repository forken, zu dem Sie beitragen möchten
* Als Zweites müssen Sie Ihre Änderungen committen und in dieses Repository pushen
* Zuletzt müssen Sie einen Pull Request öffnen
* Das war's! Jetzt müssen Sie nur noch warten, bis das capgo-Team Ihren PR überprüft
