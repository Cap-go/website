## Warum beitragen?

Zunächst einmal, vielen Dank, dass Sie in Betracht ziehen, zu den Open-Source-Projekten von capgo beizutragen! Es sind Menschen wie Sie, die die Open-Source-Projekte von capgo zu großartigen Werkzeugen machen.

Hier sind einige Gründe, warum Sie beitragen sollten:

1. Zu den Open-Source-Projekten von capgo beizutragen, ist eine großartige Möglichkeit, etwas Geld durch die [unzähligen Belohnungen](https://consolealgoraio/org/Capgo/) zu verdienen, die vom capgo-Team angeboten werden.
2. Zu den Open-Source-Projekten von capgo beizutragen, ist eine großartige Möglichkeit, eine Funktion hinzuzufügen, die Sie gerne sehen würden.
3. Zu den Open-Source-Projekten von capgo beizutragen, ist eine großartige Möglichkeit, einen Fehler zu beheben, auf den Sie gestoßen sind.
4. Die [Lizenz von Capgo main](https://githubcom/Cap-go/capgo/blob/main/LICENSE/) verlangt, dass Sie alle Änderungen, die Sie daran vornehmen, open source machen. Indem Sie Ihren Code beitragen, können Sie Ihre Änderungen open source halten und anderen die Nutzung ermöglichen.

## Wie man beiträgt

1. Zunächst müssen Sie das Repository, zu dem Sie beitragen möchten, forken.
2. Zweitens müssen Sie Ihre Änderungen an diesem Repository committen und pushen.
3. Schließlich müssen Sie einen Pull-Request öffnen.
4. Das war's! Jetzt müssen Sie nur noch auf das Review Ihres PRs durch das capgo-Team warten.
