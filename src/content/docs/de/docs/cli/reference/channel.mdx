Der Befehl `channel` ermöglicht es Ihnen, Ihre Release-Kanäle zu verwalten.

### hinzufügen

`npx @capgo/cli channel add [channelId] [appId]`

Erstellt einen neuen Kanal für eine App.

Optionen:

- `-d, --default`: Setzen Sie den neuen Kanal als Standardkanal
- `-a, --apikey <apikey>`: API-Schlüssel, um ihn mit Ihrem Konto zu verknüpfen

### löschen

`npx @capgo/cli channel delete [channelId] [appId]`

Löscht einen Kanal von einer App.

Optionen:

- `-a, --apikey <apikey>`: API-Schlüssel, um ihn mit Ihrem Konto zu verknüpfen
- `--delete-bundle`: Löschen Sie das mit dem Kanal verbundene Bundle

### auflisten

`npx @capgo/cli channel list [appId]`

Listet alle Kanäle für eine App auf.

Optionen:

- `-a, --apikey <apikey>`: API-Schlüssel, um ihn mit Ihrem Konto zu verknüpfen

### aktuellesBundle

`npx @capgo/cli channel currentBundle [channel] [appId]`

Erhalten Sie das aktuelle Bundle für einen bestimmten Kanal.

Optionen:

- `-c, --channel <channel>`: Kanal, von dem das aktuelle Bundle abgerufen werden soll
- `-a, --apikey <apikey>`: API-Schlüssel, um ihn mit Ihrem Konto zu verknüpfen
- `--quiet`: Nur die Bundle-Version drucken

### festlegen

`npx @capgo/cli channel set [channelId] [appId]`

Setzt die Eigenschaften eines Kanals.

Optionen:

- `-a, --apikey <apikey>`: API-Schlüssel, um ihn mit Ihrem Konto zu verknüpfen
- `-b, --bundle <bundle>`: Bundle-Version, die für den Kanal festgelegt werden soll
- `-s, --state <state>`: Setzen Sie den Status des Kanals (`default` oder `normal`)
- `--latest`: Verwenden Sie die neueste Version aus `packagejson` als Bundle-Version
- `--downgrade`: Erlauben Sie Downgrades auf Versionen unter der nativen Version
- `--no-downgrade`: Deaktivieren Sie Downgrades auf Versionen unter der nativen Version
- `--upgrade`: Erlauben Sie Upgrades auf Versionen über der nativen Version
- `--no-upgrade`: Deaktivieren Sie Upgrades auf Versionen über der nativen Version
- `--ios`: Erlauben Sie das Senden von Updates an iOS-Geräte
- `--no-ios`: Deaktivieren Sie das Senden von Updates an iOS-Geräte
- `--android`: Erlauben Sie das Senden von Updates an Android-Geräte
- `--no-android`: Deaktivieren Sie das Senden von Updates an Android-Geräte
- `--self-assign`: Erlauben Sie Geräten, sich selbst diesem Kanal zuzuweisen
- `--no-self-assign`: Deaktivieren Sie die Selbstzuweisung von Geräten zu diesem Kanal
- `--disable-auto-update <disableAutoUpdate>`: Deaktivieren Sie die Strategie zur automatischen Aktualisierung für diesen Kanal (Optionen: `major`, `minor`, `metadata`, `patch`, `none`)
- `--dev`: Erlauben Sie das Senden von Updates an Entwicklungsgeräte
- `--no-dev`: Deaktivieren Sie das Senden von Updates an Entwicklungsgeräte
- `--emulator`: Erlauben Sie das Senden von Updates an Emulatorgeräte
- `--no-emulator`: Deaktivieren Sie das Senden von Updates an Emulatorgeräte
- `--package-json <packageJson>`: Komma-getrennte Liste von Pfaden zu `packagejson`-Dateien (nützlich für Monorepos)
