Verwenden Sie die Live-Updates-Funktion von Capgo, um die JavaScript-Bundles Ihrer App aus der Ferne in Echtzeit zu aktualisieren. Drücken Sie JS-Updates direkt an Ihre Benutzer, ohne den Überprüfungsprozess im App Store durchlaufen zu müssen, um sofort Fehler zu beheben und neue Funktionen bereitzustellen.

<Aside>

Live-Updates sind auf Änderungen von JavaScript-Bundles beschränkt. Wenn Sie nativen Code aktualisieren müssen, z. B. ein Plugin hinzufügen oder entfernen oder die native Projektkonfiguration ändern, müssen Sie ein neues natives Binär-Build in die App-Stores einreichen.

</Aside>

## So funktionieren Live-Updates

Das Live-Update-System von Capgo hat zwei wichtige Komponenten:

1. Das Capgo SDK, das Sie in Ihrer App installieren. Das SDK überprüft verfügbare Updates und lädt sie im Hintergrund herunter.

2. Kanäle, mit denen Sie Updates an bestimmte Benutzergruppen richten können. Sie können Kanäle verwenden, um verschiedene Release-Tracks zu verwalten, wie `Production`, `Staging` und `Dev`.

Wenn Sie ein neues JS-Bundle zu Capgo hochladen und es einem Kanal zuweisen, erkennt das Capgo SDK in Apps, die für diesen Kanal konfiguriert sind, das Update und lädt es herunter. Beim nächsten Neustart der App wird das neue Bundle geladen.

## Erste Schritte

Um Live-Updates zu verwenden, folgen Sie diesen Schritten:

<Steps>

1. Schließen Sie den [Capgo Quickstart](/docs/getting-started/quickstart) ab, um Ihre App in Capgo einzurichten und das Capgo SDK zu installieren.

2. Rufen Sie in Ihrem App-Code `CapacitorUpdaternotifyAppReady()` auf, nachdem Ihre App die Initialisierung abgeschlossen hat. Dies teilt dem Capgo SDK mit, dass Ihre App bereit ist, Updates zu empfangen.

3. Erstellen Sie Ihr JS-Bundle und laden Sie es zu Capgo hoch:
   ```shell
   npm run build
   npx @capgo/cli@latest upload --channel=Production 
   ```

4. Öffnen Sie Ihre App und warten Sie auf den Download des Updates. Sie können den Status mit folgendem Befehl überprüfen:
   ```shell
   npx @capgo/cli@latest app debug
   ```

5. Sobald das Update heruntergeladen wurde, schließen Sie Ihre App und öffnen Sie sie neu, um das neue Bundle zu laden.

</Steps>

Siehe den Leitfaden [Bereitstellung von Live-Updates](/docs/getting-started/deploy) für weitere Details.

## Das Capgo CLI

Das Capgo CLI ist ein leistungsstarkes Tool, das Entwicklern ermöglicht, mit den Diensten von Capgo aus ihren eigenen CI/CD-Pipelines zu interagieren. Mit dem CLI haben Sie granularen Kontrolle darüber, wann Builds erstellt und bereitgestellt werden, was Ihnen ermöglicht, Capgo in Ihre bestehenden Unternehmens-Workflows zu integrieren.

### Für was ist das Capgo CLI?

Das Capgo CLI wurde für Entwickler und Teams entwickelt, die mehr Kontrolle und Flexibilität in ihren Live-Update-Workflows benötigen. Durch die Verwendung des CLI in Ihren CI/CD-Pipelines können Sie:

- Genau bestimmen, wann Updates erstellt und bereitgestellt werden, anstatt sich auf die integrierte Automatisierung von Capgo zu verlassen.
- Ihre eigenen Prozesse, wie Code-Signierung, QA-Tests oder Genehmigungen von Managern, zwischen den Schritten „Build“ und „Deploy“ einfügen.
- Capgo in Ihre bestehenden DevOps-Tools und -Workflows integrieren.

### Authentifizierung

Um das Capgo CLI zu verwenden, müssen Sie sich mit Ihrem API-Schlüssel authentifizieren. Sie können einen API-Schlüssel in den Kontoeinstellungen von Capgo generieren.

Um sich anzumelden und Ihren API-Schlüssel sicher zu speichern, führen Sie aus:

```shell
npx @capgo/cli@latest login [API_KEY]
```

Dieser Befehl wird dann für die zukünftige Verwendung gespeichert. Sie müssen Ihren API-Schlüssel nach der Anmeldung nicht bei jedem Befehl angeben.

### Wichtige Unterschiede zu anderen CLI-Tools

Wenn Sie mit anderen Live-Update-CLI-Tools vertraut sind, gibt es einige wichtige Punkte zu beachten, wenn es um das CLI von Capgo geht:

- Capgo verwendet eine einzige CLI für sowohl Entwicklungs- als auch CI/CD-Einsatzszenarien, da sich Capgo ausschließlich auf das Set von Live-Update-Funktionen konzentriert.

- Das Capgo CLI erfordert keinen separaten Installationsschritt. Es ist im Paket `@capgo/cli` enthalten und kann direkt mit `npx` ausgeführt werden.

- Das CLI von Capgo wurde speziell für den Live-Update-Workflow entwickelt, sodass es möglicherweise nicht einige Funktionen oder Befehle enthält, die in allgemeineren CLI-Tools zu finden sind.
