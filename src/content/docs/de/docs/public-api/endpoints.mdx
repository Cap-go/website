## Organisationen

Dieser Endpunkt ermöglicht die Verwaltung von Organisationen und deren Mitgliedern

### GET

`https://apicapgoapp/organization`

Ruft Organisationsinformationen ab. Wenn `orgId` in den Parametern angegeben ist, wird eine einzelne Organisation zurückgegeben. Andernfalls werden alle zugänglichen Organisationen zurückgegeben

```typescript
interface Organization {
  id: string
  created_by: string
  created_at: string
  updated_at: string
  logo: string | null
  name: string
  management_email: string
  customer_id: string | null
}
```

Gibt zurück:
```typescript
return Organization[] | Organization
```

### POST

`https://apicapgoapp/organization`

Aktualisiert eine bestehende Organisation. Benötigt Admin-Rolle

```typescript
interface OrganizationUpdate {
  orgId: string
  logo?: string
  name?: string
  management_email?: string
}
```

Erfordert einen Request-Body vom Typ `OrganizationUpdate`

Bei Erfolg wird Folgendes zurückgegeben:

```typescript
return { status: 'Organization updated', data: Organization }
```

### PUT

`https://apicapgoapp/organization`

Erstellt eine neue Organisation

```typescript
interface OrganizationCreate {
  name: string
}
```

Erfordert einen Request-Body vom Typ `OrganizationCreate`

Bei Erfolg wird Folgendes zurückgegeben:

```typescript
return { status: 'Organization created', id: string }
```

## API-Schlüssel

Dieser Endpunkt ermöglicht die Verwaltung von API-Schlüsseln für den Zugriff auf die Capgo API

### GET

`https://apicapgoapp/apikey`

Ruft API-Schlüssel für Ihr Konto ab. Gibt alle zugänglichen API-Schlüssel zurück

```typescript
interface ApiKey {
  created_at: string | null
  id: number
  key: string
  mode: 'read' | 'write' | 'upload' | 'all'
  name: string
  updated_at: string | null
  user_id: string
}
```

Gibt zurück:
```typescript
return ApiKey[]
```

### POST

`https://apicapgoapp/apikey`

Erstellt einen neuen API-Schlüssel für eine bestimmte Organisation

Query-Parameter:
```typescript
interface ApiKeyCreate {
  org_id: string
  mode: 'read' | 'write' | 'upload' | 'all'
}
```

Gibt zurück:
```typescript
return { apikey: ApiKey }
```

### DELETE

`https://apicapgoapp/apikey/:id`

Löscht einen bestehenden API-Schlüssel

Parameter:
- `id`: Der zu löschende API-Schlüssel

Bei Erfolg wird zurückgegeben:
```typescript
return { success: true }
```

Bei Fehler wird zurückgegeben:
```typescript
return { error: string, supabaseError?: any }
```

## Mitglieder (/organization/members)

### GET

Ruft Organisationsmitglieder ab

Erfordert einen Request-Body (Query-Parameter) vom Typ `{ orgId: string }`

Bei Erfolg wird Folgendes zurückgegeben:
```typescript
interface Member {
  uid: string;
  email: string;
  image_url: string;
  role: "invite_read" | "invite_upload" | "invite_write" | "invite_admin" | "invite_super_admin" | "read" | "upload" | "write" | "admin" | "super_admin";
}
return Member[]
```

### DELETE

`https://apicapgoapp/organization/members`

Löscht ein Organisationsmitglied

```typescript
interface MemberDelete {
  orgId: string
  email: string
}
```

Erfordert einen Request-Body (Query-Parameter) vom Typ `MemberDelete`

Bei Erfolg wird Folgendes zurückgegeben:

```typescript
return { status: 'OK' }
```

Bei Fehler wird Folgendes zurückgegeben:

```typescript
return { error: string, status: 'KO' }
```

## Statistiken

Dieser Endpunkt ermöglicht das Abrufen verschiedener Statistiken über Ihre Apps und Organisationen

### GET /statistics/app/:app_id

Ruft Statistiken für eine bestimmte App ab

Query-Parameter:
```typescript
interface StatsQuery {
  from: Date
  to: Date
  graph?: 'mau' | 'storage' | 'bandwidth'
}
```

Gibt zurück:
```typescript
interface Stats {
  date: string
  mau: number      // Monatlich aktive Nutzer
  storage: number  // Speichernutzung in Bytes
  bandwidth: number // Bandbreitennutzung in Bytes
}
```

Wenn der `graph`-Parameter angegeben ist, wird eine SVG-Bildvisualisierung der angeforderten Metrik zurückgegeben

### GET /statistics/user

Ruft aggregierte Statistiken über alle Organisationen ab, auf die der Benutzer Zugriff hat

Query-Parameter:
```typescript
interface StatsQuery {
  from: Date
  to: Date
  graph?: 'mau' | 'storage' | 'bandwidth'
}
```

Gibt zurück:
Wenn `graph` nicht angegeben ist:
```typescript
interface Stats {
  date: string
  mau: number      // Monatlich aktive Nutzer
  storage: number  // Speichernutzung in Bytes
  bandwidth: number // Bandbreitennutzung in Bytes
}[]
```

Wenn der `graph`-Parameter angegeben ist, wird eine SVG-Bildvisualisierung der angeforderten Metrik zurückgegeben

### GET /statistics/org/:org_id

Ruft Statistiken für eine bestimmte Organisation ab

Query-Parameter:
```typescript
interface StatsQuery {
  from: Date
  to: Date
  graph?: 'mau' | 'storage' | 'bandwidth'
}
```

Gibt zurück:
Wenn `graph` nicht angegeben ist:
```typescript
interface Stats {
  date: string
  mau: number      // Monatlich aktive Nutzer
  storage: number  // Speichernutzung in Bytes
  bandwidth: number // Bandbreitennutzung in Bytes
}[]
```

Wenn der `graph`-Parameter angegeben ist, wird eine SVG-Bildvisualisierung der angeforderten Metrik zurückgegeben

## Kanäle

Dieser Endpunkt ermöglicht es Ihnen, alle verfügbaren Kanäle Ihrer App zu überprüfen und zu modifizieren

### POST

`https://apicapgoapp/channel`

Senden Sie dies, um einen Kanal zu erstellen oder zu aktualisieren

```typescript
type disable_update = "major" | "minor" | "version_number" | "none"
interface ChannelSet {
  app_id: string
  channel: string
  version?: string
  public?: boolean
  disableAutoUpdateUnderNative?: boolean
  disableAutoUpdate?: disable_update
  ios?: boolean
  android?: boolean
  allow_device_self_set?: boolean
  allow_emulator?: boolean
  allow_dev?: boolean
}
```

erhalten Sie dies:

```json
{ "status": "ok" }
```

### GET

`https://apicapgoapp/channel`

Senden Sie `app_id` als URL-Parameter und erhalten Sie ein Array der ersten 50 Kanäle\
Sie können die nächsten erhalten, indem Sie `page=1` senden

```typescript
interface Channel {
    id: number;
    created_at: string;
    name: string;
    app_id: string;
    version: {
        id: number,
        name: string
    };
    created_by: string;
    updated_at: string;
    public: boolean; // Standard oder nicht
    disableAutoUpdateUnderNative: boolean;
    disableAutoUpdate: boolean;
    allow_emulator: boolean;
    allow_dev: boolean;
}[]
```

und optional `channel` Name als URL-Parameter, um einen zu erhalten:

```typescript
interface Channel {
    id: number;
    created_at: string;
    name: string;
    app_id: string;
    version: {
        id: number,
        name: string
    };
    created_by: string;
    updated_at: string;
    public: boolean; // Standard oder nicht
    disableAutoUpdateUnderNative: boolean;
    disableAutoUpdate: boolean;
    allow_emulator: boolean;
    allow_dev: boolean;
}
```

### DELETE

`https://apicapgoapp/channel`

Senden Sie dies als Query-Parameter

```typescript
interface Channel {
  channel: string
  app_id: string
}
```

erhalten Sie dies:

```json
{ "status": "ok" }
```

## Geräte

Dieser Endpunkt ermöglicht es Ihnen, alle mit Ihrer App verbundenen Geräte zu überprüfen und zu modifizieren

### POST

`https://apicapgoapp/device`

Senden Sie dies

```typescript
interface DeviceLink {
  app_id: string
  device_id: string
  version_id?: string // Versionsname (wird zu einem besseren Namen migriert)
  channel?: string // Kanalname
}
```

erhalten Sie dies:

```json
{ "status": "ok" }
```

### GET

`https://apicapgoapp/device`

Senden Sie `app_id` als URL-Parameter und erhalten Sie ein Array der ersten 50 Geräte\
Sie können den nächsten erhalten, indem Sie `page=1` senden

```typescript
interface Device{
    created_at?: string | undefined;
    updated_at?: string | undefined;
    device_id: string;
    custom_id: string;
    version: {
        id: number,
        name: string
    };
    app_id: string;
    platform?: "ios" | "android" | undefined;
    plugin_version: string;
    os_version?: string | undefined;
    version_build: string;
    is_prod: boolean;
    is_emulator: boolean;
}[]       
```

und optional `device_id` Name als URL-Parameter, um einen zu erhalten:

```typescript
interface Device {
    created_at?: string | undefined;
    updated_at?: string | undefined;
    device_id: string;
    version: {
        id: number,
        name: string
    };
    app_id: string;
    platform?: "ios" | "android" | undefined;
    plugin_version: string;
    os_version?: string | undefined;
}
```

### DELETE

Dieser Endpunkt dient zum Aufheben der Verknüpfung von Kanal und Versionsüberschreibung. Sie können ein Gerät nicht aus Capgo löschen

`https://apicapgoapp/device`

Senden Sie dies als Abfrageparameter

```typescript
interface Device {
  device_id: string
  app_id: string
}
```

erhalten Sie dies:

```json
{ "status": "ok" }
```

## Bundles

Dieser Endpunkt ermöglicht es Ihnen, alle mit Ihrer App verknüpften Bundles zu überprüfen und zu löschen

### GET

`https://apicapgoapp/bundle`

Senden Sie `app_id` als URL-Parameter und erhalten Sie ein Array der ersten 50 Bundles\
Sie können die nächsten erhalten, indem Sie `page=1` senden

```typescript
interface Bundle {
  app_id: string
  bucket_id: string | null
  checksum: string | null
  created_at: string | null
  deleted: boolean
  external_url: string | null
  id: number
  minUpdateVersion: string | null
  name: string
  native_packages: Json[] | null
  owner_org: string
  r2_path: string | null
  session_key: string | null
  storage_provider: string
  updated_at: string | null
  user_id: string | null
}[]
```

### DELETE

Löscht ein Bundle in der Cloud

`https://apicapgoapp/bundle`

Senden Sie dies als Abfrageparameter

```typescript
interface Bundle {
  app_id: string
  version: string
}
```

erhalten Sie dies:

```json
{ "status": "ok" }
```

Alternativ können Sie senden

```typescript
interface Bundle {
  app_id: string
}
```

Um alle Bundles zu löschen
