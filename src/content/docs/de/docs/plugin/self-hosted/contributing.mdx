## Warum beitragen?

Zunächst einmal vielen Dank, dass Sie erwägen, zu capgo Open-Source-Projekten beizutragen! Es sind Menschen wie Sie, die capgo Open-Source-Projekte zu so großartigen Werkzeugen machen.

Hier sind einige Gründe, warum Sie einen Beitrag in Betracht ziehen sollten:

* Die Mitarbeit an capgo Open-Source-Projekten ist eine großartige Möglichkeit, Geld durch die [zahlreichen Bounties](https://consolealgoraio/org/Capgo) zu verdienen, die vom capgo-Team angeboten werden
* Die Mitarbeit an capgo Open-Source-Projekten ist eine großartige Möglichkeit, eine Funktion hinzuzufügen, die Sie gerne sehen würden
* Die Mitarbeit an capgo Open-Source-Projekten ist eine großartige Möglichkeit, einen Bug zu beheben, auf den Sie gestoßen sind
* Die [Hauptlizenz von Capgo](https://githubcom/Cap-go/capgo/blob/main/LICENSE) verlangt, dass Sie alle Änderungen, die Sie daran vornehmen, als Open Source zur Verfügung stellen. Durch Ihren Codebeitrag können Sie Ihre Änderungen als Open Source beibehalten und anderen die Nutzung ermöglichen

## Wie man beiträgt

* Zunächst müssen Sie das Repository forken, zu dem Sie beitragen möchten
* Als Zweites müssen Sie Ihre Änderungen committen und in dieses Repository pushen
* Zuletzt müssen Sie einen Pull Request erstellen
* Das war's! Jetzt müssen Sie nur noch warten, bis das capgo-Team Ihren PR überprüft
