## Warum beitragen?

Zunächst einmal vielen Dank, dass Sie in Betracht ziehen, zu capgo Open-Source-Projekten beizutragen! Es sind Menschen wie Sie, die capgo Open-Source-Projekte zu so großartigen Werkzeugen machen.

Hier sind einige Gründe, warum Sie in Betracht ziehen könnten, beizutragen:

* Zu capgo Open-Source-Projekten beizutragen ist eine großartige Möglichkeit, etwas Geld aus den [unzähligen Prämien](https://consolealgoraio/org/Capgo) zu verdienen, die vom capgo-Team angeboten werden.
* Zu capgo Open-Source-Projekten beizutragen ist eine großartige Möglichkeit, ein Feature hinzuzufügen, das Sie gerne sehen würden.
* Zu capgo Open-Source-Projekten beizutragen ist eine großartige Möglichkeit, einen Bug zu beheben, auf den Sie gestoßen sind.
* [Die Lizenz von Capgo](https://githubcom/Cap-go/capgo/blob/main/LICENSE) verlangt, dass Sie alle Änderungen, die Sie daran vornehmen, Open Source machen. Indem Sie Ihren Code beisteuern, können Sie Ihre Änderungen Open Source halten und anderen die Nutzung ermöglichen.

## Wie man beiträgt

1. Zunächst müssen Sie das Repository, zu dem Sie beitragen möchten, forken.
2. Zweitens müssen Sie Ihre Änderungen an dieses Repository committen und pushen.
3. Zuletzt müssen Sie einen Pull-Request öffnen.
4. Das war's! Jetzt müssen Sie nur noch warten, bis das capgo-Team Ihr PR überprüft.
