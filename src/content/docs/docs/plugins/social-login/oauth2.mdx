## Introduction

The Capgo Social Login plugin supports generic OAuth2 authentication, allowing you to integrate with any OAuth2-compliant provider.

## OIDC and OAuth2 configuration

You can configure provider endpoints manually, or use `issuerUrl` for OpenID Connect discovery.

```typescript
import { SocialLogin } from '@capgo/capacitor-social-login';

await SocialLogin.initialize({
  oauth2: {
    auth0: {
      issuerUrl: 'https://your-tenant.auth0.com',
      appId: 'your-client-id',
      redirectUrl: 'myapp://oauth/auth0',
      scope: 'openid profile email',
      pkceEnabled: true,
      logsEnabled: true,
    },
  },
});
```

Alternative endpoint fields are supported:

- `authorizationEndpoint` as alias of `authorizationBaseUrl`
- `tokenEndpoint` as alias of `accessTokenEndpoint`
- `endSessionEndpoint` as alias of `logoutUrl`

Also available:

- `additionalParameters`, `additionalTokenParameters`, `additionalResourceHeaders`, `additionalLogoutParameters`
- `logoutUrl`, `postLogoutRedirectUrl`, `scopes` alias
- `issuerUrl`, `loginHint`, `prompt`

## Provider list

The plugin supports multiple providers at once:

- GitHub
- Azure AD / Microsoft Entra ID
- Auth0
- Okta
- Keycloak
- Custom OAuth2 servers

## Configuration Options

| Option | Type | Required | Description |
|--------|------|----------|-------------|
| `appId` | string | Yes | OAuth2 client identifier |
| `authorizationBaseUrl` | string | Yes | Authorization endpoint URL |
| `accessTokenEndpoint` | string | No* | Token endpoint URL (`*` if not using issuer discovery) |
| `issuerUrl` | string | No | OIDC discovery endpoint |
| `authorizationEndpoint` | string | No | Alias of `authorizationBaseUrl` |
| `tokenEndpoint` | string | No | Alias of `accessTokenEndpoint` |
| `redirectUrl` | string | Yes | Callback URL |
| `scope` / `scopes` | string / string[] | No | Requested scopes |
| `pkceEnabled` | boolean | No | Default true |
| `responseType` | `'code'` \, `'token'` | No | Default `'code'` |
| `flow` | `'popup'` \, `'redirect'` | No | Web-only flow for login popup vs full redirect |

## Using OAuth2 Login

```typescript
const result = await SocialLogin.login({
  provider: 'oauth2',
  options: {
    providerId: 'github',
    scope: 'read:user user:email',
    loginHint: 'user@example.com',
  },
});
```

### Refresh

```typescript
const result = await SocialLogin.refreshToken({
  provider: 'oauth2',
  providerId: 'github',
  refreshToken: 'existing-refresh-token',
});
```

## Security Best Practices

<Steps>
1. **Enable PKCE**
   Keep `pkceEnabled: true`.

2. **Prefer authorization code**
   Use `responseType: 'code'` with token endpoint exchange.

3. **Validate token values**
   Verify `idToken`, `accessToken`, and expiration on your backend.
</Steps>
