이 페이지는 Capgo Live Updates에서 사용할 수 있는 모든 기능에 대한 포괄적 개요를 제공합니다. 각 기능에는 간단한 설명과 자세한 문서로 연결되는 링크가 포함되어 있습니다.

## 핵심 업데이트 시스템

### OTA(무선 업데이트)
JavaScript, HTML, CSS 및 자산 업데이트를 앱 스토어 승인 없이 사용자에게 직접 배포합니다. 업데이트는 백그라운드에서 다운로드되며 다음 앱 재시작 시 적용됩니다.

**주요 기능:**
1. 백그라운드 다운로드
2. 자동 설치
3. 사용자 중단 없음
4. 크로스 플랫폼 지원(iOS, Android, Electron)

[Learn more about update behavior →](/docs/live-updates/update-behavior/)

---

### Delta 업데이트(차등 업데이트)
버전 간 변경된 파일만 다운로드하여 대역폭 사용량을 최대 95%까지 줄이고 업데이트 전송 속도를 높입니다.

**주요 기능:**
1. 파일 수준 차이 자동 계산
2. 체크섬 기반 검증
3. 매니페스트 비교
4. 필요 시 전체 업데이트로의 지능적 대체

[Learn more about delta updates →](/docs/live-updates/differentials/)

---

### 자동 롤백
업데이트가 로드되지 않거나 크래시를 일으키는 경우 시스템은 마지막으로 작동하던 버전으로 자동으로 되돌립니다.

**주요 기능:**
1. 크래시 탐지
2. 시간초과 탐지
3. 자동 되돌리기
4. 사용자 개입 필요 없음

[Learn more about rollbacks →](/docs/live-updates/rollbacks/)

---

### 체크섬 검증 및 폴백
체크섬을 통해 번들 무결성을 검증하고 손상이 감지되면 자동으로 마지막으로 작동하던 버전으로 폴백합니다.

**주요 기능:**
1. 다운로드 시 체크섬 검증
2. 손상 탐지
3. 마지막으로 작동하던 번들로 자동 폴백
4. 수동 복구 도구 사용 가능

---

### 비호환 업데이트 탐지
이전 네이티브 코드 버전을 실행 중인 디바이스에 호환되지 않는 업데이트가 적용되지 않도록 방지합니다.

**주요 기능:**
1. 네이티브 버전 호환성 검사
2. 플러그인 의존성 검증
3. 비호환 업데이트 자동 차단
4. 명확한 오류 메시지

[Learn more about version targeting →](/docs/live-updates/version-targeting/)

---

## 배포 제어

### 채널 시스템
다양한 환경과 사용자 세그먼트에 걸쳐 업데이트를 구성하고 관리하도록 유연한 채널 구성으로 정리합니다.

**주요 기능:**
1. 무제한 커스텀 채널(생산, 스테이징, 베타 등)
2. 채널별 번들 할당
3. 채널별 대상 지정 규칙
4. 디바이스 자체 할당
5. 디바이스별 채널 재정의

[Learn more about channels →](/docs/live-updates/channels/)

---

### 디바이스 대상 지정
점진적 롤아웃 및 제어된 배포를 위해 특정 디바이스, 버전 또는 사용자 세그먼트를 대상 지정합니다.

**주요 기능:**
1. 버전 기반 타깃팅
2. 디바이스별 재정의
3. 플랫폼 필터링(iOS, Android)
4. 사용자 정의 메타데이터 필터링
5. 에뮬레이터/개발 빌드 차단

---

### 채널 정책
각 채널에서 업데이트가 배포되는 방식에 대한 규칙과 제한을 구성합니다.

**주요 기능:**
1. 자동 업데이트 비활성화
2. 주요 버전 업데이트 차단
3. 에뮬레이터에서 업데이트 비활성화
4. 개발 빌드에서 업데이트 비활성화
5. 플랫폼별 정책(iOS 전용, Android 전용)

[Learn more about channel policies →](/docs/live-updates/channels/#channel-policies)

## 개발 도구

### 번들 미리보기
번들을 디바이스에 배포하기 전 라이브 웹 환경에서 미리 확인할 수 있으며 웹 대시보드에서 접근 가능합니다.

**위치:** Web Dashboard → App → Bundle → Preview 탭

---

### 실시간 디버깅
CLI를 통한 특정 디바이스의 업데이트 이벤트를 실시간으로 모니터링하며 체크, 다운로드, 설치 및 오류 이벤트를 표시합니다.

**사용법:**
```bash
npx @capgo/cli app debug [appId]
```

**다음 항목을 보여줍니다:**
- 업데이트 확인
- 다운로드 진행
- 설치 상태
- 오류 메시지
- 정책 차단

---

### 번들 매니페스트 뷰어
파일 목록, 체크섬 및 메타데이터를 포함한 모든 번들의 전체 매니페스트를 검사합니다.

**위치:** Web Dashboard → App → Bundle → Manifest 탭

**다음 항목을 보여줍니다:**
- 체크섬이 포함된 파일 목록
- 번들 메타데이터
- 네이티브 버전 호환성
- 플러그인 의존성

---

### 네이티브 플러그인 의존성
각 번들에 포함된 모든 네이티브 Capacitor 플러그인을 확인하여 버전 간 의존성 변화를 추적합니다.

**위치:** Web Dashboard → App → Bundle → Dependencies 탭

**다음 항목을 보여줍니다:**
- 플러그인 이름 및 버전
- 의존성 추가/제거
- 호환성 경고

---

### CLI 통합
자동 배포 및 CI/CD 통합을 위한 포괄적 명령줄 인터페이스.

**주요 명령어:**
- `bundle upload` - 새 번들 업로드
- `bundle list` - 모든 번들 목록
- `bundle delete` - 번들 삭제
- `bundle cleanup` - 오래된 번들 정리
- `channel set` - 채널 구성
- `app debug` - 실시간 디버깅

[View full CLI reference →](/docs/cli/reference/)

---

### 번들 암호화
AES-256 암호화를 사용하는 번들에 대한 종단 간 암호화로 전송 중 및 저장 중에 코드를 보호합니다.

**주요 기능:**
1. RSA 키 페어 생성
2. AES-256 번들 암호화
3. 코드 서명 검증
4. 암호 키 관리

[Learn more about encryption →](/docs/live-updates/encryption/)

---

### 번들 정리 및 보존
저장소 사용량을 관리하기 위해 보존 정책에 따라 오래된 번들을 자동으로 정리합니다.

**주요 기능:**
1. 설정 가능한 보존 수
2. CLI를 통한 자동 정리
3. 예약된 정리 작업
4. 저장소 사용량 추적

**사용 예:**
```bash
npx @capgo/cli bundle cleanup --keep=10
```

사용자 기반의 업데이트 채택률, 성공률 및 배포 진행 상황을 추적합니다.

**지표:**
1. 다운로드 성공률
2. 설치 성공률
3. 유형별 오류율
4. 시간 경과에 따른 업데이트 채택
5. 버전 분포

**위치:** Web Dashboard → App → Statistics

---

### 디바이스 로그
디바이스별 이벤트 로그로 체크부터 설치까지의 전체 업데이트 수명주기를 표시합니다.

**이벤트 유형:**
1. 업데이트 확인
2. 다운로드 시작/완료/실패
3. 설치 시작/완료/실패
4. 롤백 이벤트
5. 정책 차단

**위치:** 
- Web Dashboard → App → Device → Logs
- Web Dashboard → App → Logs (all devices)

[Learn more about logs →](/docs/webapp/logs/)

---

### 번들 사용 분석
활성화된 번들, 다운로드 수 및 저장소 사용량에 대한 자세한 분석.

**지표:**
1. 번들당 활성 설치
2. 다운로드 수
3. 번들별 저장소 사용량
4. 대역폭 사용량

---

### 채널 통계
채널별 성능 및 채택 지표를 추적합니다.

**지표:**
1. 채널당 디바이스
2. 채널별 업데이트 성공률
3. 배포 이력
4. 채널별 오류율

**위치:** Web Dashboard → App → Channel → Statistics

---

**추적 이벤트:**
- 번들 업로드
- 채널 할당
- 정책 변경
- 디바이스 재정의

**위치:** Web Dashboard → App → Channel → History

## 팀 협업

### 역할 기반 접근 제어(RBAC)
조직 및 애플리케이션 수준 접근 제어를 위한 세분화된 권한

**조직 역할:**  
1. `super_admin` - 전체 권한  
2. `admin` - 관리자 권한  
3. `read` - 읽기 전용 권한  
4. `upload` - 업로드 전용 권한

**앱 역할:**  
1. `app_developer` - 전체 앱 권한  
2. `app_uploader` - 번들 업로드 전용  
3. `app_reader` - 읽기 전용 권한

**위치:**  
1. 웹 대시보드 → 조직 → 구성원  
2. 웹 대시보드 → 앱 → 접근

[RBAC에 대해 자세히 보기 →](/docs/webapp/organization-system/#roles-and-permissions)

---

### 감사 로그
규정 준수 및 보안을 위한 모든 조직 및 앱 활동의 완전한 감사 기록.

**로그된 이벤트:**  
1. 사용자 작업(로그인, 로그아웃, 권한 변경)  
2. 번들 작업(업로드, 삭제, 할당)  
3. 채널 작업(생성, 업데이트, 삭제)  
4. 조직 변경(설정, 구성원)

**위치:** 웹 대시보드 → 조직 → 감사 로그

---

### 웹훅
HTTP 웹훅을 통해 애플리케이션의 이벤트에 대한 실시간 알림을 수신합니다.

**지원되는 이벤트:**  
1. `apps` - 앱 생성/수정/삭제  
2. `app_versions` - 번들 업로드/삭제  
3. `channels` - 채널 생성/수정/삭제  
4. `org_users` - 구성원 추가/제거  
5. `orgs` - 조직 업데이트

**특징:**  
1. 맞춤형 웹훅 URL  
2. 이벤트 필터링  
3. 전달 로그  
4. 재시도 메커니즘  
5. 테스트 기능

**위치:** 웹 대시보드 → 조직 → 웹훅

---

### 다중 사용자 협업
특정 역할과 권한으로 조직에 팀원을 초대합니다.

**특징:**  
1. 이메일 초대  
2. 역할 지정  
3. 구성원 관리  
4. 접근 권한 회수

**위치:** 웹 대시보드 → 조직 → 구성원

---

### API 키 관리
옵션 만료 날짜 및 해시 저장소를 갖춘 API 키를 생성, 관리 및 폐기합니다.

**키 기능:**  
1. 앱별 또는 조직별 키  
2. 선택적 만료 날짜  
3. 해시 저장소(되돌릴 수 없음)  
4. 키 순환 지원

**위치:** 웹 대시보드 → API 키

[API 키에 대해 자세히 알아보기 →](/docs/public-api/#authentication)

---

### 비밀번호 정책
보안 표준을 강제하기 위한 조직 차원의 비밀번호 요건.

**구성 가능한 정책:**  
1. 최소 길이  
2. 대문자 필요  
3. 숫자 필요  
4. 특수 문자 필요

**위치:** 웹 대시보드 → 조직 → 보안

## 고급 기능

### 맞춤 업데이트 동작
SDK를 통해 업데이트를 언제 어떻게 확인하고 적용할지 구성합니다.

**구성 가능한 옵션:**  
1. 확인 간격 (`periodCheckDelay` - 최소 600초)  
2. 직접 업데이트 시점 (`directUpdate` - atInstall, onLaunch, always)  
3. 자동 업데이트 활성화/비활성화 (`autoUpdate`)  
4. 네트워크 요구사항(Android 전용 - WorkManager를 통해)

[업데이트 동작에 대해 자세히 알아보기 →](/docs/live-updates/update-behavior/)

### 업데이트 유형
다양한 사용 사례를 위한 다양한 업데이트 유형, 즉 즉시 업데이트에서 사용자 제어 설치까지.

**가능한 유형:**  
1. 백그라운드 업데이트(기본값)  
2. 즉시 업데이트  
3. 사용자 확인 후 업데이트  
4. 조건부 업데이트

[업데이트 유형에 대해 자세히 알아보기 →](/docs/live-updates/update-types/)

---

### 크레딧 시스템
대역폭, 저장소 및 기타 자원에 대한 크레딧 기반 요금제.

**특징:**  
1. 크레딧 사용 추적  
2. 사용량 경고  
3. Stripe를 통한 충전  
4. 크레딧 원장

**위치:** 웹 대시보드 → 조직 → 크레딧
