## 이 튜토리얼에서 다루는 내용은?

이 튜토리얼은 처음부터 supabase를 시작하고 edge functions를 시작하는 방법을 보여줍니다

## 요구사항
 1. 복제된 [capgo](https://github.com/Cap-go/capgo)
 3. [supabase](https://supabase.com/)

## 시작하기

시작하려면 다음을 실행하세요
```bash
supabase start
```

다음과 같은 내용이 표시됩니다:
```js
Started supabase local development setup.

         API URL: http://localhost:54321
     GraphQL URL: http://localhost:54321/graphql/v1
          DB URL: postgresql://postgres:postgres@localhost:54322/postgres
      Studio URL: http://localhost:54323
    Inbucket URL: http://localhost:54324
      JWT secret: [truncated]
        anon key: supa_key_anon
service_role key: supa_key_admin
```

다음으로 `configs.json`을 열고 다음 값을 설정하세요:
```json
{
  "base_domain": {
    "prod": "console.capgo.app",
    "development": "development.console.capgo.app",
    "local": "localhost:3332"
  },
  "supa_anon": {
    "prod": "supa_key_anon",
    "development": "supa_key_anon",
    "local": "supa_key_anon"
  },
  "supa_url": {
    "prod": "http://localhost:54321",
    "development": "http://localhost:54321",
    "local": "http://localhost:54321"
  }
}
```

여기서 `supa_key_anon`은 이전 단계의 값입니다.

:::danger
⚠️ `configs.json`을 원격 저장소에 커밋하지 마세요
:::

다음으로 [localhost:54323](http://localhost:54323/projects)에 접속할 수 있고 `users` 테이블이 다음과 같이 보이는지 확인하세요

<figure><img src="/supabase.webp" alt="Supabase dashboard showing users table" /><figcaption></figcaption></figure>

그렇다면 다음을 실행하여 edge functions를 시작하세요:

```bash
supabase functions serve
```

그리고 다음을 실행하여 프론트엔드를 시작하세요:
```bash
bun run serve
```
