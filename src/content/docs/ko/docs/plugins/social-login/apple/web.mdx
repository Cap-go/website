웹 로그인 구성은 간단하지 않습니다. iOS에서 `Sign in with Apple`을 설정하는 것보다 더 어렵지만 Android에서 `Sign in with Apple`을 설정하는 것보다는 더 어렵습니다.

## 서비스 생성

:::note
이미 Android용 `Sign in with Apple`을 구성한 경우 이 단계는 불필요합니다.
:::

서비스를 생성하려면 [여기](/docs/plugins/social-login/apple/android/#creating-the-service-id/) 가이드를 따르세요.

## `Return URLs` 구성

<Steps>

1. **서비스 ID 구성으로 이동**

   [Apple Developer Portal](https://developer.apple.com)에서 `Identifiers` > `Services IDs`로 이동하여 서비스 ID를 클릭합니다.

2. **Sign in with Apple 구성**

   `Sign in with Apple` 옆의 `Configure`를 클릭합니다.

   ![apple_dev_configure_login](/social-login-assets/apple_dev_configure_login.webp)

3. **Return URL 추가**

   `+` 버튼을 클릭하여 새 return URL을 추가합니다.

   ![apple_dev_return_url_plus](/social-login-assets/apple_dev_return_url_plus.webp)

4. **Return URL 추가**

   `Domains and Subdomains`에 웹 애플리케이션의 도메인을 추가합니다.
   :::caution
   여기에 `localhost` 또는 `127.0.0.1`을 도메인으로 추가할 수 **없습니다**.
   :::

   그런 다음 `https://` 접두사와 Apple 로그인을 호출할 경로를 사용하여 도메인을 추가합니다.
   예를 들어, 도메인이 `https://my-app.com`이고 `/login`에서 Apple 로그인을 호출하는 경우 return URL로 `https://my-app.com/login`을 추가해야 합니다.

   :::caution
   후행 슬래시가 있는 도메인과 없는 도메인을 **모두** 추가해야 합니다.
   :::

   ![apple_dev_return_url_add](/social-login-assets/apple_dev_return_url_add.webp)

5. **변경 사항 저장**

   1. `Next` 버튼을 클릭하여 변경 사항을 저장합니다.
   2. `Save` 버튼을 클릭하여 변경 사항을 저장합니다.

6. JavaScript용 로그인을 테스트할 준비가 되었습니다. localhost에서는 테스트할 수 없습니다.
</Steps>
