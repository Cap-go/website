## 소개

이 가이드에서는 Android에 대한 Capgo 소셜 로그인을 사용하여 Google 로그인을 설정하는 방법을 알아봅니다.
[일반 설정 가이드](/docs/plugins/social-login/google/general/)를 이미 읽으셨다고 가정합니다.

## Android에서 Google 로그인 사용

이 부분에서는 Android에서 Google 로그인을 설정하는 방법을 알아봅니다.

:::caution
Android SHA1 인증서는 매우 고통스럽기 때문에 누구에게도 이를 설정하도록 하고 싶지 않습니다. 다음 단계에서는 Google Play 스토어에 게시되지 않고 로컬 시뮬레이터 또는 개발 하드웨어 장치에서만 사용되는 앱의 가장 간단한 시나리오를 가정합니다. 

앱을 Google Play 스토어에 배포한 경우 **반드시** 프로덕션 릴리스용 Google Play 콘솔의 SHA1을 포함하는 추가 Android 클라이언트 ID를 추가해야 합니다. Google Play이 릴리스 번들에 서명하는 데 사용하는 SHA1 해시는 `Test and release > Setup > App Signing`에서 찾을 수 있습니다.

마지막으로, 실수를 해도 오류가 명확하지 않다는 점을 언급하는 것이 중요합니다. 디버깅이 매우 어려울 수 있습니다. 설정에 어려움을 겪는다면 [Github 문제](https://github.com/Cap-go/capacitor-social-login/issues)를 살펴보세요.

또한 자세한 내용은 [Google Android에 대한 로그인 설정](#troubleshooting)의 문제 해결 섹션을 참조하세요.
:::

:::note
여러 개의 Android 클라이언트 ID를 생성할 수 있습니다. 이는 SHA1 인증서가 여러 개 있는 경우 필요합니다.
:::

<Steps>
1. Android 클라이언트 ID를 생성합니다.
   1. 검색창을 클릭하세요
      <img src="/social-login-assets/google_cons_search.png" alt="Google 콘솔 검색 창" />
   
   2. `credentials`을 검색하고 'API 및 서비스' 항목(스크린샷의 2번)을 클릭합니다.
      <img src="/social-login-assets/google_cons_cred_search.png" alt="API 및 서비스가 강조 표시된 자격 증명 옵션을 보여주는 검색 결과" />
   
   3. '사용자 인증 정보 생성'을 클릭하세요.
      <img src="/social-login-assets/google_cons_create_cred.png" alt="Google 콘솔에서 자격 증명 만들기 버튼" />
   
   4. `OAuth client ID`를 선택하세요.
      <img src="/social-login-assets/google_cons_cred_oauth.png" alt="자격 증명 생성 메뉴의 OAuth 클라이언트 ID 옵션" />

   5. `Android` 애플리케이션 유형을 선택합니다.
      <img src="/social-login-assets/google_cons_app_type_android.png" alt="Android 옵션이 강조 표시된 애플리케이션 유형 선택" />
   
   6. Android 스튜디오를 엽니다.
   
   7. 네비게이터 맨 아래에서 'Gradle Scripts'를 찾으세요.
      <img src="/social-login-assets/studio_gradle_scripts.png" alt="Android Studio 프로젝트 탐색기의 Gradle 스크립트 섹션" />
   
   8. `app` 모듈에 대한 `build.gradle`을 찾습니다.
      <img src="/social-login-assets/studio_build_gradle.png" alt="Gradle 스크립트 섹션의 build.gradle(모듈: 앱) 파일" />
   
   9. `android.defaultConfig.applicationId`을 복사합니다. 이것이 Google 콘솔의 `패키지 이름`이 됩니다.
      <img src="/social-login-assets/studio_build_gradle_app_id.png" alt="applicationId 구성을 보여주는 Build.gradle 파일" />
   
   10. 이제 터미널을 엽니다. 앱의 `android` 폴더에 있는지 확인하고 `./gradlew signInReport`을 실행하세요.
      <img src="/social-login-assets/term_sign_report.png" alt="gradlew signInReport 명령을 보여주는 터미널" />
   
   11. 이 명령의 맨 위로 스크롤하십시오. 다음을 확인해야 합니다. `SHA1`을 복사합니다.
      <img src="/social-login-assets/term_sign_report_res.png" alt="SHA1 인증서 지문을 보여주는 터미널 출력" />
   
   12. 이제 Google 콘솔로 돌아갑니다. '패키지 이름'으로 `applicationId`을 입력하고 인증서 필드에 SHA1을 입력한 후 `create`을 클릭하세요.
      <img src="/social-login-assets/google_cons_creat_android_client.png" alt="Android 패키지 이름과 SHA1 필드가 채워진 클라이언트 생성 양식" />
   
2. 웹 클라이언트 생성(Android에 필요함)
   
   1. Google 콘솔의 '인증서 생성' 페이지로 이동합니다.
   
   2. 애플리케이션 유형을 `Web`로 설정합니다.
      <img src="/social-login-assets/google_cons_app_type_web.png" alt="웹 옵션이 강조 표시된 애플리케이션 유형 선택" />
   
   3. `Create`을 클릭하세요.
      <img src="/social-login-assets/google_cons_web_app_create.png" alt="하단에 만들기 버튼이 있는 웹 클라이언트 생성 양식" />
   
   4. 클라이언트 ID를 복사합니다. 이를 JS/TS 코드에서 `webClientId`로 사용합니다.
      <img src="/social-login-assets/google_cons_copy_web_client_id.png" alt="복사할 웹 클라이언트 ID를 보여주는 클라이언트 ID 세부정보" />

3. `MainActivity` 수정1. Android Studio에서 앱을 열어주세요. 'cap open android'를 실행할 수 있습니다.
   
   2. `MainActivity.java` 찾기
     
      1. `app` 폴더를 엽니다.
         <img src="/social-login-assets/studio_app_folder.png" alt="Android Studio 프로젝트 탐색기의 앱 폴더" />
     
      2. `java` 찾기
         <img src="/social-login-assets/studio_app_java.png" alt="Android Studio 프로젝트 구조의 Java 폴더" />
     
      3. `MainActivity.java`을 찾아 클릭하세요.
         <img src="/social-login-assets/studio_app_java_activity_main.png" alt="패키지 구조의 MainActivity.java 파일" />
   
   3. `MainActivity.java`을 수정합니다. 다음 코드를 추가해주세요
     
      ```java
      import ee.forgr.capacitor.social.login.GoogleProvider;
      import ee.forgr.capacitor.social.login.SocialLoginPlugin;
      import ee.forgr.capacitor.social.login.ModifiedMainActivityForSocialLoginPlugin;
      import com.getcapacitor.PluginHandle;
      import com.getcapacitor.Plugin;
      import android.content.Intent;
      import android.util.Log;
      import com.getcapacitor.BridgeActivity;
      
      // ModifiedMainActivityForSocialLoginPlugin is VERY VERY important !!!!!!    
      public class MainActivity extends BridgeActivity implements ModifiedMainActivityForSocialLoginPlugin {
      
            @Override
            public void onActivityResult(int requestCode, int resultCode, Intent data) {
              super.onActivityResult(requestCode, resultCode, data);
      
              if (requestCode >= GoogleProvider.REQUEST_AUTHORIZE_GOOGLE_MIN && requestCode < GoogleProvider.REQUEST_AUTHORIZE_GOOGLE_MAX) {
                PluginHandle pluginHandle = getBridge().getPlugin("SocialLogin");
                if (pluginHandle == null) {
                  Log.i("Google Activity Result", "SocialLogin login handle is null");
                  return;
                }
                Plugin plugin = pluginHandle.getInstance();
                if (!(plugin instanceof SocialLoginPlugin)) {
                  Log.i("Google Activity Result", "SocialLogin plugin instance is not SocialLoginPlugin");
                  return;
                }
                ((SocialLoginPlugin) plugin).handleGoogleLoginIntent(requestCode, data);
              }
            }
      
            // This function will never be called, leave it empty
            @Override
            public void IHaveModifiedTheMainActivityForTheUseWithSocialLoginPlugin() {}
      }
      ```

   4. 파일 저장

4. 애플리케이션에서 Google 로그인을 사용하세요.
   
   1. 먼저 `SocialLogin`을 가져옵니다.
     
      ```typescript
      import { SocialLogin } from '@capgo/capacitor-social-login';
      ```
   
   2. 초기화를 호출합니다. 이 호출은 한 번만 호출되어야 합니다.
     
      ```typescript
      // onMounted is Vue specific
      // webClientId is the client ID you got in the web client creation step not the android client ID.
      onMounted(() => {
        SocialLogin.initialize({
          google: {
            webClientId: '673324426943-avl4v9ubdas7a0u7igf7in03pdj1dkmg.apps.googleusercontent.com',
          }
        })
      })
      ```
     
   3. `SocialLogin.login`에 전화하세요. 버튼을 만들고 클릭 시 다음 코드를 실행합니다.
     
      ```typescript
      const res = await SocialLogin.login({
        provider: 'google',
        options: {}
      })
      // handle the response
      console.log(JSON.stringify(res))
      ```

## 문제 해결

문제가 있는 경우 [Github 문제](https://github.com/Cap-go/capacitor-social-login/issues)를 살펴보세요.

Google 로그인 문제는 **항상** SHA1 인증서와 관련되어 있습니다.

개발 SHA1 인증서를 얻을 수 없는 경우 사용자 지정 키 저장소를 사용해 보세요. [여기](https://github.com/Cap-go/capacitor-social-login/issues/147#issuecomment-2849742574)는 프로젝트에 키스토어를 추가하는 방법을 설명하는 주석입니다.
