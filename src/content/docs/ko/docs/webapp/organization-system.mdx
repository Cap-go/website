## 조직 시스템이란 무엇인가요?

조직 시스템은 Capgo에서 팀 멤버들과 앱을 안전하게 공유할 수 있게 해주는 시스템입니다.

## 조직 설정 개요

<figure><img src="/orgs-settings.webp" alt="조직 설정" /><figcaption></figcaption></figure>

| 번호 | 요소 | 설명 |
|------|------|------|
| 1 | 설정 | 사이드바 메뉴에서 설정에 접근 |
| 2 | 조직 탭 | 조직 설정으로 전환 (개인 계정 설정과 비교) |
| 3 | 일반 | 조직 이름, 로고, 관리 이메일 보기 및 편집 |
| 4 | 멤버 | 팀 멤버와 권한 관리 |
| 5 | 보안 | 보안 정책 구성 (2FA, 비밀번호 요구 사항) |
| 6 | 감사 로그 | 조직의 활동 로그 보기 |
| 7 | 플랜 | 구독 플랜 보기 및 관리 |
| 8 | 사용량 | 조직의 사용 통계 모니터링 |
| 9 | 크레딧 | 추가 크레딧 보기 및 구매 |
| 10 | Webhooks | 웹훅 통합 구성 |
| 11 | 결제 | 결제 정보 및 인보이스 관리 |
| 12 | 조직 선택기 | 다른 조직 간 전환 |

### 조직 설정에 접근하는 방법

1. 왼쪽 사이드바에서 **설정** (1) 클릭
2. 상단에서 **조직** 탭 (2) 선택
3. 탭 (3-11)을 사용하여 다른 설정 섹션 간 이동

### 조직을 전환하는 방법

사이드바 왼쪽 상단 근처의 **조직 선택기** (12) 드롭다운을 클릭하여 접근 권한이 있는 조직을 보고 전환합니다.

## 멤버 관리

<figure><img src="/org-show-members.webp" alt="조직 멤버" /><figcaption></figcaption></figure>

멤버 섹션에는 조직에 접근 권한이 있는 모든 사용자가 표시됩니다:
- **멤버**: 사용자 아바타, 이름, 이메일
- **역할**: 현재 권한 레벨 (읽기, 업로드, 쓰기, 관리자, 슈퍼 관리자)
- **작업**: 멤버 편집 또는 제거

### 새 멤버를 초대하는 방법

<figure><img src="/orgs-add-member.webp" alt="멤버 초대" /><figcaption></figcaption></figure>

1. **+ 추가** 버튼 (1) 클릭
2. 모달에서 사용자의 이메일 주소 입력 (2)
3. **초대** 클릭

### 권한 선택

<figure><img src="/select-perm-orgs.webp" alt="권한 선택" /><figcaption></figcaption></figure>

이메일을 입력한 후 초대된 사용자의 권한 레벨을 선택합니다:
- **읽기** - 읽기 전용 접근
- **업로드** - 새 버전 업로드 가능
- **쓰기** - 설정 및 콘텐츠 수정 가능
- **관리자** - 결제 및 삭제를 제외한 전체 접근
- **슈퍼 관리자** - 조직 전체 제어

### 초대 완료

<figure><img src="/invite_org_user.webp" alt="초대 양식" /><figcaption></figcaption></figure>

나머지 세부 정보를 입력합니다:
- **이메일** - 이전 단계에서 미리 채워짐
- **역할** - 선택한 권한 레벨 표시
- **이름** - 선택 사항: 초대받는 사람의 이름
- **성** - 선택 사항: 초대받는 사람의 성
- **Captcha** - 사람임을 확인

**초대 보내기**를 클릭하여 프로세스를 완료합니다.

## 결제

**슈퍼 관리자** 권한이 있는 사용자는 주어진 조직의 결제를 관리할 수 있습니다. 플랜은 개인 계정이 아닌 조직에 연결됩니다.

:::danger
⚠️ 플랜 구매는 현재 선택한 조직에만 영향을 미칩니다
:::

## FAQ

### 하나 이상의 조직을 만들 수 있나요?

아니요, 아직은 불가능합니다.

### 조직의 보안 정책을 어떻게 구성할 수 있나요?

슈퍼 관리자는 다음을 포함한 포괄적인 보안 정책을 구성할 수 있습니다:
- **2FA 적용** - 모든 멤버가 이중 인증을 활성화하도록 요구
- **비밀번호 정책** - 비밀번호 복잡성 요구 사항 설정 (길이, 대문자, 숫자, 특수 문자)
- **API 키 보안** - 보안/해시된 API 키 및 만료 정책 적용

자세한 구성 지침은 [조직 보안](/docs/webapp/organization-security/) 문서를 참조하세요.
