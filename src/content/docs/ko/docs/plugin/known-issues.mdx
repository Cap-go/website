## Ionic 실시간 리로드

* 개발 중일 때 CLI의 Ionic 실시간 리로드 기능을 사용하면 플러그인을 덮어쓰므로 업데이트를 볼 수 없습니다

## Quasar 실시간 리로드

## 업데이트 실패

* 일반적으로 큰 업데이트(> 20MB)를 푸시할 때 발생하며, 많은 사용자가 최신 버전을 받지 못합니다

## Android

### 다운로드 불가

인도의 기기에서 문제가 발생한 것을 확인했고, 사용자와 통화하여 다른 DNS 서버를 시도하게 했더니 작동했습니다

따라서 문제가 있다면 Cloudflare나 Google DNS와 같은 다른 DNS 서버를 사용해보세요

Cloudflare: 1111 및 1001

Google DNS: 8888 및 8844 또는 dnsgoogle

### Clearfix 문제

* usesCleartextTraffic 문제가 있다면, 플러그인이 sonar cloud에서 권장하는 모범 사례를 따르기 때문입니다. 90%의 경우에는 잘 작동하지만 일부 플러그인에서는 문제가 발생합니다

해결하려면 `android/app/src/main/AndroidManifestxml`의 `<application>` 키에 다음을 추가하세요:

```xml
tools:replace="android:usesCleartextTraffic"
xmlns:tools="http://schemasandroidcom/tools"
```

## iOS

### 개인정보 보호 매니페스트

[Privacy Manifest](https://capacitorjscom/docs/ios/privacy-manifest)에 `NSPrivacyAccessedAPICategoryUserDefaults` 딕셔너리 키를 추가하세요 (일반적으로 `ios/App/PrivacyInfoxcprivacy`):

```xml
<?xml version="10" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 10//EN" "http://wwwapplecom/DTDs/PropertyList-10dtd">
<plist version="10">
  <dict>
    <key>NSPrivacyAccessedAPITypes</key>
    <array>
      <!-- 파일이 이미 존재하는 경우 배열에 이 dict 항목을 추가하세요 -->
      <dict>
        <key>NSPrivacyAccessedAPIType</key>
        <string>NSPrivacyAccessedAPICategoryUserDefaults</string>
        <key>NSPrivacyAccessedAPITypeReasons</key>
        <array>
          <string>CA921</string>
        </array>
      </dict>
    </array>
  </dict>
</plist>
```

[`UserDefaults`](https://developerapplecom/documentation/foundation/userdefaults) API에 접근하는 이유로 [`CA921`](https://developerapplecom/documentation/bundleresources/privacy_manifest_files/describing_use_of_required_reason_api#4278401)을 선언하는 것을 권장합니다

### 네트워크 권한

업데이트 테스트를 위해 로컬 서버를 사용할 때 앱이 네트워크 권한을 요청하는 것은 정상적인 동작입니다. 원격 서버를 사용할 때는 이런 현상이 발생하지 않습니다

## 양쪽 OS

수동 모드 업데이트를 수행할 때 일부 이벤트는 감지하기 어렵습니다. 예를 들어 업데이트 실패는 JS 코드가 리로드되기 직전에 트리거되므로 감지할 수 없습니다

대안으로 번들을 나열하고 오류 통계를 확인하여 업데이트 실패 여부를 확인할 수 있습니다

앞으로 더 나은 방법을 찾아야 하지만, 자동 모드가 권장되는 업데이트 방식이므로 우선순위는 아닙니다

개선을 위한 PR을 환영합니다
