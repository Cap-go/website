## 이 튜토리얼에서 다룰 내용은?

이 튜토리얼은 capgo와 함께 사용하기 위해 [minio](https://minio/)를 설정하는 방법을 보여줍니다.

이것은 capgo를 실행하는데 기술적으로 필수는 아닙니다.

S3 설정을 통해 [CLI](/docs/cli/commands/)에서 번들을 업로드할 수 있습니다.

:::danger
⚠️ 이 튜토리얼은 프로덕션 환경에서 사용하기 위한 것이 아닙니다\
로컬 개발에서만 사용하세요
:::

## 요구사항

1. [Docker](https://wwwdockercom/)

## 시작하기

먼저, 새로운 디렉토리를 만드세요

그리고 그 안에 `data`라는 폴더를 만드세요

그런 다음 다음 명령어를 실행하세요:
```bash
docker run \
   -p 9000:9000 \
   -p 9090:9090 \
   --user $(id -u):$(id -g) \
   --name minio1 \
   -e "MINIO_ROOT_USER=ROOTUSER" \
   -e "MINIO_ROOT_PASSWORD=CHANGEME123" \
   -v PATH_TO_DATA_FOLDER_CREATED_IN_PREVIOUS_STEP:/data \
   quayio/minio/minio server /data --console-address ":9090"
```

이 컨테이너가 있는 콘솔 창을 닫은 경우 다음으로 시작할 수 있습니다:
```bash
docker start minio1
```

minio 설정을 변경해야 하는 경우 다음 명령어로 컨테이너를 제거할 수 있습니다:
```bash
docker rm minio1
```
⚠️ 이 명령어는 minio 데이터를 제거하지 않습니다

## 엣지 함수 설정하기

이제 S3 서버가 실행 중이므로 capgo 엣지 함수가 우리의 S3 서버를 사용하도록 설정해야 합니다

이를 위해 `capgo/supabase`에 `envlocal`이라는 ENV 파일을 만들어야 합니다

이 파일은 다음과 같아야 합니다:
```bash
STRIPE_WEBHOOK_SECRET=test
STRIPE_SECRET_KEY=test
API_SECRET=testsecret
PLAN_MAKER=test
PLAN_SOLO=test
PLAN_TEAM=test
# 아래는 S3 설정에 실제로 중요한 부분입니다
S3_ENDPOINT=hostdockerinternal
S3_REGION=dev-region
S3_PORT=9000
S3_SSL=false
R2_ACCESS_KEY_ID=ROOTUSER
R2_SECRET_ACCESS_KEY=CHANGEME123
```

`hostdockerinternal` IP는 도커 내부에서만 접근할 수 있는 IP이지만, minio가 localhost에서 접근할 수 있도록 코드에서 `0000`으로 대체됩니다

새 env 파일로 엣지 함수를 실행하려면 다음을 사용하세요
```bash
supabase functions serve --env-file /supabase/envlocal
```

## CLI에서 S3 사용 설정하기

CLI는 기본적으로 minio와 작동하지 않습니다. `capacitor.config.ts`[^1]에 다음과 같은 변경이 필요합니다
```ts
const config: CapacitorConfig = {
  appId: 'com.demo.app',
  appName: 'demoApp',
  webDir: 'dist',
  bundledWebRuntime: false,
  plugins: {
    CapacitorUpdater : {
      // 이 localS3가 없으면 업로드 명령이 실패합니다
      localS3: true
    },
  },
};
```

[^1]: 파일은 앱 디렉토리에 있습니다
