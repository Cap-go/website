Capgoのライブアップデート機能を使用して、アプリのUIとビジネスロジックをリモートからリアルタイムで更新し、アプリストアを経由せずにJSバンドルをユーザーに直接プッシュしてバグを即座に修正し、新機能を提供できます

このガイドは[Capgoクイックスタート](/docs/getting-started/quickstart)を完了し、以下をすでに実施していることを前提としています：

<Steps>

1 Capacitorアプリに`@capgo/capacitor-updater` SDKをインストール

2 `capacitorconfig.ts`でアプリIDとアップデートチャンネルを設定

3 コードに`CapacitorUpdater.notifyAppReady()`メソッドを追加

</Steps>

これらのステップをまだ完了していない場合は、先にクイックスタートを完了してください

<LinkCard href="/docs/getting-started/add-an-app/" title="アプリを追加" description="Capgoアカウントにアプリを追加し、アプリにプラグインをインストールする" />

## バンドルのアップロード

Capgo SDKをインストールして設定したら、最初のライブアップデートバンドルをアップロードする準備が整いました：

1 Webアセットをビルドします：
   ```shell
   npm run build
   ```

2 バンドルをCapgoにアップロードします：
    <Tabs>
      <TabItem label="Console">
        ```shell
      npx @capgo/cli@latest upload --channel=Production
      ```
      </TabItem>
      <TabItem label="Github Actions">
      ```yml
      // github/workflows/build_and_deploy.yml
    name: Build source code and send to Capgo

    concurrency:
      group: ${{ github.workflow }}-${{ github.ref }}
      cancel-in-progress: true

    on:
      push:
        branches:
          - main

    jobs:
      deploy_to_capgo:
        runs-on: ubuntu-latest
        steps:
          - name: Checkout
            uses: actions/checkout@v4
          - uses: actions/setup-node@v4
            with:
              node-version: 18
          - name: Install dependencies
            run: npm install
          - name: Build
            run: npm run build
          - name: Deploy to Capgo
            run: bunx @capgo/cli@latest bundle upload -a ${{ secrets.CAPGO_TOKEN }} --channel ${{ env.CHANNEL }}
            env:
              CAPGO_TOKEN: ${{ secrets.CAPGO_TOKEN }}
      ```
      </TabItem>
      <TabItem label="Gitlab">
      ```yml
      // gitlab-ci.yml
      stages:
        - build

      build:
        stage: build
        image: node:18
        cache:
          - key:
              files:
                - package-lock.json
            paths:
              - node_modules/
        script:
          - npm install
          - npm run build
          - npx @capgo/cli@latest bundle upload -a $CAPGO_TOKEN --channel $CAPGO_CHANNEL
        artifacts:
          paths:
            - node_modules/
            - dist/
        only:
          - master
      ```
      </TabItem>
    </Tabs>
   これにより、コマンドで指定されたチャンネルに新しいバンドルバージョンがアップロードされます

### アップロードのトラブルシューティング

アップロードが失敗した場合は、以下を確認してください：
- `capacitor.config.ts`のアプリIDがCapgoダッシュボードのアプリと一致していること
- アップロードコマンドをCapacitorプロジェクトのルートから実行していること
- Webアセットがビルドされ最新であること

まだ問題が解決しない場合は、[トラブルシューティング](/docs/getting-started/troubleshooting/)セクションをご確認ください

## デバイスでのアップデートの受信

バンドルがアップロードされたら、デバイスでライブアップデートをテストできます：

1 アプリをデバイスと同期します：
   ```shell
   npx cap sync ios
   ```

2 別のターミナルを開き、以下のコマンドを実行してアップデートステータスを確認します：
   ```shell
   npx @capgo/cli@latest app debug
   ```

3 アプリをローカルで実行します：
   ```shell
   npx cap run ios
   ```
   またはiOS/AndroidプロジェクトをXcode/Android Studioで開いてネイティブ実行します

4 アップデートがバックグラウンドでダウンロードされるまで、アプリを約30秒間開いたままにします

5 ログが更新されてアップデートステータスが表示されるまで数秒かかります

6 アプリを閉じて再度開くと、ライブアップデートが適用されているはずです！

ライブアップデートのテストについての詳細は、[Capgoクイックスタート](/docs/getting-started/quickstart#receiving-a-live-update-on-a-device)をご参照ください
