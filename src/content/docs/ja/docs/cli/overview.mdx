Capgoのライブアップデート機能を使用して、アプリのJavaScriptバンドルをリモートでリアルタイムに更新し、アプリストアのレビュープロセスを経ずに、JSの更新を直接ユーザーにプッシュして、バグを即座に修正し、新機能を提供できます。

<Aside>

ライブアップデートはJavaScriptバンドルの変更に限定されます。プラグインの追加や削除、ネイティブプロジェクトの設定変更などのネイティブコードを更新する必要がある場合は、新しいネイティブバイナリビルドをアプリストアに提出する必要があります。

</Aside>

## ライブアップデートの仕組み

Capgoのライブアップデートシステムには2つの主要なコンポーネントがあります：

1. アプリにインストールするCapgo SDK。SDKは利用可能な更新をチェックし、バックグラウンドでダウンロードします。

2. 特定のユーザーグループに更新を配信できるチャンネル。チャンネルを使用して、`Production`、`Staging`、`Dev`などの異なるリリーストラックを管理できます。

新しいJSバンドルをCapgoにアップロードしチャンネルに割り当てると、そのチャンネル用に設定されたアプリのCapgo SDKが更新を検出してダウンロードします。アプリが次回再起動したときに、新しいバンドルが読み込まれます。

## はじめに

ライブアップデートを使い始めるには、以下の手順に従ってください：

<Steps>

1. [Capgoクイックスタート](/docs/getting-started/quickstart)を完了して、アプリをCapgoで設定し、Capgo SDKをインストールします。

2. アプリコードで、アプリの初期化が完了した後に`CapacitorUpdaternotifyAppReady()`を呼び出します。これにより、Capgo SDKにアプリが更新を受信する準備ができたことを通知します。

3. JSバンドルをビルドしCapgoにアップロードします：
   ```shell
   npm run build
   npx @capgo/cli@latest upload --channel=Production 
   ```

4. アプリを開き、更新のダウンロードを待ちます。ステータスは以下のコマンドで確認できます：
   ```shell
   npx @capgo/cli@latest app debug
   ```

5. 更新がダウンロードされたら、アプリを閉じて再度開き、新しいバンドルを読み込みます。

</Steps>

詳細については、[ライブアップデートのデプロイ](/docs/getting-started/deploy)ガイドを参照してください。

## Capgo CLI

Capgo CLIは、開発者が独自のCI/CDパイプラインからCapgoのサービスを操作できる強力なツールです。CLIを使用することで、ビルドの生成とデプロイのタイミングを細かく制御でき、既存のエンタープライズワークフローにCapgoを統合できます。

### Capgo CLIの用途

Capgo CLIは、ライブアップデートワークフローでより多くの制御と柔軟性を必要とする開発者やチーム向けに設計されています。CI/CDパイプラインでCLIを使用することで：

- Capgoの組み込み自動化に依存せず、更新のビルドとデプロイのタイミングを正確に決定できます
- コード署名、QAテスト、管理者承認などの独自のプロセスをビルドとデプロイのステップの間に挿入できます
- 既存のDevOpsツールやワークフローにCapgoを統合できます

### 認証

Capgo CLIを使用するには、APIキーで認証する必要があります。APIキーはCapgoのアカウント設定で生成できます。

ログインしてAPIキーを安全に保存するには、以下を実行します：

```shell
npx @capgo/cli@latest login [API_KEY]
```

このコマンドは将来の使用のために保存されます。ログイン後は、各コマンドでAPIキーを提供する必要はありません。

### 他のCLIツールとの主な違い

他のライブアップデートCLIツールをご存知の方は、Capgo CLIについて以下の重要な点に注意してください：

- Capgoはライブアップデート機能セットのみに焦点を当てているため、開発とCI/CD両方のユースケースに単一のCLIを使用します

- Capgo CLIは別途インストールする必要はありません。`@capgo/cli`パッケージにバンドルされており、`npx`を使用して直接実行できます

- Capgo CLIはライブアップデートワークフロー専用に設計されているため、より汎用的なCLIツールにある一部の機能やコマンドが含まれていない場合があります
