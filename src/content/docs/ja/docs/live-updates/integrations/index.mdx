人気のあるCI/CDプラットフォームと統合することで、Capgo Live Updatesのデプロイメントプロセスを自動化します。これらの統合により、コード変更をプッシュするたびに自動的にアプリのアップデートをデプロイし、フィーチャーブランチをテストし、複数のデプロイメント環境を管理できます。

## 得られるもの

すべての統合ガイドには以下が含まれます：

- **シンプルなセットアップ**: 迅速に開始するための基本構成
- **高度なワークフロー**: ステージングと本番環境を含むマルチ環境デプロイメント
- **フィーチャーブランチテスト**: テストチャネルへのフィーチャーブランチの自動デプロイメント
- **セキュリティのベストプラクティス**: 安全なシークレット管理と環境保護
- **監視**: デプロイメントステータスの通知とロギング

<Aside type="tip">

**CI/CDが初めてですか？** プラットフォームのシンプルな構成から始めて、ニーズが増えるにつれて、マルチチャネルデプロイメントや自動テストなどのより高度な機能を徐々に追加していきます。

</Aside>

## 共通機能

各統合は以下をサポートします：

- **自動ビルド**: コード変更時にデプロイメントをトリガー
- **マルチチャネルサポート**: 異なるチャネル（development、staging、production）へのデプロイ
- **プルリクエスト/マージリクエストテスト**: 隔離された環境で変更をテスト
- **暗号化サポート**: Capgoの暗号化機能を使用した安全なデプロイメント
- **環境保護**: 本番環境用の手動承認と制限付きアクセス
- **通知**: Slack、メール、その他の通知統合

## 前提条件

統合を設定する前に、以下を確認してください：

- Capgoアカウントとアプリが構成されている
- アプリのソースコードがGitリポジトリにある
- [console.capgo.app/apikeys](https://console.capgo.app/apikeys)からのCapgo APIトークン
- プロジェクトにNode.jsとnpm/yarnが構成されている
