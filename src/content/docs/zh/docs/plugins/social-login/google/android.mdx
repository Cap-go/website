## 简介

在本指南中，您将了解如何为 Android 设置 Google 使用 Capgo 社交登录登录。
我假设您已经阅读了[一般设置指南](/docs/plugins/social-login/google/general/)。

## 使用 Google 登录 Android

在本部分中，您将学习如何在 Android 中设置 Google 登录。

:::caution
Android SHA1 证书非常痛苦，我不希望任何人必须设置它。以下步骤假设应用程序未发布到 Google Play Store 并且仅由本地模拟器或开发硬件设备使用的最简单场景。 

如果您已将应用程序部署到 Google Play 商店，则**必须**添加一个额外的 Android 客户端 ID，其中包含来自 Google Play 控制台的 SHA1 以用于生产版本。您可以在 `Test and release > Setup > App Signing` 下找到 Google Play 用于签署发布包的 SHA1 哈希值。

最后，值得一提的是，如果你搞砸了，错误不会很明显。调试起来可能会非常困难。如果您在设置时遇到困难，请查看 [Github issues](https://github.com/Cap-go/capacitor-social-login/issues)。

此外，您可以查看 [Google Android 的登录设置](#troubleshooting) 的故障排除部分以获取更多信息。
:::

:::note
您可以创建多个 Android 客户端 ID。如果您有多个 SHA1 证书，则这是必需的。
:::

<Steps>
1. 创建 Android 客户端 ID。
   1. 点击搜索栏
      <img src="/social-login-assets/google_cons_search.png" alt="Google 控制台搜索栏" />
   
   2. 搜索 `credentials` 并单击“API 和服务”（屏幕截图中的数字 2）
      <img src="/social-login-assets/google_cons_cred_search.png" alt="搜索结果显示凭据选项，并突出显示 API 和服务" />
   
   3. 单击“创建凭据”
      <img src="/social-login-assets/google_cons_create_cred.png" alt="在 Google 控制台中创建凭据按钮" />
   
   4. 选择`OAuth client ID`
      <img src="/social-login-assets/google_cons_cred_oauth.png" alt="凭证创建菜单中的 OAuth 客户端 ID 选项" />

   5. 选择`Android`应用程序类型
      <img src="/social-login-assets/google_cons_app_type_android.png" alt="应用程序类型选择，突出显示 Android 选项" />
   
   6.打开Android工作室
   
   7. 在导航器的最底部，找到“Gradle Scripts”
      <img src="/social-login-assets/studio_gradle_scripts.png" alt="Android Studio 项目导航器中的 Gradle 脚本部分" />
   
   8. 找到模块 `app` 的 `build.gradle`
      <img src="/social-login-assets/studio_build_gradle.png" alt="Gradle 脚本部分中的 build.gradle（模块：app）文件" />
   
   9. 复制 `android.defaultConfig.applicationId`。这将是您在 Google 控制台中的“包名称”
      <img src="/social-login-assets/studio_build_gradle_app_id.png" alt="显示 applicationId 配置的 Build.gradle 文件" />
   
   10. 现在，打开终端。确保您位于应用程序的 `android` 文件夹中并运行 `./gradlew signInReport`
      <img src="/social-login-assets/term_sign_report.png" alt="显示 gradlew signInReport 命令的终端" />
   
   11. 滚动到该命令的顶部。您应该看到以下内容。复制 `SHA1`。
      <img src="/social-login-assets/term_sign_report_res.png" alt="显示 SHA1 证书指纹的终端输出" />
   
   12. 现在，返回 Google 控制台。在证书字段中输入您的 `applicationId` 作为“包名称”和 SHA1，然后单击 `create`
      <img src="/social-login-assets/google_cons_creat_android_client.png" alt="Android 客户端创建表单，其中填写了包名称和 SHA1 字段" />
   
2. 创建一个 Web 客户端（Android 需要此）
   
   1. 转到 Google 控制台中的“创建凭证”页面
   
   2. 将应用程序类型设置为 `Web`
      <img src="/social-login-assets/google_cons_app_type_web.png" alt="应用程序类型选择，突出显示 Web 选项" />
   
   3. 单击`Create`
      <img src="/social-login-assets/google_cons_web_app_create.png" alt="Web 客户端创建表单，底部带有“创建”按钮" />
   
   4. 复制客户端 ID，您将在 JS/TS 代码中使用它作为 `webClientId`
      <img src="/social-login-assets/google_cons_copy_web_client_id.png" alt="显示要复制的 Web 客户端 ID 的客户端 ID 详细信息" />

3.修改您的`MainActivity`1. 请在Android Studio中打开您的应用程序。您可以运行“cap open android”
   
   2.找到`MainActivity.java`
     
      1.打开`app`文件夹
         <img src="/social-login-assets/studio_app_folder.png" alt="Android Studio 项目导航器中的应用程序文件夹" />
     
      2.找到`java`
         <img src="/social-login-assets/studio_app_java.png" alt="Android Studio 项目结构中的 Java 文件夹" />
     
      3. 找到您的 `MainActivity.java` 并单击它
         <img src="/social-login-assets/studio_app_java_activity_main.png" alt="包结构中的MainActivity.java文件" />
   
   3.修改`MainActivity.java`。请添加以下代码
     
      ```java
      import ee.forgr.capacitor.social.login.GoogleProvider;
      import ee.forgr.capacitor.social.login.SocialLoginPlugin;
      import ee.forgr.capacitor.social.login.ModifiedMainActivityForSocialLoginPlugin;
      import com.getcapacitor.PluginHandle;
      import com.getcapacitor.Plugin;
      import android.content.Intent;
      import android.util.Log;
      import com.getcapacitor.BridgeActivity;
      
      // ModifiedMainActivityForSocialLoginPlugin is VERY VERY important !!!!!!    
      public class MainActivity extends BridgeActivity implements ModifiedMainActivityForSocialLoginPlugin {
      
            @Override
            public void onActivityResult(int requestCode, int resultCode, Intent data) {
              super.onActivityResult(requestCode, resultCode, data);
      
              if (requestCode >= GoogleProvider.REQUEST_AUTHORIZE_GOOGLE_MIN && requestCode < GoogleProvider.REQUEST_AUTHORIZE_GOOGLE_MAX) {
                PluginHandle pluginHandle = getBridge().getPlugin("SocialLogin");
                if (pluginHandle == null) {
                  Log.i("Google Activity Result", "SocialLogin login handle is null");
                  return;
                }
                Plugin plugin = pluginHandle.getInstance();
                if (!(plugin instanceof SocialLoginPlugin)) {
                  Log.i("Google Activity Result", "SocialLogin plugin instance is not SocialLoginPlugin");
                  return;
                }
                ((SocialLoginPlugin) plugin).handleGoogleLoginIntent(requestCode, data);
              }
            }
      
            // This function will never be called, leave it empty
            @Override
            public void IHaveModifiedTheMainActivityForTheUseWithSocialLoginPlugin() {}
      }
      ```

   4. 保存文件

4. 在您的应用程序中使用 Google 登录
   
   1.首先导入`SocialLogin`
     
      ```typescript
      import { SocialLogin } from '@capgo/capacitor-social-login';
      ```
   
   2. 调用初始化。这应该只被调用一次。
     
      ```typescript
      // onMounted is Vue specific
      // webClientId is the client ID you got in the web client creation step not the android client ID.
      onMounted(() => {
        SocialLogin.initialize({
          google: {
            webClientId: '673324426943-avl4v9ubdas7a0u7igf7in03pdj1dkmg.apps.googleusercontent.com',
          }
        })
      })
      ```
     
   3. 致电 `SocialLogin.login`。创建一个按钮并在单击时运行以下代码。
     
      ```typescript
      const res = await SocialLogin.login({
        provider: 'google',
        options: {}
      })
      // handle the response
      console.log(JSON.stringify(res))
      ```

   6.点击下一步
      <img src="/social-login-assets/studio_new_dev_next_1.png" alt="系统映像选择屏幕中的“下一步”按钮" />
   
   7. 确认您的设备。您可以根据自己的喜好命名您的模拟器

## 故障排除

如果您有任何问题，请查看 [Github issues](https://github.com/Cap-go/capacitor-social-login/issues)。

Google 登录问题**始终**与 SHA1 证书相关。

如果无法获取开发 SHA1 证书，请尝试使用自定义密钥库。 [此处](https://github.com/Cap-go/capacitor-social-login/issues/147#issuecomment-2849742574) 是一条注释，解释如何将密钥库添加到您的项目中。
