## 简介

在本指南中，您将了解如何为 iOS 设置 Google 使用 Capgo 社交登录登录。
我假设您已经阅读了[一般设置指南](/docs/plugins/social-login/google/general/)。

## 使用 Google 登录 iOS

在本部分中，您将学习如何在 iOS 中设置 Google 登录。

<Steps>
1. 在 Google 控制台中创建 iOS 客户端 ID
   
   1. 点击搜索栏
      <img src="/social-login-assets/google_cons_search.png" alt="Google 控制台搜索栏" />
   
   2. 搜索 `credentials` 并单击“API 和服务”（屏幕截图中的数字 2）
      <img src="/social-login-assets/google_cons_cred_search.png" alt="搜索结果显示凭据选项，并突出显示 API 和服务" />
   
   3. 单击“创建凭据”
      <img src="/social-login-assets/google_cons_create_cred.png" alt="在 Google 控制台中创建凭据按钮" />
   
   4. 选择`OAuth client ID`
      <img src="/social-login-assets/google_cons_cred_oauth.png" alt="凭证创建菜单中的 OAuth 客户端 ID 选项" />
   
   5. 选择“应用程序类型”为 `iOS`
      <img src="/social-login-assets/goolge_cons_cred_type_app_tye.png" alt="应用程序类型选择，突出显示 iOS 选项" />
   
   6. 查找捆绑包 ID
     
      1. 打开Xcode
     
      2. 双击`App`
         <img src="/social-login-assets/xcode_app_click.png" alt="Xcode 项目导航器中的应用程序目标" />
     
      3. 确保您处于`Targets -> App`
         <img src="/social-login-assets/xcode_targets_app.png" alt="Xcode 中的目标部分并选择了应用程序" />
     
      4. 找到您的“捆绑包标识符” 
         <img src="/social-login-assets/xcode_bundle_id.png" alt="Xcode 项目设置中的捆绑包标识符字段" />
     
      5. 返回 Google 控制台并将“Bundle Identifier”粘贴到“Bundle ID”中
         <img src="/social-login-assets/google_cons_ios_bd_id.png" alt="Google 控制台 iOS 客户端创建表单中的捆绑 ID 字段" />
   
   7. （可选）如果您已将应用程序发布到 App Store，请将您的 `App Store ID` 或“团队 ID”添加到客户端 ID 中
   
   8. 填写完所有详细信息后，单击`create`
      <img src="/social-login-assets/google_cons_ios_cred_creat.png" alt="在 iOS 客户端创建表单底部创建按钮" />
   
   9. 单击`OK`
      <img src="/social-login-assets/google_cons_ios_click_ok.png" alt="客户端 ID 上的“确定”按钮已创建确认对话框" />
   
   10. 打开新创建的 iOS 客户端
       <img src="/social-login-assets/google_cons_open_new_ios.png" alt="在凭据列表中新创建的 iOS 客户端" />
   
   11. 复制以下数据
       <img src="/social-login-assets/google_cons_ios_what_to_copy.png" alt="客户端 ID 详细信息显示要复制的客户端 ID 和反向客户端 ID" />
     
       :::注意
       此图像中的 `nr. 1` 稍后将成为 `initialize` 调用中的 `iOSClientId` 。
       
       此图像中的 `nr. 2` 稍后将变为 `YOUR_DOT_REVERSED_IOS_CLIENT_ID`
       :::

2. 修改应用程序的 Info.plist
   
   1. 打开 Xcode 并找到 `Info.plist` 文件
      <img src="/social-login-assets/xcode_info_file.png" alt="Xcode 项目导航器中的 Info.plist 文件" />
   
   2. 右键单击该文件并将其作为源代码打开
      <img src="/social-login-assets/xcode_open_as_src_code.png" alt="显示“打开为源代码”选项的右键菜单" />
   
   3. 在 `Plist` 文件的底部，您将看到 `</dict>` 标签
      <img src="/social-login-assets/xcode_dict_tag.png" alt="Info.plist 文件中的关闭 dict 标签" />
   
   4. 在 `</dict>` 结束标记之前插入以下片段
      <img src="/social-login-assets/xcode_plist_inserted.png" alt="Info.plist，在关闭 dict 标签之前插入了 URL 方案代码" />
     
      ```xml
      <key>CFBundleURLTypes</key>
      <array>
       <dict>
         <key>CFBundleURLSchemes</key>
         <array>
           <string>YOUR_DOT_REVERSED_IOS_CLIENT_ID</string>
         </array>
       </dict>
      </array>
      ```
   
   5. 将 `YOUR_DOT_REVERSED_IOS_CLIENT_ID` 更改为上一步中复制的值
      <img src="/social-login-assets/xcode_plist_final.png" alt="Info.plist，在 URL 方案中插入实际反向客户端 ID" />
     
      :::注意
      确保该值**以 `com.googleusercontent.apps` 开头**
      :::
   
   6. 使用“Command + S”保存文件

3. 修改 `AppDelegate.swift`
   
   1. 打开 AppDelegate
      <img src="/social-login-assets/xcode_app_deleg.png" alt="Xcode 项目导航器中的 AppDelegate.swift 文件" />
   
   2. 在文件顶部插入 `import GoogleSignIn`
      <img src="/social-login-assets/xcode_app_deleg_google_sign_in.png" alt="添加了 GoogleSignIn 导入的 AppDelegate.swift" />
   
   3. 找到 `func application(_ app: UIApplication, open url: URL, options: [UIApplication.OpenURLOptionsKey: Any] = [:])` 函数
      <img src="/social-login-assets/xcode_app_deleg_app_fn.png" alt="AppDelegate中原始应用程序的openURL函数" />
   
   4. 将函数修改为如下所示
     
      ```swift
      func application(_ app: UIApplication, open url: URL, options: [UIApplication.OpenURLOptionsKey: Any] = [:]) -> Bool {
          // Called when the app was launched with a url. Feel free to add additional processing here,
          // but if you want the App API to support tracking app url opens, make sure to keep this call
      
          var handled: Bool
      
          handled = GIDSignIn.sharedInstance.handle(url)
          if handled {
            return true
          }
      
          return ApplicationDelegateProxy.shared.application(app, open: url, options: options)
      }
      ```
      
      <img src="/social-login-assets/xcode_app_deleg_app_fn_mod.png" alt="使用 GoogleSignIn 处理修改应用程序 openURL 函数" />
   
   5. 使用“Command + S”保存文件

4. 在 JavaScript/TypeScript 代码中设置 Google 登录
   
   1. 导入 `SocialLogin` 和 `Capacitor`
     
      ```typescript
      import { SocialLogin } from '@capgo/capacitor-social-login';
      import { Capacitor } from '@capacitor/core';
      ```
   
   2. 调用 `initialize` 方法（应该只调用一次）

      **基本设置（在线模式，建议大多数应用程序使用）：**
      ```typescript
      // onMounted is Vue specific
      onMounted(() => {
        SocialLogin.initialize({
          google: {
            iOSClientId: '673324426943-redacted.apps.googleusercontent.com',
            mode: 'online' // Default mode
          }
        })
      })
      ```

      **使用附加客户端 ID 进行高级设置：**
      ```typescript
      onMounted(() => {
        SocialLogin.initialize({
          google: {
            webClientId: 'YOUR_WEB_CLIENT_ID',           // Optional: for web platform support
            iOSClientId: 'YOUR_IOS_CLIENT_ID',           // Required: from step 1
            iOSServerClientId: 'YOUR_WEB_CLIENT_ID',     // Optional: same as webClientId, needed for some advanced features
            mode: 'online'                               // 'online' or 'offline'
          }
        })
      })
      ```
     
      :::注意
      **客户端 ID 要求：**
      - `iOSClientId`：**必需** - 必须以 `googleusercontent.com` 结尾（来自上面的步骤 1）
      - `webClientId`：**可选** - 仅当您也支持 Web 平台或需要高级功能时才需要
      - `iOSServerClientId`：**可选** - 提供时应与 `webClientId` 的值相同
      
      有关 `webClientId` 和 `iOSServerClientId` 的高级设置，请参阅 [Web 设置指南](/docs/plugins/social-login/google/web/) 创建这些凭据。
      :::

      :::注意
      **关于离线模式：** 使用`mode: 'offline'`时，登录响应不会直接包含用户数据。相反，您将收到一个服务器身份验证代码，必须通过后端服务器将其交换为用户信息。请参阅[一般设置指南](/docs/plugins/social-login/google/general/#using-offline-access-with-your-own-backend) 了解实施细节。
      :::
   
   3. 实现登录功能。创建一个按钮并在单击时运行以下代码
     
      **对于在线模式：**
      ```typescript
      const res = await SocialLogin.login({
        provider: 'google',
        options: {}
      })
      // handle the response - contains user data
      console.log(JSON.stringify(res))
      ```

      **对于离线模式：**
      ```typescript
      const res = await SocialLogin.login({
        provider: 'google',
        options: {
          forceRefreshToken: true // Recommended for offline mode
        }
      })
      // res contains serverAuthCode, not user data
      // Send serverAuthCode to your backend to get user information
      console.log('Server auth code:', res.result.serverAuthCode)
      ```

5. 测试您的应用程序
   
   1. 构建您的应用程序并运行“capsync”
   
   2. 如果您已正确完成所有操作，您应该会看到 Google 登录流程正常工作
      <img src="/social-login-assets/google_final_ios_v2.gif" alt="iOS 上的 Google 登录流程演示显示登录过程和成功的身份验证" />
</Steps>

:::note
Google 提示中的语言取决于您设备的语言设置。
:::

## 已知问题

### 隐私屏幕插件不兼容

Google 登录插件与 [@capacitor/privacy-screen](https://github.com/ionic-team/capacitor-privacy-screen) 不兼容。当同时使用这两个插件时，Google 登录 Web 视图将被隐私屏幕中断。

**解决方法：** 在调用登录函数之前调用 `await PrivacyScreen.disable();`：

```typescript
import { PrivacyScreen } from '@capacitor/privacy-screen';
import { SocialLogin } from '@capgo/capacitor-social-login';

await PrivacyScreen.disable();
await SocialLogin.login({
  provider: 'google',
  options: {}
});
```
