## 功能对等

如果您选择使用自己的服务器,您将失去 5 分钟的设置流程。\
您需要自己实现所有这些功能。


| Features                 | Capgo | Self hosted |
| ------------------------ | ----- | ----------- |
| Updates                  | ✅     | 🚧         |
| Auto revert              | ✅     | 🚧         |
| Email alerts on fail     | ✅     | 🚧         |
| Channels                 | ✅     | 🚧         |
| Channels Override        | ✅     | 🚧         |
| Device Override          | ✅     | 🚧         |
| Channels Settings        | ✅     | 🚧         |
| Device Settings          | ✅     | 🚧         |
| Custom ID                | ✅     | 🚧         |
| Auto Set Channels        | ✅     | 🚧         |
| API Channels             | ✅     | 🚧         |
| Updates Statistics       | ✅     | 🚧         |
| Fail Download Statistics | ✅     | 🚧         |
| App Usage Statistics     | ✅     | 🚧         |
| Update Encryption        | ✅     | 🚧         |
| Differential updates     | ✅     | ❌         |

:::danger
如果您向用户发送错误的更新,您可能会并且将会破坏他们的应用。
:::

> 请注意,您不能同时使用 Capgo 云和您的服务器。



:::danger
空中下载 (OTA) 更新功能仅适用于对 HTML、CSS 和 JavaScript 文件所做的修改。

如果您对原生代码进行任何更改,例如更新 Capacitor 插件,则必须将应用重新提交到应用商店以供审批。
:::

## 在自动和手动之间选择
在自动模式下,部分逻辑由原生代码处理,更新由服务器端决定,这更安全并允许细粒度更新、部分部署到一个设备或组等。

在手动模式下,所有逻辑都由 JS 处理。

## 安装 Capacitor Updater

```bash title="安装 Capacitor updater"
npm install @capgo/capacitor-updater
npx cap sync
```

## 准备您的 bundle

要向应用发送更新,您需要压缩它。
确保 zip 文件正确的最佳方法是使用 Capgo CLI 进行压缩。
```bash title="创建一个包含要从服务器提供的文件的 bundle"
npx @capgo/cli@latest bundle zip
```


您将必须自己从服务器提供此 zip 文件。
