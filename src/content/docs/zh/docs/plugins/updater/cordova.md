您一直在想这个插件是否会支持 Cordova。

我们为此启动了一个研发仓库，但这是一项巨大的工作量。

## 问题

我们知道我们可以做到，但为此，我们必须像为 Capacitor 所做的那样阅读 Cordova 代码库的所有代码，以了解如何使其与所有 Capgo 功能配合使用。

Android 版本更容易实现，因为两者都使用 Java，但 iOS 需要完全重写，因为 Swift 在 Cordova 中仍然没有得到很好的支持

## 解决方案

与此同时，您可以这样做：

* 在 GitHub 上[支持我们](https://github.com/sponsors/cap-go)，我们可以优先考虑这一点。
这至少需要 1 个月的工作。
* 聘请我们作为[顾问](https://capgo.app/consulting/)，我们习惯于帮助大公司迁移到 Capacitor，通常需要一个月，而且对您的团队来说[好处](https://ionic.io/resources/articles/capacitor-vs-cordova-modern-hybrid-app-development)巨大
