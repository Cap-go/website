## 理念

Ionic 团队根据自己的优先级和发布计划维护 Capacitor。社区贡献可能需要等待数月甚至数年才能合并。Capacitor+ 采用不同的方法：

1. **合并来自分支的 PR** - 积极合并上游队列中受阻的有价值 PR
2. **持续同步** - 自动拉取和测试每个上游更改
3. **快速发布** - 更改通过 CI 后立即发布到 npm
4. **社区优先** - 您的贡献很重要并获得优先考虑
5. **透明度** - 所有自动化都是开源的且可见
