El comando `channel` te permite gestionar tus canales de lanzamiento

### add

`npx @capgo/cli channel add [channelId] [appId]`

Crea un nuevo canal para una aplicación

Opciones:

- `-d, --default`: Establecer el nuevo canal como canal predeterminado
- `-a, --apikey <apikey>`: Clave API para vincular a tu cuenta

### delete

`npx @capgo/cli channel delete [channelId] [appId]`

Elimina un canal de una aplicación

Opciones:

- `-a, --apikey <apikey>`: Clave API para vincular a tu cuenta
- `--delete-bundle`: Eliminar el paquete asociado con el canal

### list

`npx @capgo/cli channel list [appId]`

Lista todos los canales de una aplicación

Opciones:

- `-a, --apikey <apikey>`: Clave API para vincular a tu cuenta

### currentBundle

`npx @capgo/cli channel currentBundle [channel] [appId]`

Obtiene el paquete actual de un canal específico

Opciones:

- `-c, --channel <channel>`: Canal del cual obtener el paquete actual
- `-a, --apikey <apikey>`: Clave API para vincular a tu cuenta
- `--quiet`: Solo mostrar la versión del paquete

### set

`npx @capgo/cli channel set [channelId] [appId]`

Establece las propiedades de un canal

Opciones:

- `-a, --apikey <apikey>`: Clave API para vincular a tu cuenta
- `-b, --bundle <bundle>`: Número de versión del paquete a establecer para el canal
- `-s, --state <state>`: Establecer el estado del canal (`default` o `normal`)
- `--latest`: Usar la última versión de `packagejson` como versión del paquete
- `--downgrade`: Permitir degradaciones a versiones inferiores a la versión nativa
- `--no-downgrade`: Deshabilitar degradaciones a versiones inferiores a la versión nativa
- `--upgrade`: Permitir actualizaciones a versiones superiores a la versión nativa
- `--no-upgrade`: Deshabilitar actualizaciones a versiones superiores a la versión nativa
- `--ios`: Permitir enviar actualizaciones a dispositivos iOS
- `--no-ios`: Deshabilitar envío de actualizaciones a dispositivos iOS
- `--android`: Permitir enviar actualizaciones a dispositivos Android
- `--no-android`: Deshabilitar envío de actualizaciones a dispositivos Android
- `--self-assign`: Permitir que los dispositivos se auto-asignen a este canal
- `--no-self-assign`: Deshabilitar la auto-asignación de dispositivos a este canal
- `--disable-auto-update <disableAutoUpdate>`: Deshabilitar estrategia de actualización automática para este canal (opciones: `major`, `minor`, `metadata`, `patch`, `none`)
- `--dev`: Permitir enviar actualizaciones a dispositivos de desarrollo
- `--no-dev`: Deshabilitar envío de actualizaciones a dispositivos de desarrollo
- `--emulator`: Permitir enviar actualizaciones a dispositivos emuladores
- `--no-emulator`: Deshabilitar envío de actualizaciones a dispositivos emuladores
- `--package-json <packageJson>`: Lista separada por comas de rutas a archivos `packagejson` (útil para monorepos)
