Para tener un control más preciso sobre el sistema de actualización, puedes configurarlo con estas opciones:

<Aside type="Consejo">
 Cualquier cambio a estas configuraciones en el archivo Capacitor.config, requiere sincronizar la plataforma y lanzar a la tienda para que las aplicaciones en producción las reciban.
</Aside>


## `appReadyTimeout`

> Configura el número de milisegundos que el Plugin nativo debe esperar antes de considerar una actualización como 'fallida'.

Solo disponible para Android e iOS.

Por defecto: `10000` (10 segundos)

```json
// capacitor.config.json
{
  "plugins": {
    "CapacitorUpdater": {
      "appReadyTimeout": 1000
    }
  }
}
```

## `responseTimeout`

> Configura el número de milisegundos que el Plugin nativo debe esperar antes de considerar el timeout de la API.

Solo disponible para Android e iOS.

Por defecto: `20` (20 segundos)

```json
// capacitor.config.json
{
  "plugins": {
    "CapacitorUpdater": {
      "responseTimeout": 10 // (10 segundos)
    }
  }
```

## `directUpdate`

> Configura cuándo el Plugin debe instalar actualizaciones directamente. Solo para modo autoUpdate.
> Funciona bien para Aplicaciones menores a 10MB y con subidas hechas usando la bandera --partial.
> Zip o Aplicaciones de más de 10MB serán relativamente lentas para que los usuarios actualicen.

Opciones:
- `false`: Nunca hacer actualizaciones directas (usar comportamiento por defecto: descargar al iniciar, establecer al pasar a segundo plano)
- `'atInstall'`: Actualización directa solo cuando la Aplicación está instalada, actualizada desde la tienda, de lo contrario actuar como directUpdate = false
- `'onLaunch'`: Actualización directa solo en Aplicación instalada, actualizada desde la tienda o después de matar Aplicación, de lo contrario actuar como directUpdate = false
- `'always'`: Actualización directa en todos los casos anteriores (Aplicación instalada, actualizada desde la tienda, después de matar Aplicación o reanudar Aplicación), nunca actuar como directUpdate = false
- `true`: (obsoleto) Igual que "always" para compatibilidad hacia atrás

Solo disponible para Android e iOS.

Por defecto: `false`

```json
// capacitor.config.json
{
  "plugins": {
    "CapacitorUpdater": {
      "autoUpdate": true,
      "directUpdate": "atInstall"
    }
  }
}
```

> Esta configuración anulará el canal por defecto establecido en la nube, pero seguirá respetando las anulaciones hechas en la nube.

Solo disponible para Android e iOS.

Por defecto: `undefined`

```json
// capacitor.config.json
{
  "plugins": {
    "CapacitorUpdater": {
      "defaultChannel": "production"
    }
  }
}
```

## `autoSplashscreen`

> Manejar automáticamente el ocultamiento del splashscreen cuando se usa directUpdate. Cuando está habilitado, el Plugin ocultará automáticamente el splashscreen después de que se apliquen las actualizaciones o cuando no se necesite actualización.
> Esto elimina la necesidad de escuchar manualmente eventos appReady y llamar a SplashScreen.hide().
> Solo funciona cuando directUpdate está establecido como "atInstall", "always" o true.
> Requiere que el Plugin @Capacitor/splash-screen esté instalado y configurado con launchAutoHide: false.
> Requiere que autoUpdate y directUpdate estén habilitados.

Solo disponible para Android e iOS.

Por defecto: `false`

```json
// capacitor.config.json
{
  "plugins": {
    "CapacitorUpdater": {
      "autoUpdate": true,
      "directUpdate": "atInstall",
      "autoSplashscreen": true
    }
  }
}
```

## `periodCheckDelay`

> Configura el periodo de retraso para la verificación de actualización periódica. La unidad está en segundos.
> No puede ser menor a 600 segundos (10 minutos).

Solo disponible para Android e iOS.

Por defecto: `600` (10 minutos)

```json
// capacitor.config.json
{
  "plugins": {
    "CapacitorUpdater": {
      "periodCheckDelay": 600 // (10 minutos)
    }
  }
}
```

Solo disponible para Android e iOS.

Por defecto: `false`

```json
// capacitor.config.json
{
  "plugins": {
    "CapacitorUpdater": {
      "allowModifyUrl": true
    }
  }
}
```

## `keepUrlPathAfterReload`

> Configura el Plugin para mantener la ruta de URL después de una recarga.

<Aside type="Precaución">
Cuando se activa una recarga, 'window.history' será borrado.
</Aside>

Solo disponible para Android e iOS.

Por defecto: `false`

```json
// capacitor.config.json
{
  "plugins": {
    "CapacitorUpdater": {
      "keepUrlPathAfterReload": true
    }
  }
}
```

Solo disponible para Android e iOS.

Por defecto: `false`

```json

    "CapacitorUpdater": {

  }
```
