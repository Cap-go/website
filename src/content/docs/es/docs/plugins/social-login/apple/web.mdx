Configurar el inicio de sesión web no es trivial. Es más difícil que configurar `Sign in with Apple` en iOS pero más difícil que configurar `Sign in with Apple` en Android.

## Generando el servicio

:::Nota
Este paso es redundante si ya has configurado `Sign in with Apple` en Android.
:::

Por favor, sigue la guía [aquí](/docs/plugins/social-login/apple/android/#creating-the-service-id/) para generar el servicio.

## Configurando las `Return URLs`

<Steps>

1. **Ve a la configuración de tu Service ID**

   En el [Portal de Desarrolladores de Apple](https://developer.apple.com), navega a `Identifiers` > `Services IDs` y haz clic en tu Service ID.

2. **Configura Sign in with Apple**

   Haz clic en `Configure` junto a `Sign in with Apple`.

   ![apple_dev_configure_login](/social-login-assets/apple_dev_configure_login.webp)

3. **Agrega las Return URLs**

   Haz clic en el botón `+` para agregar una nueva URL de retorno.

   ![apple_dev_return_url_plus](/social-login-assets/apple_dev_return_url_plus.webp)

4. **Agrega las Return URLs**

   Agrega tu dominio para tu aplicación web en `Domains and Subdomains`.
   :::Precaución
   **NO PUEDES** agregar `localhost` o `127.0.0.1` como dominio aquí.
   :::

   Luego, agrega tu dominio con el prefijo `https://` y la ruta desde la cual llamarás al inicio de sesión de Apple.
   Por ejemplo, si tu dominio es `https://my-app.com` y llamarás al inicio de sesión de Apple desde `/login`, debes agregar `https://my-app.com/login` como URL de retorno.

   :::Precaución
   **DEBES** agregar tanto el dominio con una barra diagonal al final como sin una barra diagonal al final.
   :::

   ![apple_dev_return_url_add](/social-login-assets/apple_dev_return_url_add.webp)

5. **Guarda los cambios**

   1. Haz clic en el botón `Next` para guardar los cambios.
   2. Haz clic en el botón `Save` para guardar los cambios.

6. Deberías estar listo para probar el inicio de sesión para JavaScript. Ten en cuenta que no puedes probar desde localhost.
</Steps>
