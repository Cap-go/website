## Cosa otterrai

Tutte le guide di integrazione includono:

- **Configurazione semplice**: Configurazione di base per iniziare rapidamente
- **Flussi di lavoro avanzati**: Distribuzioni multi-ambiente con staging e produzione
- **Test dei branch di funzionalità**: Distribuzione automatica dei branch di funzionalità ai canali di test
- **Best practice di sicurezza**: Gestione sicura dei segreti e protezione degli ambienti
- **Monitoraggio**: Notifiche e logging per lo stato della distribuzione

<Aside type="tip">

**Nuovo a CI/CD?** Inizia con la configurazione semplice per la tua piattaforma, poi aggiungi gradualmente funzionalità più avanzate come distribuzioni multi-canale e test automatizzati man mano che le tue esigenze crescono.

</Aside>

## Funzionalità comuni

Ogni integrazione supporta:

- **Build automatizzate**: Attiva distribuzioni sulle modifiche del codice
- **Supporto multi-canale**: Distribuisci su canali diversi (development, staging, production)
- **Test di Pull Request/Merge Request**: Testa le modifiche in ambienti isolati
- **Supporto crittografia**: Distribuzioni sicure con la funzionalità di crittografia di Capgo
- **Protezione degli ambienti**: Approvazioni manuali e accesso limitato per la produzione
- **Notifiche**: Integrazioni con Slack, email e altre notifiche

## Prerequisiti

Prima di configurare qualsiasi integrazione, assicurati di avere:

- Un account Capgo con un'app configurata
- Il codice sorgente della tua app in un repository Git
- Un token API Capgo da [console.capgo.app/apikeys](https://console.capgo.app/apikeys)
- Node.js e npm/yarn configurati nel tuo progetto
