La configurazione dell'accesso web non è banale. È più difficile che impostare `Sign in with Apple` su iOS ma più difficile che impostare `Sign in with Apple` su Android.

## Generazione del servizio

:::note
Questo passaggio è ridondante se hai già configurato `Sign in with Apple` su Android.
:::

Segui la guida [qui](/docs/plugins/social-login/apple/android/#creating-the-service-id/) per generare il servizio.

## Configurazione degli "URL di restituzione".

<Steps>

1. **Vai alla configurazione dell'ID servizio**
   
   Nel [Apple Portale per sviluppatori](https://developer.apple.com), vai a `Identifiers` > "ID servizi" e fai clic sul tuo ID servizio.

2. **Configura l'accesso con Apple**
   
   Fai clic su `Configure` accanto a `Sign in with Apple`.

   ![apple_dev_configure_login](/social-login-assets/apple_dev_configure_login.webp)

3. **Aggiungi gli URL di ritorno**

   Fare clic sul pulsante "+" per aggiungere un nuovo URL di ritorno.

   ![apple_dev_return_url_plus](/social-login-assets/apple_dev_return_url_plus.webp)

4. **Aggiungi gli URL di ritorno**

   Aggiungi il tuo dominio per la tua applicazione web in "Domini e sottodomini".
   :::attenzione
   **NON PUOI** aggiungere `localhost` o `127.0.0.1` come dominio qui.
   :::

   Successivamente, aggiungi il tuo dominio con il prefisso `https://` e il percorso da cui chiamerai Apple Login.
   Ad esempio, se il tuo dominio è `https://my-app.com` e chiamerai Apple Accedi da `/login`, dovresti aggiungere `https://my-app.com/login` come URL di ritorno.

   :::attenzione
   **DEVI** aggiungere sia il dominio con una barra finale che quello senza barra finale.
   :::

   ![apple_dev_return_url_add](/social-login-assets/apple_dev_return_url_add.webp)

5. **Salva le modifiche**

   1. Fare clic sul pulsante `Next` per salvare le modifiche.
   2. Fare clic sul pulsante `Save` per salvare le modifiche.

6. Dovresti essere pronto per testare l'accesso per JavaScript. Tieni presente che non puoi eseguire il test da localhost.
</Steps>
