## Cosa coprirà questo tutorial?

Questo tutorial mostrerà come avviare supabase da zero e come avviare le edge functions

## Requisiti
 1. [capgo](https://github.com/Cap-go/capgo) clonato
 3. [supabase](https://supabasecom/)

## Per iniziare

Per iniziare esegui
```bash
supabase start
```

Dovresti vedere qualcosa del genere:
```js
Started supabase local development setup

         API URL: http://localhost:54321
     GraphQL URL: http://localhost:54321/graphql/v1
          DB URL: postgresql://postgres:postgres@localhost:54322/postgres
      Studio URL: http://localhost:54323
    Inbucket URL: http://localhost:54324
      JWT secret: [truncated]
        anon key: supa_key_anon
service_role key: supa_key_admin
```

Successivamente apri `configsjson` e imposta i seguenti valori:
```json
{
  "base_domain": {
    "prod": "webcapgoapp",
    "development": "developmentwebcapgoapp",
    "local": "localhost:3332"
  },
  "supa_anon": {
    "prod": "supa_key_anon",
    "development": "supa_key_anon",
    "local": "supa_key_anon"
  },
  "supa_url": {
    "prod": "http://localhost:54321",
    "development": "http://localhost:54321",
    "local": "http://localhost:54321"
  }
}
```

dove `supa_key_anon` è il valore del passaggio precedente

:::danger
⚠️ Non committare `configsjson` nel repository remoto
:::

Successivamente, verifica di poter accedere a [localhost:54323](http://localhost:54323/projects) e che la tabella `users` appaia più o meno così

<figure><img src="/supabasewebp" alt="" /><figcaption></figcaption></figure>

Se funziona avvia le edge functions eseguendo:

```bash
supabase functions serve
```

e avvia il frontend eseguendo:
```bash
bun run serve
```
