## Cosa coprirà questo tutorial?

Questo tutorial mostrerà come utilizzare capacitor updater in un ambiente di sviluppo con capgo self-hosted

## Requisiti

1 [Cloned capgo](https://githubcom/Cap-go/capgo)

## Per iniziare

Per utilizzare capacitor updater con capgo self-hosted modifica il `capacitorconfigts` dalla directory della tua app e impostalo così:

```ts
const config: CapacitorConfig = {
  appId: 'comdemoapp',
  appName: 'demoApp',
  webDir: 'dist',
  bundledWebRuntime: false,
  plugins: {
    CapacitorUpdater : {
      statsUrl: "http://localhost:54321/functions/v1/stats",
      channelUrl: "http://localhost:54321/functions/v1/channel_self",
      updateUrl: "http://localhost:54321/functions/v1/updates"
    },
  },
};
```

Questo ti permetterà di utilizzare capgo locale in sviluppo. Tuttavia, di default, questo non è sufficiente

In Android è necessario abilitare la [comunicazione in chiaro](https://developerandroidcom/topic/security/risks/cleartext) poiché di default SSL è disabilitato in locale
Questo può essere fatto modificando [AndroidManifestxml](https://githubcom/Cap-go/capacitor-updater/blob/main/android/src/main/AndroidManifestxml) e aggiungendo `android:usesCleartextTraffic="true"` nel tag `application`
Un esempio completo di questa modifica può essere visto [qui](https://gistgithubcom/WcaleNieWolny/061a015acdebe35eaf3afd7030797701)

Poiché di default Android non consente `http`, capgo ha implementato un controllo per http [qui](https://githubcom/Cap-go/capgo/blob/4cce6c0707d745faa2afbab643e636bad927c602/supabase/functions/updates/indexts#L396)
Questo codice potrebbe non essere sempre qui, ma per ora è necessaria una modifica a questa riga per far funzionare gli aggiornamenti
Questa riga dovrebbe apparire così:
```ts
    if (!signedURL || (signedURLstartsWith('http://') || signedURLstartsWith('https://'))) {
```

:::danger
⚠️ Non committare `supabase/functions/updates/indexts` nel repository remoto\
Questo codice è qui [per un motivo](https://githubcom/Cap-go/capacitor-updater/issues/37#issuecomment-1684916354)
:::

Potrebbe anche esserci un problema che impedisce all'app Android di connettersi. Se non vedi alcuna richiesta inviata alle edge functions esegui
```bash
adb reverse tcp:54321 tcp:54321
```
