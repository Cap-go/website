## Mengelola Channel

Pertama, mari kita lihat halaman channel. Anda dapat mengaksesnya dengan [mengklik aplikasi Anda](/docs/webapp/main-page) dan kemudian [mengklik tab channel](/docs/webapp/main-app-page)

## Membuat Channel

Seperti yang Anda lihat, ada tombol plus di pojok kanan bawah (`1` dalam gambar). Mengklik tombol tersebut akan membuka modal di mana Anda dapat membuat channel baru
<figure><img style="margin-left: auto; margin-right: auto" src="/new_channel_modal.webp" alt="new channel" /><figcaption></figcaption></figure>

Kemudian setelah Anda mengklik `Add` channel baru akan muncul dalam daftar
<figure><img src="/post-channel-create.webp" alt="after channel create" /><figcaption></figcaption></figure>

## Apa arti misconfigured?
Terkadang konfigurasi channel tidak valid. Dalam kasus tersebut, Anda akan mendapatkan peringatan besar dan kolom `Misconfigured` akan menunjukkan `Yes` untuk satu atau lebih channel.
Anda dapat mempelajari lebih lanjut tentang hal ini [di sini](/docs/cli/commands/#disable-updates-strategy)

## Menghapus Channel

Menghapus channel sangat mudah. Cukup klik ikon sampah dan konfirmasi penghapusan (`2` dalam gambar)

## Mengelola Channel

Mengklik nama channel akan membuka modal di mana Anda dapat mengelola pengaturan channel (`3` dalam gambar).

<figure><img src="/channel_settings.webp" alt="Pengaturan channel" /><figcaption></figcaption></figure>

Halaman pengaturan channel berisi semua opsi konfigurasi untuk channel Anda. Mari kita bahas setiap pengaturan.

---
Pertama toggle `Channel default`. Ketika diaktifkan, channel ini menjadi default untuk perangkat baru. Untuk penjelasan lengkap tentang cara kerja channel default, termasuk cara mengatur default khusus platform (satu untuk iOS, satu untuk Android), lihat bagian [Konfigurasi Channel Default](/docs/webapp/main-app-page/#konfigurasi-channel-default).

---
Kedua pengaturan `IOS`. Ini cukup sederhana. Jika ini false maka perangkat iOS tidak akan diizinkan mengunduh pembaruan dari channel ini.

Ketiga adalah pengaturan `Android`. Ini mirip dengan `IOS`. Jika ini false maka perangkat Android tidak akan diizinkan mengunduh pembaruan dari channel ini.

Keempat adalah pengaturan `Nonaktifkan downgrade otomatis di bawah native`. Jika ini true maka akan tidak mungkin untuk menurunkan versi dari versi native. Ini berarti jika Anda telah mengunggah versi `1.2.0` ke App Store atau Play Store dan mencoba mengatur versi channel ke `1.1.0` maka pembaruan (downgrade) akan gagal.

Kelima adalah `Nonaktifkan pembaruan otomatis`. Pengaturan ini cukup kompleks, dan Anda dapat mempelajari lebih lanjut tentang hal ini [di sini](/docs/cli/commands/#disable-updates-strategy)

Untuk `Izinkan build pengembangan`. Jika ini true maka build pengembangan akan diizinkan mengunduh pembaruan dari channel ini. Jika tidak maka setiap permintaan pembaruan yang memiliki `prod` diatur ke false akan ditolak. Ini sebagian besar berguna untuk tujuan pengujian.

Ketujuh adalah `Izinkan Emulator`. Jika ini false maka Capgo akan menolak setiap permintaan pembaruan yang berasal dari emulator. Ini sebagian besar berguna untuk tujuan pengujian.

Kedelapan adalah `Izinkan perangkat untuk mengasosiasi sendiri`. Jika ini true maka metode [setChannel](/docs/plugin/api/#setchannel) akan tersedia. Jika ini diatur ke false dan Anda mencoba memanggil metode [setChannel](/docs/plugin/api/#setchannel) dengan channel ini maka panggilan akan gagal.
