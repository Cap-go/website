## Penggunaan

Impor plugin dan gunakan metodenya untuk mengupload file:

```typescript
import { Uploader } from '@capgo/capacitor-uploader';

// Mulai upload
const startUpload = async () => {
  const upload = await Uploader.startUpload({
    filePath: 'file:///path/to/your/file.jpg',
    serverUrl: 'https://your-server.com/upload',
    method: 'POST',
    headers: {
      'Authorization': 'Bearer your-token'
    },
    parameters: {
      'userId': '12345',
      'type': 'profile'
    },
    notificationTitle: 'Uploading Photo'
  });

  console.log('Upload ID:', upload.id);
};

// Dengarkan event upload
const listener = await Uploader.addListener('events', (event) => {
  switch (event.name) {
    case 'uploading':
      console.log(`Upload ${event.payload.id}: ${event.payload.percent}%`);
      break;
    case 'completed':
      console.log('Upload completed:', event.payload.id);
      console.log('Status code:', event.payload.statusCode);
      break;
    case 'failed':
      console.error('Upload failed:', event.payload.id);
      console.error('Error:', event.payload.error);
      break;
  }
});

// Ingat untuk menghapus listener saat selesai
// listener.remove();

// Batalkan upload
const cancelUpload = async (uploadId: string) => {
  await Uploader.cancelUpload({ id: uploadId });
};

// Dapatkan semua upload aktif
const getActiveUploads = async () => {
  const uploads = await Uploader.getUploads();
  console.log('Active uploads:', uploads);
};
```

## Referensi API

### startUpload(options)

Memulai upload file baru.

```typescript

  filePath: string;

  notificationBody?: string;
  useUtf8Charset?: boolean;
```

## Event

### events

Semua event upload dikirimkan melalui satu listener `events`. Event berisi field `name` untuk mengidentifikasi jenisnya:

```typescript
Uploader.addListener('events', (event: UploadEvent) => {
  switch (event.name) {
    case 'uploading':
      // Upload sedang berlangsung
      console.log(`Progress: ${event.payload.percent}%`);
      console.log(`ID: ${event.payload.id}`);
      break;

    case 'completed':
      // Upload selesai dengan sukses
      console.log(`Completed: ${event.payload.id}`);
      console.log(`Status code: ${event.payload.statusCode}`);
      break;

    case 'failed':
      // Upload gagal
      console.error(`Failed: ${event.payload.id}`);
      console.error(`Error: ${event.payload.error}`);
      break;
  }
});
```

**Jenis Event:**
- `uploading` - Pembaruan progres dengan `percent` dan `id`
- `completed` - Upload selesai dengan `id` dan `statusCode`
- `failed` - Upload gagal dengan `id` dan pesan `error`

## Praktik Terbaik

1. **Tangani semua jenis event**
   ```typescript
   const setupUploadHandlers = () => {
     Uploader.addListener('events', (event) => {
       switch (event.name) {
         case 'uploading':
           handleProgress(event.payload);
           break;
         case 'completed':
           handleCompleted(event.payload);
           break;
         case 'failed':
           handleError(event.payload);
           break;
       }
     });
   };
   ```

2. **Bersihkan listener**
   ```typescript
   // Tambahkan listener
   const listener = await Uploader.addListener('events', handleEvent);

   // Bersihkan saat selesai
   listener.remove();
   ```

3. **Coba lagi upload yang gagal**
   ```typescript
   const retryUpload = async (filePath: string, serverUrl: string) => {
     try {
       await Uploader.startUpload({
         filePath,
         serverUrl,
         maxRetries: 3
       });
     } catch (error) {
       console.error('Upload failed after retries:', error);
     }
   };
   ```

4. **Tampilkan notifikasi upload**
   ```typescript
   await Uploader.startUpload({
     filePath: 'file:///path/to/file',
     serverUrl: 'https://server.com/upload',
     notificationTitle: 'Uploading File',
     notificationBody: 'Your file is being uploaded...'
   });
   ```

## Catatan Platform

### iOS
- Menggunakan `URLSession` untuk upload latar belakang
- Memerlukan kemampuan pemrosesan latar belakang
- Upload berlanjut saat aplikasi ditangguhkan

### Android
- Menggunakan `WorkManager` untuk upload yang andal
- Menampilkan notifikasi layanan latar depan selama upload
- Menghormati pengaturan optimasi baterai
