## Isi ulang langsung ionik

* Saat Anda mengembangkan, jika Anda menggunakan fitur live reload Ionic dari CLI, itu akan menimpa plugin, sehingga Anda tidak akan pernah melihat pembaruan Anda.

## Quasar isi ulang langsung

* Ia menggunakan sistem yang sama dengan ionic, jadi Anda tidak akan melihat pembaruan Anda.

## Pembaruan gagal

* Ini biasanya terjadi ketika pembaruan besar (> 20MB) didorong, sebagian besar pengguna tidak akan mendapatkan versi terakhir.  
Dulu, pengguna harus tetap membuka aplikasi hingga pengunduhan selesai, sekarang kami menggunakan pengunduhan di latar belakang, namun masih dibatasi beberapa detik.

## Android

### Tidak dapat mengunduh

Kami telah melihat beberapa masalah dengan perangkat di India, dan menghubungi pengguna, meminta mereka mencoba server DNS yang berbeda, dan berhasil.

Jadi jika Anda mengalami masalah tersebut, coba gunakan server DNS lain seperti Cloudflare atau Google DNS.

Cloudflare: 1.1.1.1 dan 1.0.0.1

Google DNS: 8.8.8.8 dan 8.8.4.4 atau dns.google

### Selesaikan masalah perbaikan

* Jika Anda mengalami masalah dengan penggunaanCleartextTraffic, itu karena plugin mengikuti praktik baik yang direkomendasikan oleh sonar cloud, dalam 90% kasus plugin akan berfungsi dengan baik, tetapi dengan beberapa plugin yang menyebabkan masalah.

Untuk memperbaikinya, tambahkan `android/app/src/main/AndroidManifest.xml` di kunci `<application>` :

```xml
tools:replace="android:usesCleartextTraffic"
xmlns:tools="http://schemas.android.com/tools"
```

## IOS

### Manifes privasi

Tambahkan kunci kamus `NSPrivacyAccessedAPICategoryUserDefaults` ke [Manifes Privasi](https://capacitorjs.com/docs/ios/privacy-manifest) Anda (biasanya `ios/App/PrivacyInfo.xcprivacy`):

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
  <dict>
    <key>NSPrivacyAccessedAPITypes</key>
    <array>
      <!-- Add this dict entry to the array if the file already exists. -->
      <dict>
        <key>NSPrivacyAccessedAPIType</key>
        <string>NSPrivacyAccessedAPICategoryUserDefaults</string>
        <key>NSPrivacyAccessedAPITypeReasons</key>
        <array>
          <string>CA92.1</string>
        </array>
      </dict>
    </array>
  </dict>
</plist>
```

Kami menyarankan untuk mendeklarasikan [`CA92.1`](https://developer.apple.com/documentation/bundleresources/privacy_manifest_files/describing_use_of_required_reason_api#4278401) sebagai alasan untuk mengakses [`UserDefaults`](https://developer.apple.com/documentation/foundation/userdefaults) API.


### Izin jaringan

Saat menggunakan server lokal untuk menguji pembaruan, aplikasi akan meminta izin jaringan, ini adalah perilaku normal, tidak demikian halnya saat Anda menggunakan server jarak jauh.


## Kedua OS

Saat melakukan pembaruan mode manual, beberapa peristiwa tidak mudah ditangkap, misalnya kegagalan pembaruan terpicu tepat sebelum kode JS Anda dimuat ulang, sehingga Anda tidak akan dapat menangkapnya.

Salah satu alternatifnya adalah membuat daftar bundel dan memeriksa statistik kesalahan untuk mengetahui apakah pembaruan gagal.

Kami perlu menemukan cara yang lebih baik untuk menangani hal ini di masa mendatang, namun ini bukan prioritas, karena mode otomatis adalah cara yang disarankan untuk melakukan pembaruan.

PR dipersilakan untuk membantu kami meningkatkan hal ini.
