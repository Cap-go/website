## Introduzione

In questa guida imparerai come configurare Google Accedi con Capgo Accesso social per Android.
Presumo che tu abbia già letto la [guida generale alla configurazione](/docs/plugins/social-login/google/general/).

## Utilizzando Google accedi su Android

In questa parte imparerai come configurare l'accesso a Google in Android.

:::caution
Il certificato Android SHA1 è più che doloroso e non augurerei a nessuno di doverlo configurare. I passaggi seguenti presuppongono lo scenario più semplice di un'app non pubblicata su Google Play Store e utilizzata solo dal simulatore locale o dal dispositivo hardware di sviluppo. 

Se hai distribuito la tua app su Google Play Store, **DEVI** aggiungere un ulteriore ID client Android che contenga SHA1 dalla console Google Play per le versioni di produzione. Puoi trovare l'hash SHA1 che Google Play utilizza per firmare il tuo bundle di rilascio in `Test and release > Setup > App Signing`.

Infine, è importante ricordare che se sbagli, l'errore NON sarà evidente. Potrebbe essere molto difficile eseguire il debug. Se hai difficoltà con la configurazione, consulta i [problemi di Github](https://github.com/Cap-go/capacitor-social-login/issues).

Inoltre, puoi consultare la sezione relativa alla risoluzione dei problemi della [Google Configurazione dell'accesso per Android](#risoluzione dei problemi) per ulteriori informazioni.
:::

:::note
Puoi creare più ID cliente Android. Ciò è necessario se disponi di più certificati SHA1.
:::

1. Crea un ID cliente Android.
   1. Fare clic sulla barra di ricerca
      <img src="/social-login-assets/google_cons_search.png" alt="Google Barra di ricerca della console" />

   4. Seleziona `OAuth client ID`
      <img src="/social-login-assets/google_cons_cred_oauth.png" alt="Opzione OAuth ID client nel menu di creazione delle credenziali" />

   5. Seleziona il tipo di applicazione `Android`
      <img src="/social-login-assets/google_cons_app_type_android.png" alt="Selezione del tipo di applicazione con l'opzione Android evidenziata" />
   
   6. Apri Android Studio
   
   7. Nella parte inferiore del navigatore, trova gli "Script Gradle".
      <img src="/social-login-assets/studio_gradle_scripts.png" alt="Sezione Script Gradle nel navigatore del progetto Android Studio" />
   
   8. Trova `build.gradle` per il modulo `app`
      <img src="/social-login-assets/studio_build_gradle.png" alt="build.gradle (Modulo: app) nella sezione Script Gradle" />
   
   9. Copia `android.defaultConfig.applicationId`. Questo sarà il tuo "nome pacchetto" nella console Google
      <img src="/social-login-assets/studio_build_gradle_app_id.png" alt="File Build.gradle che mostra la configurazione applicationId" />
   
   10. Ora apri il terminale. Assicurati di essere nella cartella `android` della tua app ed esegui `./gradlew signInReport`
      <img src="/social-login-assets/term_sign_report.png" alt="Terminale che mostra il comando gradlew signInReport" />
   
   11. Scorri fino all'inizio di questo comando. Dovresti vedere quanto segue. Copia `SHA1`.
      <img src="/social-login-assets/term_sign_report_res.png" alt="Output del terminale che mostra l'impronta digitale del certificato SHA1" />
   
   12. Ora torna alla console Google. Inserisci il tuo `applicationId` come `Nome pacchetto` e il tuo SHA1 nel campo del certificato e fai clic su `create`
      <img src="/social-login-assets/google_cons_creat_android_client.png" alt="Modulo di creazione del client Android con il nome del pacchetto e i campi SHA1 compilati" />
   
2. Crea un client Web (richiesto per Android)
   
   1. Vai alla pagina "Crea credenziali" nella Google Console
   
   2. Imposta il tipo di applicazione su `Web`
      <img src="/social-login-assets/google_cons_app_type_web.png" alt="Selezione del tipo di applicazione con l'opzione Web evidenziata" />
   
   3. Fai clic su `Create`
      <img src="/social-login-assets/google_cons_web_app_create.png" alt="Modulo di creazione del client Web con il pulsante Crea in basso" />
   
   4. Copia l'ID cliente, lo utilizzerai come `webClientId` nel tuo codice JS/TS
      <img src="/social-login-assets/google_cons_copy_web_client_id.png" alt="Dettagli dell'ID client che mostrano l'ID client Web da copiare" />

3. Modifica il tuo `MainActivity`1. Apri l'app in Android Studio. Puoi eseguire "cap open Android".
   
   2. Trova `MainActivity.java`
     
      1. Apri la cartella `app`
         <img src="/social-login-assets/studio_app_folder.png" alt="Cartella dell'app nel navigatore del progetto Android Studio" />
     
      2. Trova `java`
         <img src="/social-login-assets/studio_app_java.png" alt="Cartella Java nella struttura del progetto Android Studio" />
     
      3. Trova il tuo `MainActivity.java` e fai clic su di esso
         <img src="/social-login-assets/studio_app_java_activity_main.png" alt="File MainActivity.java nella struttura del pacchetto" />
   
   3. Modifica `MainActivity.java`. Si prega di aggiungere il seguente codice
     
      ```java
      import ee.forgr.capacitor.social.login.GoogleProvider;
      import ee.forgr.capacitor.social.login.SocialLoginPlugin;
      import ee.forgr.capacitor.social.login.ModifiedMainActivityForSocialLoginPlugin;
      import com.getcapacitor.PluginHandle;
      import com.getcapacitor.Plugin;
      import android.content.Intent;
      import android.util.Log;
      import com.getcapacitor.BridgeActivity;
      
      // ModifiedMainActivityForSocialLoginPlugin is VERY VERY important !!!!!!    
      public class MainActivity extends BridgeActivity implements ModifiedMainActivityForSocialLoginPlugin {
      
            @Override
            public void onActivityResult(int requestCode, int resultCode, Intent data) {
              super.onActivityResult(requestCode, resultCode, data);
      
              if (requestCode >= GoogleProvider.REQUEST_AUTHORIZE_GOOGLE_MIN && requestCode < GoogleProvider.REQUEST_AUTHORIZE_GOOGLE_MAX) {
                PluginHandle pluginHandle = getBridge().getPlugin("SocialLogin");
                if (pluginHandle == null) {
                  Log.i("Google Activity Result", "SocialLogin login handle is null");
                  return;
                }
                Plugin plugin = pluginHandle.getInstance();
                if (!(plugin instanceof SocialLoginPlugin)) {
                  Log.i("Google Activity Result", "SocialLogin plugin instance is not SocialLoginPlugin");
                  return;
                }
                ((SocialLoginPlugin) plugin).handleGoogleLoginIntent(requestCode, data);
              }
            }
      
            // This function will never be called, leave it empty
            @Override
            public void IHaveModifiedTheMainActivityForTheUseWithSocialLoginPlugin() {}
      }
      ```

   4. Salvare il file

4. Utilizza Google Accedi nella tua applicazione
   
   1. Innanzitutto, importa `SocialLogin`
     
      ```typescript
      import { SocialLogin } from '@capgo/capacitor-social-login';
      ```
   
   2. Chiama l'inizializzazione. Questo dovrebbe essere chiamato solo una volta.
     
      ```typescript
      // onMounted is Vue specific
      // webClientId is the client ID you got in the web client creation step not the android client ID.
      onMounted(() => {
        SocialLogin.initialize({
          google: {
            webClientId: '673324426943-avl4v9ubdas7a0u7igf7in03pdj1dkmg.apps.googleusercontent.com',
          }
        })
      })
      ```
     
   3. Chiama `SocialLogin.login`. Crea un pulsante ed esegui il seguente codice al clic.
     
      ```typescript
      const res = await SocialLogin.login({
        provider: 'google',
        options: {}
      })
      // handle the response
      console.log(JSON.stringify(res))
      ```

## Risoluzione dei problemi

In caso di problemi, consulta i [problemi di Github](https://github.com/Cap-go/capacitor-social-login/issues).

I problemi con l'accesso Google sono **SEMPRE** correlati al certificato SHA1.

Se non riesci a ottenere il certificato SHA1 di sviluppo, prova a utilizzare un archivio chiavi personalizzato. [Qui](https://github.com/Cap-go/capacitor-social-login/issues/147#issuecomment-2849742574) è un commento che spiega come aggiungere il keystore al tuo progetto.
