## Overview

`@capgo/capacitor-disqo` wraps Disqo's Android Pulse SDK behind a Capacitor TypeScript API.

The plugin initializes the SDK, starts and stops tracking, opens Android accessibility settings, reports service state, and bridges Pulse token callbacks back to JavaScript so the host app can keep using its own auth stack.

## Scope

This plugin does not implement client-facing onboarding UI, Google Play prominent disclosures, Play Console declarations, privacy policy updates, or review videos. The host app is still responsible for those requirements.

## Core Capabilities

- `getSdkStatus` - Checks whether the required Pulse SDK classes are available.
- `initialize` - Configures the Pulse SDK bridge with an API URL and optional cached token.
- `start` and `stop` - Starts or stops a Pulse session.
- `getServiceState` - Reads the current accessibility-backed service state.
- `openAccessibilitySettings` - Opens Android accessibility settings.
- Token callback listeners - Requests access or refresh tokens from JavaScript when native needs them.

## Platform Support

| Platform | Support |
| --- | --- |
| Android | Supported. |
| iOS | Not supported by the underlying Pulse SDK. |
| Web | Not supported. |
