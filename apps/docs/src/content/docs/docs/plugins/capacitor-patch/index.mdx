## Overview

`@capgo/capacitor-patch` applies vetted Capgo patches to Capacitor packages or generated native projects during `cap sync` and `cap update`.

The package is hook-only. It does not register a native runtime API. It is useful when an app needs a repeatable fix for a known Capacitor issue before that fix is available upstream.

## Core Capabilities

- Package phase patches run before `sync` and `update`.
- Native project patches run after `sync` and `update`.
- Version ranges prevent incompatible patches from applying to the wrong Capacitor release.
- Recommended patches can be enabled with one config flag.
- The CLI can list, dry-run, and apply selected patches manually.

## Patch Selection

| Option | Type | Default | Description |
| --- | --- | --- | --- |
| `recommended` | `boolean` | `false` | Applies Capgo-recommended compatible patches. |
| `patches` | `string[]` | `[]` | Applies explicit patch IDs. |
| `disabled` | `string[]` | `[]` | Skips selected patch IDs, even when recommended. |
| `strict` | `boolean` | `false` | Throws when a selected patch is incompatible or cannot apply. |
