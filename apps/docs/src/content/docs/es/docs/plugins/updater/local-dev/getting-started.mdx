## ¿Qué cubrirá este tutorial?

Este tutorial mostrará cómo iniciar la supabase desde cero, así como iniciar las funciones de borde. 

## Requisitos
 1. Clonado [capgo](https://github.com/Cap-go/capgo)
 3. [supabase](https://supabase.com/)

## Empezando

Para empezar a ejecutar
```bash
supabase start
```

A continuación debería ver algo como esto:
```js
Started supabase local development setup.

         API URL: http://localhost:54321
     GraphQL URL: http://localhost:54321/graphql/v1
          DB URL: postgresql://postgres:postgres@localhost:54322/postgres
      Studio URL: http://localhost:54323
    Inbucket URL: http://localhost:54324
      JWT secret: [truncated]
        anon key: supa_key_anon
service_role key: supa_key_admin
```

Luego abra `configs.json` y establezca los siguientes valores:
```json
{
  "base_domain": {
    "prod": "console.capgo.app",
    "development": "development.console.capgo.app",
    "local": "localhost:3332"
  },
  "supa_anon": {
    "prod": "supa_key_anon",
    "development": "supa_key_anon",
    "local": "supa_key_anon"
  },
  "supa_url": {
    "prod": "http://localhost:54321",
    "development": "http://localhost:54321",
    "local": "http://localhost:54321"
  }
}
```

donde `supa_key_anon` es el valor del paso anterior.

:::danger
⚠️ No envíe `configs.json` al repositorio remoto
:::

A continuación, verifique que puede ir a [localhost:54323](http://localhost:54323/projects) y que la tabla `users` se parece a esta

<figure><img src="/supabase.webp" alt="Panel Supabase que muestra la tabla de usuarios" /><figcaption></figcaption></figure>

Si inicia funciones perimetrales ejecutando:

```bash
supabase functions serve
```

e inicie la interfaz ejecutando:
```bash
bun run serve
```
