## APIキーで何ができますか？

APIキーは再生成、削除、または新規追加が可能です。有効期限の設定や、セキュリティ強化のための暗号化キーの作成もできます。

## すべてのAPIキーを確認するには？

[APIキーページ](https://console.capgo.app/dashboard/apikeys/)にアクセスすると、すべてのAPIキーを確認できます。

## 新しいAPIキーを追加するには？

新しいAPIキーを追加するには、小さなプラスボタンをクリックします

<figure><img src="/apikeys-add.webp" alt="add apikey" /><figcaption></figcaption></figure>

そして、新しいAPIキーに付与したい権限を選択します。以下から選択できます：

- **読み取り** - APIキーはすべてのデータを読み取ることができます
- **アップロード** - APIキーはCLIから新しいバージョンをアップロードし、読み取ることができます
- **書き込み** - APIキーは設定、チャネル、デバイスの変更およびアップロードができます
- **すべて** - APIキーはすべての操作を実行できます

## APIキー権限の仕組み

APIキー権限は、組織でRBACシステムが有効かどうかによって、2つのモデルのいずれかに従います。

### レガシーモード（すべての組織）

レガシーモードでは、APIキーは**作成したユーザーの権限を継承**し、キーのモードによって上限が設定されます：

| キーモード | キーでできること |
| -------- | --------------------------------------------------------- |
| `read`   | データの読み取りのみ |
| `upload` | 読み取り + 新しいバンドルのアップロード |
| `write`  | 読み取り + アップロード + チャネル、デバイス、バンドルメタデータの変更 |
| `all`    | 組織内でオーナーができるすべてのこと |

<Aside type="note">
  `super_admin`が`read`キーを作成した場合、そのキーは読み取りしかできません —
  オーナーがフルアクセス権を持っていても同様です。モードは下限ではなく上限として機能します。
</Aside>

これは以下を意味します：

- キーのオーナーが組織へのアクセスを失うと、キーも動作しなくなります
- キーはオーナー自身ができること以上のことはできません
- 異なるモードにより、きめ細かい制御なしでCI/CDキーを簡単に制限できます

### RBACロールバインディング（RBAC有効な組織）

組織でRBACが有効な場合、明示的な**ロールバインディング**をキーに直接割り当てることができます（例：特定の組織に対する`app_uploader`）。キーに明示的なバインディングがある場合：

- **そのバインディングのみが評価されます** — オーナーのユーザー権限は完全に無視されます
- キーは、誰がオーナーであるかに関係なく、バインディングで許可された範囲に厳密に制限されます
- これにより、スーパー管理者であっても最小権限のキーを作成できます

RBAC有効な組織でキーに明示的なバインディングがない場合、レガシーモード（オーナーの権限 + モードによる上限）にフォールバックします。

<Aside type="tip">
  RBACバインディングは組織設定から管理します。詳細な説明と例については、[APIキー権限システム](/docs/webapp/api-key-permissions/)を参照してください。
</Aside>

## 暗号化APIキー

新しいAPIキーを作成する際、**暗号化キー**を作成することを選択できます。暗号化キーはセキュリティを強化します：

- キーはデータベースに保存される前にSHA-256でハッシュ化されます
- **プレーンキーは作成時に一度だけ表示されます** - すぐにコピーして保存してください
- 作成後、キーを取得または再表示することはできません
- 暗号化キーを紛失した場合、新しいキーを作成する必要があります

これは本番環境で推奨されるアプローチです。データベースが侵害されても、実際のAPIキーは安全に保たれます。

## APIキーの有効期限

APIキーに**有効期限**を設定できます。これはキーが漏洩した場合の露出期間を制限するセキュリティのベストプラクティスです。

### 有効期限の設定

APIキーの作成または編集時に、オプションで有効期限を設定できます。有効期限が過ぎると：

- APIキーは認証に使用できなくなります
- 期限切れのキーを使用したリクエストはすべて拒否されます
- 期限切れのキーは30日後に自動的にクリーンアップされます

### 組織ポリシー

組織は有効期限ポリシーを強制できます：

- **有効期限の必須化**: すべてのAPIキーに有効期限が必要です
- **最大有効期間**: 有効期限をどれだけ先に設定できるかを制限します（例：最大90日）
- **セキュアAPIキーの強制**: すべての新しいAPIキーを暗号化/ハッシュ化キーとして作成することを要求します

組織でこれらのポリシーが有効な場合、新しいキーを作成する際に適切な有効期限を設定する必要があります。

組織全体のAPIキーポリシーの設定については、[組織セキュリティ](/docs/webapp/organization-security/)を参照してください。
