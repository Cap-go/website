## 概要

このガイドでは、`@capacitor-community/facebook-login` から `@capgo/capacitor-social-login` に移行するための包括的な手順を説明します。新しいプラグインは、複数のソーシャル プロバイダー、改良された TypeScript サポート、強化された機能をサポートする統合された API で Facebook 認証を最新化します。

## インストール

<Steps>
1. 古いパッケージを削除します。
   ```bash
   npm uninstall @capacitor-community/facebook-login
   ```

2. 新しいパッケージをインストールします。
   ```bash
   npm install @capgo/capacitor-social-login
   npx cap sync
   ```
</Steps>

## コードの変更

### 変更のインポート
```diff
- import { FacebookLogin } from '@capacitor-community/facebook-login';
+ import { SocialLogin } from '@capgo/capacitor-social-login';
```

### 初期化

**重要な変更**: 新しいパッケージでは、コード内で明示的にセットアップする必要があります。

```diff
- // Old package required no explicit initialization in code
- // Configuration was done only in native platforms

+ // New package requires explicit initialization
+ await SocialLogin.initialize({
+   facebook: {
+     appId: 'YOUR_FACEBOOK_APP_ID',        // Required for web and Android
+     clientToken: 'YOUR_CLIENT_TOKEN'       // Required for Android
+   }
+ });
```

### ログイン

ログイン メソッドはプロバイダー パラメーターを受け入れるようになりました。

```diff
- const FACEBOOK_PERMISSIONS = ['email', 'public_profile'];
- const result = await FacebookLogin.login({ permissions: FACEBOOK_PERMISSIONS });

+ const result = await SocialLogin.login({
+   provider: 'facebook',
+   options: {
+     permissions: ['email', 'public_profile'],
+     limitedLogin: false,
+     nonce: 'optional_nonce'
+   }
+ });
```

### 応答タイプの変更

応答構造は、より包括的なプロファイル オブジェクトで最新化されました。

```typescript
// Old response type
interface FacebookLoginResponse {
  accessToken: {
    applicationId: string;
    userId: string;
    token: string;
    expires: string;
  };
  recentlyGrantedPermissions: string[];
  recentlyDeniedPermissions: string[];
}

// New response type
interface FacebookLoginResponse {
  provider: 'facebook';
  result: {
    accessToken: {
      token: string;
      applicationId?: string;
      expires?: string;
      userId?: string;
      permissions?: string[];
      declinedPermissions?: string[];
    } | null;
    idToken: string | null;
    profile: {
      userID: string;
      email: string | null;
      friendIDs: string[];
      birthday: string | null;
      ageRange: { min?: number; max?: number } | null;
      gender: string | null;
      location: { id: string; name: string } | null;
      hometown: { id: string; name: string } | null;
      profileURL: string | null;
      name: string | null;
      imageURL: string | null;
    };
  };
}
```

**主な違い**:
- 応答には、認証プロバイダーを識別する `provider` フィールドが含まれるようになりました。
- 追加のユーザー情報を含む、より詳細な `profile` オブジェクト
- すべてのソーシャル ログイン プロバイダーにわたる一貫した構造

### ログインステータスの確認
```diff
- const result = await FacebookLogin.getCurrentAccessToken();
- const isLoggedIn = result && result.accessToken;

+ const status = await SocialLogin.isLoggedIn({
+   provider: 'facebook'
+ });
+ const isLoggedIn = status.isLoggedIn;
```

### ログアウト
```diff
- await FacebookLogin.logout();

+ await SocialLogin.logout({
+   provider: 'facebook'
+ });
```

## プラットフォーム固有の変更

### Android セットアップ

構成は、initialize メソッドを通じて処理されるようになりました。

```diff
- // AndroidManifest.xml changes remain the same
- // strings.xml become irrelevant
+ // Additionally initialize in your code:
+ await SocialLogin.initialize({
+   facebook: {
+     appId: 'your-app-id',
+     clientToken: 'your-client-token'  // New requirement
+   }
+ });
```

**重要**: Android 認証にはクライアント トークンが必要になりました。

### iOS セットアップ

1. `AppDelegate.swift` の iOS セットアップは変わりません。

```swift
import FBSDKCoreKit

// In application:didFinishLaunchingWithOptions:
FBSDKCoreKit.ApplicationDelegate.shared.application(
    application,
    didFinishLaunchingWithOptions: launchOptions
)

// In application:openURL:options:
ApplicationDelegate.shared.application(
    app,
    open: url,
    sourceApplication: options[UIApplication.OpenURLOptionsKey.sourceApplication] as? String,
    annotation: options[UIApplication.OpenURLOptionsKey.annotation]
)
```

2. `Info.plist` 構成は変わりません。
```xml
<key>CFBundleURLTypes</key>
<array>
    <dict>
        <key>CFBundleURLSchemes</key>
        <array>
            <string>fb[APP_ID]</string>
        </array>
    </dict>
</array>
<key>FacebookAppID</key>
<string>[APP_ID]</string>
<key>FacebookClientToken</key>
<string>[CLIENT_TOKEN]</string>
<key>FacebookDisplayName</key>
<string>[APP_NAME]</string>
<key>LSApplicationQueriesSchemes</key>
<array>
    <string>fbapi</string>
    <string>fbauth</string>
    <string>fb-messenger-share-api</string>
    <string>fbauth2</string>
    <string>fbshareextension</string>
</array>
```

## 重大な変更

移行時の重大な変更の概要:

1. **明示的な初期化が必要になりました** - 使用前に `initialize()` を呼び出す必要があります
2. **応答オブジェクトの構造が大幅に変更されました** - 強化されたプロファイル データを備えた新しいネストされた結果形式
3. **Android__にはクライアント トークンが必要になりました** - 追加の構成が必要です
4. **異なるメソッド名とパラメーター構造** - プロバイダーベースのアプローチ
5. **エラー処理とエラーの種類が変更されました** - より詳細なエラー情報

## 主な利点

新しいプラグインは以下を提供します。

- **複数のソーシャル プロバイダーにわたって API** を統合 (Google、Apple、Facebook)
- **型定義が改善され、TypeScript サポートが改善されました**
- **より多くのユーザー情報を含む強化されたプロフィール データ**
- **アクティブなメンテナンス**とコミュニティ サポート
- **すべてのプロバイダーにわたる一貫したエラー処理**
- **適切な有効期限処理による優れたトークン管理**

セットアップ手順の詳細については、[公式ドキュメント](/docs/plugins/social-login/facebook/) を参照してください。
