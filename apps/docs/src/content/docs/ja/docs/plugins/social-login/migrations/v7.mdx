## はじめに

:::caution
このガイドは、`V1`/`V7` バージョンのプラグインを対象とし、`0.x.x` バージョンでこのプラグインを引き続き使用しているユーザーを対象としています。
:::

このガイドでは次の内容について説明します。

- `main` バージョンから V1 バージョンへの移行
- `development` バージョンから V1 バージョンへの移行

## V1 での重要な変更点

V1 は開発バージョンをメインに移植しただけです。ただし、`main` V0 バージョンでは利用できない重要な変更が多数含まれています。それらの変更には次のものが含まれます。

- Google ログインのアクセス スコープ
- Google ログインのオフライン モード
- 異なる実装の統合
  - Google プロバイダーのすべての実装がプラットフォーム間で同じように動作することを確認するために、広範なテストが実施されました。

## V0 メインバージョンからの移行

- Android の `MainActivity.java` の変更
  - [Google セットアップガイド](/docs/plugins/social-login/google/android/) に従ってください。具体的には、`MainActivity.java` を検索してください。

- Google コンソールにリダイレクト URL を追加してください。リダイレクト URL を追加しないと、Google ログインは機能しません。
  - 再度、[Google セットアップ ガイド](/docs/plugins/social-login/google/android/) に従ってください。具体的には、「承認されたリダイレクト URI」を検索してください。

- 構成で `grantOfflineAccess` を使用していないことを確認してください。この機能は V1 ではサポートされていません。

- すべてのプラットフォームで認証が機能することを確認してください。

## V0 開発版からの移行

- Android の `MainActivity.java` の変更
  - [Google セットアップガイド](/docs/plugins/social-login/google/android/) に従ってください。具体的には、`MainActivity.java` を検索してください。 V1 では、メイン アクティビティに `ModifiedMainActivityForSocialLoginPlugin` を**実装する必要があります**。この変更はプラグインが機能するために重要です

- Google コンソールにリダイレクト URL を追加してください。リダイレクト URL を追加しないと、Google ログインは機能しません。
  - 再度、[Google セットアップ ガイド](/docs/plugins/social-login/google/android/) に従ってください。具体的には、「承認されたリダイレクト URI」を検索してください。

- 型と変数名が正しいことを確認してください。開発時と V1 では型と変数が一致しない可能性があることに注意してください。

- すべてのプラットフォームで認証が機能することを確認してください。
