## 소개

이 가이드는 Android에서 Google 로그인과 Firebase 인증을 통합하는 데 도움이 됩니다. [일반 Firebase Google 설정](./general/)을 이미 완료했다고 가정합니다. 

:::note
Firebase 콘솔에서 아직 Android 앱을 만들지 않았다고 가정하겠습니다. 그렇다면 단계가 약간 다를 것입니다.
:::

## 설정 단계

<Steps>
1. [console.cloud.google.com](https://console.cloud.google.com/)에서 프로젝트 개요로 이동합니다.
    <img src="/social-login-assets/firebase_project_overview.webp" alt="Firebase 프로젝트 개요" />

2. '앱 추가' 버튼을 클릭하세요.
    <img src="/social-login-assets/firebase_project_add_app.webp" alt="Firebase 앱 버튼 추가" />

    :::참고
    이 버튼을 찾으려면 여기를 찾아야 할 수도 있습니다. 이는 Firebase 콘솔에서 이미 다른 앱을 만든 경우에만 적용됩니다.
    :::

    <img src="/social-login-assets/firebase_project_add_app_2.webp" alt="Firebase 앱 버튼 추가" />

3. `Android`을 선택하세요.
    <img src="/social-login-assets/firebase_project_add_app_android.webp" alt="Firebase 앱 추가 Android 버튼" />

4. 양식의 첫 번째 부분을 작성하세요.
   1. `Android package name` 채우기
      1. `npx cap open android`를 사용하여 앱에서 Android Studio를 엽니다.
      2. 네비게이터 맨 아래에서 'Gradle Scripts'를 찾습니다.
         <img src="/social-login-assets/studio_gradle_scripts.png" alt="Android Studio 프로젝트 탐색기의 Gradle 스크립트 섹션" />
      3. `app` 모듈에 대한 `build.gradle`을 찾습니다.
         <img src="/social-login-assets/studio_build_gradle.png" alt="Gradle 스크립트 섹션의 build.gradle(모듈: 앱) 파일" />
      4. `android.defaultConfig.applicationId`을 복사합니다. 이것이 Firebase 콘솔의 `패키지 이름`이 됩니다.
         <img src="/social-login-assets/studio_build_gradle_app_id.png" alt="applicationId 구성을 보여주는 Build.gradle 파일" />
      :::참고
      여기에 표시된 ID는 나머지 가이드에서 사용할 ID와 다릅니다. 가이드의 나머지 부분에서는 `app.capgo.plugin.SocialLogin`을 사용하겠습니다.
      :::
      5. Firebase 콘솔에 붙여넣습니다.
          <img src="/social-login-assets/firebase_project_add_app_android_package_name.webp" alt="Firebase 앱 추가 Android 패키지 이름 필드" />
    2. '앱 등록' 버튼을 클릭하세요.
        <img src="/social-login-assets/firebase_project_add_app_android_register.webp" alt="Firebase 앱 추가 Android 등록 버튼" />
5. `구성 파일 다운로드 후 추가` 단계를 건너뜁니다.
    <img src="/social-login-assets/firebase_project_add_app_android_skip_download.webp" alt="Firebase 앱 추가 Android '다운로드 후 구성 파일 추가' 건너뛰기" />
6. `Add firebase SDK` 단계를 건너뜁니다.
   <img src="/social-login-assets/firebase_project_add_app_android_skip_download_firebase_sdk.webp" alt="Firebase 앱 추가 Android 'firebase 추가 SDK' 건너뛰기" />
7. '콘솔로 계속' 버튼을 클릭하세요.
   <img src="/social-login-assets/firebase_project_add_app_android_continue_to_console.webp" alt="Firebase 앱 추가 Android 콘솔 버튼으로 계속 진행" />
8. 자동으로 인증되지 않으면 `settings` -> `general` -> `내 앱`으로 이동하여 -> Android 앱을 찾아 클릭하세요.
   <img src="/social-login-assets/firebase_project_add_app_android_settings_general_your_apps.webp" alt="Firebase 앱 추가 Android 설정 일반 앱 버튼" />
9. SHA1 지문을 얻으세요
   
   [Google 로그인 Android 설정 가이드](/docs/plugins/social-login/google/android/#using-google-login-on-android)의 10~11단계를 따르세요.
   
   1. 이제 터미널을 열어주세요. 앱의 `android` 폴더에 있는지 확인하고 `./gradlew signInReport`을 실행하세요.
      <img src="/social-login-assets/term_sign_report.png" alt="gradlew signInReport 명령을 보여주는 터미널" />
   
   2. 이 명령의 맨 위로 스크롤하십시오. 다음을 확인해야 합니다. `SHA1`을 복사합니다.
      <img src="/social-login-assets/term_sign_report_res.png" alt="SHA1 인증서 지문을 보여주는 터미널 출력" />

   :::참고
   원래 Google 로그인 안드로이드 설정 가이드를 작성한 이후 계산을 변경했기 때문에 가이드의 나머지 부분에서는 약간 다른 SHA1을 사용하겠습니다.
   :::

   :::주의
   SHA1은 올바르게 작동하는 데 매우 중요합니다. 엉망으로 만들면 이상한 방식으로 인증이 실패합니다.
   올바르게 설정하려면 __**[Google 로그인 Android 설정 가이드를 읽어보세요](/docs/plugins/social-login/google/android/#using-google-login-on-android)**__하세요.
   :::
   
10. Firebase 프로젝트에 SHA1을 추가합니다.
      1. '지문 추가' 버튼을 클릭하세요.
         <img src="/social-login-assets/firebase_project_add_app_android_add_fingerprint.webp" alt="Firebase 앱 추가 Android 지문 추가 버튼" />
      2. 이전 단계에서 복사한 SHA1을 붙여넣습니다.
         <img src="/social-login-assets/firebase_project_add_app_android_add_fingerprint_sha1.webp" alt="Firebase 앱 추가 Android 지문 SHA1 필드 추가" />
      3. `Save` 버튼을 클릭하세요.
         <img src="/social-login-assets/firebase_project_add_app_android_add_fingerprint_add.webp" alt="Firebase 앱 추가 Android 지문 저장 버튼 추가" />11. 웹 클라이언트 ID 받기
      1. `Build` -> `Authentication`로 이동합니다.
         <img src="/social-login-assets/firebase_select_authentication.webp" alt="Firebase 인증 메뉴" />
      2. '로그인 방법' 버튼을 클릭하세요.
         <img src="/social-login-assets/firebase_select_authentication_sign_in_method.webp" alt="Firebase 인증 로그인 방법 버튼" />
      3. `Google` 공급자를 클릭하세요.
         <img src="/social-login-assets/firebase_select_authentication_sign_in_method_google.webp" alt="Firebase 인증 로그인 방법 Google 공급자" />
      4. `Web SDK configuration` 버튼을 클릭하세요.
         <img src="/social-login-assets/firebase_select_authentication_sign_in_method_web_sdk_configuration.webp" alt="Firebase 인증 로그인 방법 웹 SDK 구성 버튼" />
      5. '웹 클라이언트 ID'를 복사하세요. 이는 플러그인의 `initialize` 메서드에서 `webClientId`이 됩니다.
        <img src="/social-login-assets/firebase_select_authentication_sign_in_method_web_sdk_configuration_web_client_id.webp" alt="Firebase 인증 로그인 방법 웹 SDK 구성 웹 클라이언트 ID" />
12. JS에서 웹 클라이언트 ID를 사용합니다.
      :::참고
      예제 앱의 [authUtils.ts](https://github.com/Cap-go/capacitor-social-login/blob/main/example-app/src/authUtils.ts) 파일에 있는 `authenticateWithGoogle` 도우미 함수를 사용하는 것이 좋습니다.
      :::
</Steps>

이제 Android에서 Firebase 인증을 사용하여 Google 로그인을 사용할 수 있는 **__기술적으로__** 준비가 되었습니다. 하지만 다음 단계에 설명된 대로 Google Cloud 콘솔에서 설정을 다시 확인하는 것이 좋습니다.

## Google 클라우드 콘솔에서 설정을 다시 확인하세요.

설정이 올바른지 확인하려면 Google Cloud 콘솔에서 설정을 다시 확인해야 합니다.
<Steps>
   1. [console.cloud.google.com](https://console.cloud.google.com/)으로 이동합니다.
   2. 프로젝트 찾기
      1. 프로젝트 선택기를 클릭하세요
         <img src="/social-login-assets/firebase_double_check_gc_project_select.webp" alt="Google Cloud Console 프로젝트 선택기" />
      2. Firebase 프로젝트의 정확한 이름으로 프로젝트를 검색하고 클릭하세요. 제 경우에는 `sociallogin-tutorial-app`입니다.
         <img src="/social-login-assets/firebase_gc_project_select_project.webp" alt="Firebase 프로젝트 선택기 프로젝트" />
   3. 검색창을 열고 `credentials`을 엽니다.
      1. 검색창을 엽니다
         <img src="/social-login-assets/firebase_double_check_gc_search_bar.webp" alt="Google Cloud Console 검색창" />
      2. `credentials`을 검색하고 'API 및 서비스' 항목(스크린샷의 2번)을 클릭합니다.
         <img src="/social-login-assets/firebase_gc_credentials_search.webp" alt="Google Cloud Console 자격 증명 검색" />
   4. 목록에 Android 및 웹 클라이언트 ID가 모두 표시되는지 확인합니다.
      <img src="/social-login-assets/firebase_gc_credentials_list.webp" alt="Google Cloud Console 자격 증명 목록" />

      :::주의
      목록에 Android 및 웹 클라이언트 ID가 모두 표시되지 않으면 설정에 실수가 있는 것입니다. 돌아가서 단계를 확인하세요.

      또한 다른 프로젝트에 동일한 앱 ID를 사용하여 이미 Android SHA1 해시를 추가했을 수도 있습니다. 이로 인해 Firbase는 Android 클라이언트 ID를 생성할 수 없게 됩니다. 이 경우 다른 프로젝트와 Firebase(Android 앱을 제거하려면 Firebase 콘솔을 사용)에서 SHA1을 제거하고 Firbase에서 다시 생성해야 합니다.
      :::
   5. Android 클라이언트 ID가 Firebase 콘솔에 올바르게 구성되었는지 확인합니다.
      1. `Android` 앱을 클릭하세요
         <img src="/social-login-assets/firebase_gc_android_app_select.webp" alt="Google 클라우드 콘솔 Android 앱 선택" />
      2. SHA1 해시가 올바르게 구성되었는지, 이전 단계에서 복사한 해시와 일치하는지 확인합니다.
         <img src="/social-login-assets/firebase_gc_android_app_sha1_configured.webp" alt="Google Cloud Console Android 앱 SHA1 구성됨" />
   6. Firebase 콘솔에서 웹 클라이언트 ID가 올바르게 구성되었는지 확인합니다.
      1. `Web` 앱을 클릭하세요
         <img src="/social-login-assets/firebase_gc_web_app_select.webp" alt="Google Cloud 콘솔 웹 앱 선택" />
      2. 클라이언트 ID가 이전 단계에서 복사한 ID와 일치하는지 확인합니다.
         <img src="/social-login-assets/firebase_gc_web_app_client_id_configured.webp" alt="Google Cloud Console 웹 앱 클라이언트 ID가 구성됨" />
      :::참고
      웹 클라이언트의 나머지 설정은 무시하십시오. 이에 대해서는 [웹 설정 가이드](/docs/plugins/social-login/firebase/google/web/)에서 논의하겠습니다.
</Steps>

짜잔! 이제 Android에서 Firebase 인증으로 Google 로그인을 사용할 준비가 되었습니다.

## 문제 해결

인증이 중단되거나 실패하는 경우:

- `idToken` 대상이 Firebase 웹 클라이언트 ID와 일치하는지 확인하세요.
- Google 로그인이 Firebase 콘솔에서 활성화되어 있는지 확인하세요.
- SHA-1 지문이 올바르게 구성되었는지 확인하세요.
- 참고용으로 [예제 앱 코드](https://github.com/Cap-go/capacitor-social-login/tree/main/example-app/src/authUtils.ts)를 검토하세요.
