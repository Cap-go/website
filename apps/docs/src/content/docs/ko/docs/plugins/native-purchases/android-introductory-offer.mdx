신규 할인 혜택을 사용하면 자격을 갖춘 사용자에게 무료 평가판 또는 할인된 신규 출시 가격을 제공할 수 있습니다. 신규 출시 기간이 끝난 후 취소하지 않는 한 구독은 표준 가격으로 자동 갱신됩니다.

## 개요

입문 혜택은 다음을 수행하는 강력한 도구입니다.
- 신규 가입자의 진입 장벽을 낮춥니다.
- 전환율 증가
- 사용자가 위험 부담 없이 프리미엄 기능을 사용해 볼 수 있도록 허용
- 장기적인 구독자 관계 구축

## 자격

사용자가 이전에 구독에 대한 신규 할인 혜택을 구매하거나 받은 적이 없는 경우 신규 할인 혜택을 받을 수 있습니다. Google Play은 자격을 자동으로 처리합니다.

## 전제조건

신규 할인 혜택을 추가하기 전에 먼저 [자동 갱신 구독을 생성](/docs/plugins/native-purchases/android-create-subscription/)해야 합니다.

## 입문 제안 만들기

<Steps>

1. **액세스 제안 구성**

   Google Play 콘솔에서 구독으로 이동하여 **혜택 추가** 버튼을 선택하세요.

   ![쿠폰 추가 버튼](/native-purchases/android/introductory-offer/add-offer-button.webp)

2. **기본 요금제 선택**

   기본 요금제를 선택하라는 모달이 표시됩니다. 일반적으로 기본 계획은 하나만 있습니다. 계속하려면 **쿠폰 추가**를 클릭하세요.

   ![기본 요금제 선택](/native-purchases/android/introductory-offer/select-base-plan.webp)

3. **혜택 세부정보 구성**

   다음 정보를 입력하세요.

   **제안 ID**: 이 제안에 대한 고유 식별자입니다.

   **자격**: 이 혜택을 받을 수 있는 사람을 선택하세요.
   - **신규고객** : 한번도 구독한 적이 없는 사용자만
   - **기존 고객**: 이전에 구독한 사용자
   - **개발자 결정**: 맞춤 자격 로직(기본 구매에서는 지원되지 않음)

   :::주의
   기본 구매 플러그인은 "개발자가 결정한" 자격 옵션을 지원하지 않습니다. 대신 '신규 고객' 또는 '기존 고객'을 사용하세요.
   :::

   ![쿠폰 구성](/native-purchases/android/introductory-offer/configure-offer-details.webp)

4. **단계 추가**

   제안 구조를 정의하려면 페이지 하단의 **단계 추가**를 클릭하세요.

   최대 2개의 단계를 추가하여 다음과 같은 조합을 허용할 수 있습니다.
   - 무료 평가판만 제공
   - 가격만 할인됨
   - 무료 체험 후 할인된 반복 결제가 가능합니다.

5. **단계 유형 선택**

   세 가지 위상 유형 중에서 선택하십시오.

   **무료 평가판**
   - 정해진 기간 동안 무료 이용
   - 예: 7일 무료 후 $9.99/월

   **단일 결제**
   - 특정 기간 동안 일회성 할인 가격
   - 예: 2개월 동안 $1.99, 이후 월 $9.99

   **할인된 반복 결제**
   - 여러 주기에 대한 청구 주기당 요금 감소
   - 예: 3개월 동안 월 $4.99, 이후 월 $9.99

6. **단계 기간 구성**

   도입 단계의 지속 시간을 설정합니다.
   - 일, 주 또는 월
   - 청구 주기 수

7. **완료 및 활성화**

   **적용**을 클릭한 다음 **저장**을 클릭하여 혜택을 활성화하세요. 저장되면 **활성화** 버튼을 사용할 수 있습니다.

</Steps>

## 제안 단계 예시

### 예시 1: 간단한 무료 평가판
- 1단계: 7일 무료
- 이후: $9.99/월 표준 가격

### 예시 2: 소개 할인
- 1단계: 첫 달 $1.99
- 이후: $9.99/월 표준 가격### 예 3: 연장된 평가판 + 할인
- 1단계: 14일 무료
- 2단계: 2개월 동안 월 $4.99
- 이후: $9.99/월 표준 가격

## 앱에서 사용하기

기본 구매 플러그인은 소개 제안 자격 및 프리젠테이션을 자동으로 처리합니다.

```typescript
import { NativePurchases, PURCHASE_TYPE } from '@capgo/native-purchases';

// Fetch products (includes intro offer metadata)
const { products } = await NativePurchases.getProducts({
  productIdentifiers: ['com.example.premium.monthly'],
  productType: PURCHASE_TYPE.SUBS,
});

const product = products[0];

if (product.introductoryPrice) {
  console.log(`Intro price: ${product.introductoryPriceString}`);
  console.log(`Regular price: ${product.priceString}`);
  console.log(
    `Offer duration: ${product.introductoryPrice.subscriptionPeriod?.numberOfUnits} ${product.introductoryPrice.subscriptionPeriod?.unit}`,
  );
} else {
  console.log('No intro offer configured for this product');
}

// Purchase (Google Play applies intro pricing automatically if the user is eligible)
const transaction = await NativePurchases.purchaseProduct({
  productIdentifier: 'com.example.premium.monthly',
  planIdentifier: 'monthly-plan', // Base Plan ID from Google Play Console
  productType: PURCHASE_TYPE.SUBS,
});

console.log('Introductory purchase transaction', transaction.transactionId);
```

## 모범 사례

### 혜택 기간
- **무료 평가판**: 대부분의 앱에서는 3~14일이 최적입니다.
- **할인된 기간**: 습관을 기르는 데는 1~3개월이 적합합니다.
- **가격 할인**: 정가를 50~70% 할인하면 전환율이 높아집니다.

### 마케팅
- 신규 할인 혜택과 정상 가격을 명확하게 표시하세요.
- 인트로 기간 이후에 무슨 일이 일어나는지 보여주세요.
- 취소를 쉽고 투명하게 만듭니다.
- 소개 기간이 끝나기 전에 사용자에게 알림

### A/B 테스트
다양한 제안 구조를 테스트하세요.
- 무료 평가판 기간
- 할인율
- 할인 기간
- 단상 vs. 다상

## 중요 참고사항

- 구독당 한 번에 하나의 신규 프로모션만 활성화할 수 있습니다.
- 사용자는 구독당 한 번만 신규 할인 혜택을 요청할 수 있습니다.
- 구독 업그레이드/다운그레이드에는 신규 할인 혜택이 적용되지 않습니다.
- 소개 혜택 변경사항은 기존 구독자에게 영향을 주지 않습니다.

## 문제 해결

**소개 혜택이 표시되지 않음:**
- Play Console에서 혜택이 활성화되었는지 확인하세요.
- 사용자 자격 확인(이전에 제안을 사용한 적이 있을 수 있음)
- 앱이 최신 제품 정보를 사용하고 있는지 확인하세요.

**잘못된 사용자가 혜택을 받음:**
- 자격 설정 검토(신규 고객 및 기존 고객)
- 사용자가 이전에 다른 기기에서 구독했는지 확인
- Play Store 계정 내역 확인

**구매 시 혜택이 적용되지 않음:**
- 상품ID가 정확히 일치하는지 확인
- 제안이 아직 활성화되어 있고 만료되지 않았는지 확인하세요.
- 제안의 기간 설정을 확인하세요.
