## 소개

이 가이드는 iOS에서 Firebase 인증과 Google 로그인을 통합하는 데 도움이 됩니다. [일반 Firebase Google 설정](./general/)을 이미 완료했다고 가정합니다. 

:::note
Firebase 콘솔에서 아직 iOS 앱을 만들지 않았다고 가정하겠습니다. 그렇다면 단계가 약간 다를 것입니다.
:::

## 설정 단계

<Steps>
1. [console.cloud.google.com](https://console.cloud.google.com/)에서 프로젝트 개요로 이동합니다.
    <img src="/social-login-assets/firebase_project_overview.webp" alt="Firebase 프로젝트 개요" />

2. '앱 추가' 버튼을 클릭하세요.
    <img src="/social-login-assets/firebase_project_add_app.webp" alt="Firebase 앱 버튼 추가" />

    :::참고
    이 버튼을 찾으려면 여기를 찾아야 할 수도 있습니다. 이는 Firebase 콘솔에서 이미 다른 앱을 만든 경우에만 적용됩니다.
    :::

    <img src="/social-login-assets/firebase_project_add_app_2.webp" alt="Firebase 앱 버튼 추가" />
3. `iOS`을 선택하세요.
    <img src="/social-login-assets/firebase_project_add_app_ios.webp" alt="Firebase 앱 추가 iOS 버튼" />

4. 양식의 첫 번째 부분을 작성하세요.
   1. `Apple bundle ID`을 채우세요.
      1. `npx cap open ios`을 사용하여 앱에서 Xcode을 엽니다.
      2. `App`을 두 번 클릭합니다.
         <img src="/social-login-assets/xcode_app_click.png" alt="Xcode 프로젝트 탐색기의 앱 대상" />
      3. `Targets -> App`에 있는지 확인하세요.
         <img src="/social-login-assets/xcode_targets_app.png" alt="앱이 선택된 Xcode의 타겟 섹션" />
      4. '번들 식별자'를 찾으세요.
          <img src="/social-login-assets/xcode_bundle_id.png" alt="Xcode 프로젝트 설정의 번들 식별자 필드" />
          :::참고
          여기에 표시된 ID는 나머지 가이드에서 사용할 ID와 다릅니다. 가이드의 나머지 부분에서는 `app.capgo.plugin.SocialLogin`을 사용하겠습니다.
          :::
      5. `번들 식별자`를 복사하여 Firebase 콘솔에 붙여넣습니다.
         <img src="/social-login-assets/firebase_project_add_app_ios_bundle_id.webp" alt="Firebase 앱 추가 iOS 번들 ID 필드" />
   2. '앱 등록' 버튼을 클릭하세요.
         <img src="/social-login-assets/firebase_project_add_app_ios_register.webp" alt="Firebase 앱 추가 iOS 등록 버튼" />
5. `구성 파일 다운로드` 단계를 건너뜁니다.
    <img src="/social-login-assets/firebase_project_add_app_ios_skip_download.webp" alt="Firebase 앱 추가 iOS 다운로드 건너뛰기 버튼" />
6. `Add firebase SDK` 단계를 건너뜁니다.
    <img src="/social-login-assets/firebase_project_add_app_ios_skip_download_firebase_sdk_2.webp" alt="Firebase 앱 추가 iOS 다운로드 건너뛰기 Firebase SDK 버튼" />
7. '초기화 코드 추가' 단계를 건너뜁니다.
    <img src="/social-login-assets/firebase_project_add_app_ios_skip_download_firebase_sdk.webp" alt="Firebase 앱 추가 iOS 초기화 코드 추가 건너뛰기 버튼" />
8. '콘솔로 계속' 버튼을 클릭하세요.
    <img src="/social-login-assets/firebase_project_add_app_ios_continue_to_console.webp" alt="Firebase 앱 추가 iOS 콘솔 버튼으로 계속 진행" />
9. iOS 클라이언트 ID와 `YOUR_DOT_REVERSED_IOS_CLIENT_ID`를 받으세요.
   1. [console.cloud.google.com](https://console.cloud.google.com/)에서 Google Cloud Console로 이동합니다.
   2. 프로젝트 찾기
      1. 프로젝트 선택기를 클릭하세요
         <img src="/social-login-assets/firebase_double_check_gc_project_select.webp" alt="Google Cloud Console 프로젝트 선택기" />
      2. Firebase 프로젝트의 정확한 이름으로 프로젝트를 검색하고 클릭하세요. 제 경우에는 `sociallogin-tutorial-app`입니다.
         <img src="/social-login-assets/firebase_gc_project_select_project.webp" alt="Firebase 프로젝트 선택기 프로젝트" />
   3. 검색창을 열고 `credentials`을 엽니다.
      1. 검색창을 엽니다
         <img src="/social-login-assets/firebase_double_check_gc_search_bar.webp" alt="Google Cloud Console 검색창" />
      2. `credentials`을 검색하고 'API 및 서비스' 항목(스크린샷의 2번)을 클릭합니다.
         <img src="/social-login-assets/firebase_gc_credentials_search.webp" alt="Google Cloud Console 자격 증명 검색" />
   4. `iOS client for [YOUR_APP_ID] (auto created by Google Service)`을 클릭하세요. 제 경우에는 `sociallogin-tutorial-app`입니다.
      <img src="/social-login-assets/firebase_gc_credentials_ios_client_id.webp" alt="Google Cloud Console 사용자 인증 정보 iOS 클라이언트 ID" />
   5. `클라이언트 ID`와 `iOS URL scheme`을 복사합니다. 이는 각각 `iOSClientId` 및 `YOUR_DOT_REVERSED_IOS_CLIENT_ID`이 됩니다.

      :::참고
      이 가이드의 다음 부분에 설명된 대로 플러그인의 `initialize` 메서드에서 `iOSClientId`을 전달하고, 앱의 `Info.plist` 파일에서 `YOUR_DOT_REVERSED_IOS_CLIENT_ID`을 사용합니다.
      :::<img src="/social-login-assets/firebase_gc_credentials_ios_client_id_copy.webp" alt="Google Cloud Console 사용자 인증 정보 iOS 클라이언트 ID 복사" />
10. 웹 클라이언트 ID 받기
    1. Firebase 콘솔로 돌아가서 `Build` -> `Authentication`로 이동합니다.
        <img src="/social-login-assets/firebase_select_authentication.webp" alt="Firebase 인증 메뉴" />
    2. '로그인 방법' 버튼을 클릭하세요.
        <img src="/social-login-assets/firebase_select_authentication_sign_in_method.webp" alt="Firebase 인증 로그인 방법 버튼" />
    3. `Google` 공급자를 클릭하세요.
        <img src="/social-login-assets/firebase_select_authentication_sign_in_method_google.webp" alt="Firebase 인증 로그인 방법 Google 공급자" />
    4. `Web SDK configuration` 버튼을 클릭하세요.
        <img src="/social-login-assets/firebase_select_authentication_sign_in_method_web_sdk_configuration.webp" alt="Firebase 인증 로그인 방법 웹 SDK 구성 버튼" />
    5. '웹 클라이언트 ID'를 복사하세요. 이는 플러그인의 `initialize` 메서드에서 `webClientId`이 됩니다.
        <img src="/social-login-assets/firebase_select_authentication_sign_in_method_web_sdk_configuration_web_client_id.webp" alt="Firebase 인증 로그인 방법 웹 SDK 구성 웹 클라이언트 ID" />
11. 앱의 Info.plist 수정
    1. Xcode을 열고 `Info.plist` 파일을 찾습니다.
        <img src="/social-login-assets/xcode_info_file.png" alt="Xcode 프로젝트 탐색기의 Info.plist 파일" />
    2. 이 파일을 마우스 오른쪽 버튼으로 클릭하고 소스 코드로 엽니다.
        <img src="/social-login-assets/xcode_open_as_src_code.png" alt="소스 코드로 열기 옵션을 표시하는 마우스 오른쪽 버튼 클릭 메뉴" />
    3. `Plist` 파일 하단에 `</dict>` 태그가 표시됩니다.
        <img src="/social-login-assets/xcode_dict_tag.png" alt="Info.plist 파일에서 dict 태그 닫기" />
    4. 닫는 `</dict>` 태그 바로 앞에 다음 조각을 삽입합니다.
        <img src="/social-login-assets/xcode_plist_inserted.png" alt="dict 태그를 닫기 전에 URL 구성표 코드가 삽입된 Info.plist" />
        
        ```xml
        <key>CFBundleURLTypes</key>
        <array>
        <dict>
          <key>CFBundleURLSchemes</key>
          <array>
            <string>YOUR_DOT_REVERSED_IOS_CLIENT_ID</string>
          </array>
        </dict>
        </array>
        <key>GIDClientID</key>
        <string>YOUR_IOS_CLIENT_ID.apps.googleusercontent.com</string>
        ```
    
    5. `YOUR_DOT_REVERSED_IOS_CLIENT_ID`를 9단계에서 복사한 값(iOS URL 구성표)으로 변경합니다.
        <img src="/social-login-assets/xcode_plist_final.png" alt="URL 구성표에 삽입된 실제 역방향 클라이언트 ID가 포함된 Info.plist" />
        
        :::주의
        이 값이 `com.googleusercontent.apps`으로 **STARTS**되는지 확인하세요.
        :::
    
   6. `YOUR_IOS_CLIENT_ID`을 9단계에서 복사한 iOS 클라이언트 ID로 변경합니다.
    
   7. `Command + S`로 파일을 저장하세요.

12. `AppDelegate.swift` 수정
    1. AppDelegate를 엽니다.
        <img src="/social-login-assets/xcode_app_deleg.png" alt="Xcode 프로젝트 탐색기의 AppDelegate.swift 파일" />
    2. 파일 상단에 `import GoogleSignIn`을 삽입하세요.
        <img src="/social-login-assets/xcode_app_deleg_google_sign_in.png" alt="GoogleSignIn 가져오기가 추가된 AppDelegate.swift" />
    3. `func application(_ app: UIApplication, open url: URL, options: [UIApplication.OpenURLOptionsKey: Any] = [:])` 함수 찾기
        <img src="/social-login-assets/xcode_app_deleg_app_fn.png" alt="AppDelegate의 원래 애플리케이션 openURL 기능" />
    4. 다음과 같이 함수를 수정합니다.
        
        ```swift
        func application(_ app: UIApplication, open url: URL, options: [UIApplication.OpenURLOptionsKey: Any] = [:]) -> Bool {
            // Called when the app was launched with a url. Feel free to add additional processing here,
            // but if you want the App API to support tracking app url opens, make sure to keep this call
        
            var handled: Bool
        
            handled = GIDSignIn.sharedInstance.handle(url)
            if handled {
              return true
            }
        
            return ApplicationDelegateProxy.shared.application(app, open: url, options: options)
        }
        ```
        
        <img src="/social-login-assets/xcode_app_deleg_app_fn_mod.png" alt="GoogleSignIn 처리로 애플리케이션 openURL 기능을 수정했습니다." />
    
    5. `Command + S`로 파일을 저장하세요.

15. 앱에서 Google 로그인 사용

    이 단계에서는 앱에서 Google 로그인을 사용할 준비가 되었습니다.
    Google으로 인증하려면 예시 앱의 [authUtils.ts](https://github.com/Cap-go/capacitor-social-login/blob/main/example-app/src/authUtils.ts) 파일을 사용하세요.
</Steps>

 사용자는 처음 로그인할 때 Firebase 인증에서 자동으로 생성됩니다.

## 문제 해결

인증이 중단되거나 실패하는 경우:

- `idToken` 대상이 Firebase 웹 클라이언트 ID와 일치하는지 확인하세요.
- Google 로그인이 Firebase 콘솔에서 활성화되어 있는지 확인하세요.
- Info.plist에 올바른 URL 구성표와 GIDClientID가 있는지 확인하세요.
- `iOSServerClientId`이 웹 클라이언트 ID와 일치하는지 확인하세요.
- 참고용으로 [예제 앱 코드](https://github.com/Cap-go/capacitor-social-login/tree/main/example-app/src/authUtils.ts)를 검토하세요.
