## Chiffrement de bout en bout

À partir de la version 4.15.0, le plugin vous permet d'envoyer des mises à jour cryptées.

### Étape 1 : Créer une clé privée
```bash title='Create a private key'
npx @capgo/cli key create
```

### Étape 2 : Créez et compressez votre bundle
```bash title="Create bundle zip with checksum"
npx @capgo/cli bundle zip [appId] --key-v2 --json
```

L'indicateur `--key-v2` utilise le nouveau système de cryptage avec de meilleures sommes de contrôle, et l'indicateur `--json` affichera les informations du bundle, y compris la somme de contrôle dont vous aurez besoin pour le cryptage.

### Étape 3 : Chiffrez votre forfait
```bash title="Encrypt bundled zip with checksum"
npx @capgo/cli encrypt [path/to/zip] [checksum]
```

Le paramètre `checksum` est la somme de contrôle SHA256 générée par la commande zip à l'étape 2. La commande encrypt renverra un `ivSessionKey` et générera une somme de contrôle chiffrée. N'oubliez pas de renommer la clé `ivSessionKey` en `session_key` dans la charge utile de mise à jour.

### Étape 4 : Utiliser dans votre charge utile de mise à jour
```json
{
"version": "1.2.3",
"url": "https://myserver.com/app/updates/my-new-app-2.0.0.zip",
"session_key": "encrypted_session_key",
"checksum": "encrypted_checksum_from_encrypt_command"
}
```

Le `session_key` est le `ivSessionKey` renvoyé par la commande encrypt, et le `checksum` est la somme de contrôle chiffrée générée lors du chiffrement (et non la somme de contrôle d'origine de la commande zip).

Votre application pourra ensuite utiliser la clé privée pour déchiffrer le `session_key` et utiliser le `session_key` déchiffré pour déchiffrer la mise à jour. La somme de contrôle cryptée garantit la vérification de l’intégrité du bundle.
