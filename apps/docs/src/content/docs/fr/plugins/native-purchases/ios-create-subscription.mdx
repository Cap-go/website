Les abonnements renouvelables automatiquement offrent un accès récurrent au contenu, aux services ou aux fonctionnalités premium de votre application iOS. Ce guide vous guide dans la création d'abonnements dans App Store Connect.

## Aperçu

Les abonnements auto-renouvelables se renouvellent automatiquement à la fin de chaque période de facturation jusqu'à ce que les utilisateurs annulent. Ils sont parfaits pour :
- Contenu et fonctionnalités premium
- Expériences sans publicité
- Stockage et synchronisation cloud
- Services de diffusion en continu
- Outils et utilitaires professionnels

## Prérequis

Avant de créer des abonnements, vous devez :
1. [Créer un groupe d'abonnement](/docs/plugins/native-purchases/ios-subscription-group/) pour organiser vos abonnements
2. Avoir un abonnement actif au programme pour développeurs Apple
3. Complétez les informations bancaires et fiscales dans App Store Connect

## Création d'un abonnement

<Steps>

1. **Accédez à Abonnements**

   Dans App Store Connect, sélectionnez votre application et accédez à **Monétiser > Abonnements**.

   Sélectionnez votre groupe d'abonnement ou créez-en un nouveau si nécessaire.

   ![Accéder aux abonnements](/native-purchases/ios/create-subscription/navigate-to-subscriptions.webp)

2. **Créer un nouvel abonnement**

   Cliquez sur l'icône **** à côté de votre groupe d'abonnement pour ajouter un nouvel abonnement.

3. **Entrez les informations de base**

   **Nom de référence** : nom descriptif pour votre usage interne (non affiché aux clients)
   - Exemples : "Premium mensuel", "Ultimate Annual", "Basic Plan"

   **ID produit** : identifiant unique de cet abonnement (ne peut pas être modifié ultérieurement)
   -Format : `com.yourcompany.yourapp.premium_monthly`
   - Utilisez des noms descriptifs en minuscules avec des traits de soulignement
   - Nécessaire pour configurer le plugin natif-achats

   ![Entrez les détails de l'abonnement](/native-purchases/ios/create-subscription/enter-subscription-info.webp)

4. **Configurer la durée**

   Sélectionnez la durée de l'abonnement parmi les options disponibles :
   - 1 semaine
   - 1 mois
   - 2 mois
   - 3 mois
   - 6 mois
   - 1 an

   La durée détermine la fréquence à laquelle les utilisateurs sont facturés.

5. **Configurer les prix**

   Cliquez sur **Ajouter le prix de l'abonnement** pour configurer les tarifs :

   **Territoire de base** : sélectionnez votre marché principal (généralement votre pays)

   **Prix** : définissez le prix de l'abonnement
   - Apple se convertit automatiquement en d'autres devises
   - Choisissez parmi les niveaux de prix de Apple
   - Tenir compte de la valeur perçue et des taux du marché

   ![Configurer les tarifs](/native-purchases/ios/create-subscription/configure-pricing.webp)

6. **Partage familial (facultatif)**

   Décidez si vous souhaitez activer le partage familial, qui permet à jusqu'à 6 membres de la famille d'accéder à l'abonnement.

   :::attention
   Une fois le partage familial activé, il ne peut pas être désactivé pour ce produit.
   :::

   **Activer si :**
   - Le contenu est adapté à un usage familial
   - Vous souhaitez augmenter la proposition de valeur
   - Votre modèle économique le soutient

   **Ne pas activer si :**
   - L'abonnement est réservé à un usage individuel
   - Le contenu est personnalisé à l'utilisateur
   - Vous souhaitez maximiser les revenus par utilisateur

7. **Ajouter des localisations**

   Ajoutez des informations d'affichage d'abonnement dans toutes les langues prises en charge par votre application :

   **Nom d'affichage de l'abonnement** : nom destiné au client (par exemple, "Premium Monthly")

   **Description** : Brève description de ce que comprend l'abonnement
   - Soyez concis et axé sur les avantages
   - Mentionner les fonctionnalités clés
   - Mettre en valeur la proposition de valeur

   ![Ajouter des localisations](/native-purchases/ios/create-subscription/add-localization.webp)8. **App Store Image promotionnelle (facultatif)**

   Téléchargez une image promotionnelle pour cet abonnement (312x390 pixels) :
   - S'affiche dans la page d'abonnement App Store
   - Doit correspondre au design de votre application
   - Inclure le nom de l'abonnement pour plus de clarté

   :::remarque
   Bien que les images soient facultatives lors de la soumission initiale, elles sont requises pour un affichage promotionnel dans le App Store. Vous pourrez les ajouter plus tard.
   :::

9. **Enregistrer et soumettre**

   Cliquez sur **Enregistrer** pour créer l'abonnement.

   **Pour le premier abonnement :**
   - Doit être soumis avec une nouvelle version de l'application
   - À inclure dans votre prochaine soumission App Store
   - Impossible de soumettre indépendamment

   **Pour les abonnements ultérieurs :**
   - Peut être soumis directement à partir de la page Abonnements
   - Ne nécessite pas de nouvelle version de l'application
   - Disponible après l'approbation du premier abonnement

</Steps>

## Statut de l'abonnement

Votre abonnement aura l'un de ces statuts :

| Statut | Descriptif | Peut-on tester ? |
|--------|-------------|---------------|
| **Métadonnées manquantes** | Configuration incomplète | Oui (bac à sable) |
| **Prêt à soumettre** | Complété mais non soumis | Oui (bac à sable) |
| **En attente d'examen** | Soumis à Apple | Oui (bac à sable) |
| **En revue** | En cours de révision par Apple | Oui (bac à sable) |
| **Approuvé** | Disponible à l'achat | Oui |
| **Rejeté** | Besoin de changements | Oui (bac à sable) |

:::tip
Vous pouvez tester les abonnements en mode sandbox même s'ils affichent « Métadonnées manquantes » ou « Prêt à soumettre » !
:::

## Utilisation dans votre application

Une fois créé, référencez l'abonnement dans votre application à l'aide de l'ID produit :

```typescript
import { NativePurchases, PURCHASE_TYPE } from '@capgo/native-purchases';

// Fetch subscription products direct from StoreKit
const { products } = await NativePurchases.getProducts({
  productIdentifiers: [
    'com.yourcompany.yourapp.premium_monthly',
    'com.yourcompany.yourapp.premium_annual',
  ],
  productType: PURCHASE_TYPE.SUBS,
});

products.forEach((product) => {
  console.log(`${product.title}: ${product.priceString}`);
  console.log(`Duration: ${product.subscriptionPeriod}`);
  console.log(`Description: ${product.description}`);
});

// Purchase a subscription (StoreKit 2 automatically handles intro pricing and offers)
try {
  const transaction = await NativePurchases.purchaseProduct({
    productIdentifier: 'com.yourcompany.yourapp.premium_monthly',
    productType: PURCHASE_TYPE.SUBS,
  });

  console.log('Transaction ID:', transaction.transactionId);

  // StoreKit receipts are included on iOS for server-side validation
  await sendReceiptToBackend(transaction.receipt);
} catch (error) {
  console.error('Purchase failed:', error);
}

// Check subscription status using the store's data
const { purchases } = await NativePurchases.getPurchases({
  productType: PURCHASE_TYPE.SUBS,
});

const premium = purchases.find(
  (purchase) => purchase.productIdentifier === 'com.yourcompany.yourapp.premium_monthly',
);

if (premium?.isActive) {
  console.log('Expires:', premium.expirationDate);
  console.log('Will renew:', premium.willCancel === false);
  console.log('Store state:', premium.subscriptionState);
  unlockPremiumFeatures();
} else {
  showPaywall();
}
```

## meilleures pratiques

### Stratégie de tarification
- **Forfaits mensuels** : réduction de la barrière à l'entrée, création d'habitudes
- **Forfaits annuels** : meilleure valeur, LTV plus élevé, taux de désabonnement inférieur
- **Plusieurs niveaux** : Basic, Premium, Ultimate pour différents segments d'utilisateurs
- **Analyse concurrentielle** : recherchez les prix d'applications similaires

### ID de produit
- Utilisez un nom cohérent : `company.app.tier_duration`
- Inclure le niveau et la durée dans l'ID : `premium_monthly`, `ultimate_annual`
- Évitez de modifier les identifiants de produits (ils sont permanents)
- Documentez tous les identifiants de produits pour votre équipe

### Partage familial
- Activer les applications destinées à la famille (jeux, éducation, divertissement)
- Tenir compte de l'impact sur les revenus
- Testez minutieusement le comportement de partage
- Communiquer la capacité de partage en marketing

### Localisation
- Traduire tous les noms et descriptions d'abonnement
- Tenir compte des différences de prix régionales
- Test d'affichage dans toutes les langues prises en charge
- Utiliser un langage marketing culturellement approprié

### Images promotionnelles
- Maintenir un style visuel cohérent
- Inclure le nom de l'abonnement et l'avantage clé
- Mise à jour pour les promotions saisonnières
- Correspondre au langage de conception global de l'application

## Modèles d'abonnement courants

### Niveau unique (Freemium)
```
Free App + Premium Subscription
- Basic: Free (limited features)
- Premium Monthly: $4.99
- Premium Annual: $39.99 (save 33%)
```

### Multi-niveaux (bon, meilleur, meilleur)
```
- Basic Monthly: $4.99
- Premium Monthly: $9.99
- Ultimate Monthly: $19.99

- Basic Annual: $49.99
- Premium Annual: $99.99
- Ultimate Annual: $199.99
```

### Consommable + Abonnement Hybride
```
- Credit packs (consumable)
- Monthly subscription (unlimited credits)
- Annual subscription (unlimited + bonus features)
```

## Dépannage

**L'abonnement ne se charge pas dans l'application :**
- Vérifier que l'ID du produit correspond exactement (sensible à la casse)
- Vérifier que l'abonnement est dans le groupe d'abonnement
- Assurez-vous que l'identifiant du bundle correspond à App Store Connect
- Attendez 2-3 heures après la création du produit**Impossible de soumettre un abonnement :**
- Remplissez tous les champs obligatoires (nom, description, prix)
- Ajouter au moins une localisation
- Vérifier que les informations bancaires/fiscales sont approuvées
- Vérifiez si le premier abonnement (nécessite la version de l'application)

**Le bouton de partage familial est désactivé :**
- Déjà activé (ne peut pas être désactivé)
- Vérifier les détails de l'abonnement
- Contactez le support Apple si vous êtes bloqué

**Niveau de prix non disponible :**
- Peut être restreint dans certains territoires
- Choisissez un niveau alternatif
- Contactez Apple pour des questions sur les prix

**Erreur « ID de produit non valide » :**
- Doit être au format de domaine inversé
- Ne peut pas contenir d'espaces ou de caractères spéciaux
- Vérifiez les fautes de frappe
- Vérifier l'unicité de tous les produits
