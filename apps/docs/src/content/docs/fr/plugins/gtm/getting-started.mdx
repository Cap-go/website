## Configuration de la plateforme

### iOS

1. Téléchargez le fichier de configuration de votre conteneur GTM depuis la console Google Tag Manager.
2. Ajoutez le fichier conteneur JSON à votre projet iOS dans Xcode
3. Le fichier doit être nommé quelque chose comme `GTM-XXXXXX.json`

### Android

1. Téléchargez le fichier de configuration de votre conteneur GTM depuis la console Google Tag Manager.
2. Ajoutez le fichier conteneur JSON à `android/app/src/main/assets/`
3. Le fichier doit être nommé quelque chose comme `GTM-XXXXXX.json`

## Exemple d'utilisation

```typescript
import { CapacitorGTM } from '@capgo/capacitor-gtm';

// Initialize GTM with your container ID
await CapacitorGTM.init({
  containerId: 'GTM-XXXXXX'
});

// Push events to dataLayer
await CapacitorGTM.pushEvent({
  event: 'screen_view',
  parameters: {
    screen_name: 'Home',
    screen_class: 'HomeScreen'
  }
});

// Push custom events with parameters
await CapacitorGTM.pushEvent({
  event: 'purchase',
  parameters: {
    transaction_id: 'T12345',
    value: 25.00,
    currency: 'USD',
    items: [
      {
        item_id: 'SKU_12345',
        item_name: 'Product Name',
        quantity: 1,
        price: 25.00
      }
    ]
  }
});

// Set user properties
await CapacitorGTM.setUserProperty({
  key: 'user_type',
  value: 'premium'
});

// Get values from dataLayer
const result = await CapacitorGTM.getVariable({
  key: 'config_value'
});
console.log('Config value:', result.value);
```

## API Référence

### initialisation(options)

```typescript
init(options: { containerId: string }) => Promise<void>
```

Initialisez Google Tag Manager avec votre ID de conteneur.

| Paramètres | Tapez |
| ------------- | -------------------------------- |
| **`options`** | `{ containerId: string }` |

### pushEvent(options)

```typescript
pushEvent(options: GTMEvent) => Promise<void>
```

Envoyez un événement au dataLayer avec des paramètres facultatifs.

| Paramètres | Tapez |
| ------------- | ----------------- |
| **`options`** | `GTMEvent` |

### setUserProperty(options)

```typescript
setUserProperty(options: { key: string; value: string }) => Promise<void>
```

Définissez une propriété utilisateur pour la segmentation et le ciblage.

| Paramètres | Tapez |
| ------------- | ----------------------------------------- |
| **`options`** | `{ key: string; value: string }` |

### getVariable(options)

```typescript
getVariable(options: { key: string }) => Promise<{ value: any }>
```

Obtenez une valeur de la configuration du conteneur GTM.

| Paramètres | Tapez |
| ------------- | -------------------- |
| **`options`** | `{ key: string }` |

**Retours :** `Promise<{ value: any }>`

##Interfaces

### Événement GTME

| Accessoire | Tapez | Descriptif |
| ---------------- | ------------------------------------ | ---------------------------------- |
| **`event`** | `string` | Le nom de l'événement |
| **`parameters`** | `Record<string, any>` | Paramètres d'événement facultatifs (facultatif) |

## meilleures pratiques

- Initialisez GTM au début du cycle de vie de votre application
- Utiliser des conventions de dénomination d'événements cohérentes
- Inclure les paramètres pertinents avec chaque événement
- Testez votre configuration GTM en mode aperçu
- Surveiller les données dans la console Google Tag Manager
- Utilisez les propriétés utilisateur pour une segmentation significative

## Débogage

Activez le mode débogage dans la console Google Tag Manager pour tester votre implémentation :

1. Ouvrez la console GTM
2. Accédez au mode Aperçu
3. Connectez-vous à votre application à l'aide de l'ID du conteneur
4. Vérifiez que les événements sont correctement suivis
