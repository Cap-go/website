## Aperçu

Le plugin Capacitor Persistent Account permet le stockage sécurisé et la persistance des données de compte utilisateur entre les installations d'applications. Ce plugin garantit que les informations du compte utilisateur restent disponibles même après la réinstallation de l'application, offrant ainsi une expérience utilisateur transparente et une continuité de compte.

##Installation

```bash
npm install @capgo/capacitor-persistent-account
npx cap sync
```

## Méthodes principales API

### Gestion des données
- `saveAccount(options: { data: unknown })` - Enregistrez en toute sécurité les données du compte sur un stockage persistant
- `readAccount()` - Récupère les données de compte stockées, renvoie `Promise<{ data: unknown | null }>`

## Principales fonctionnalités

- **Persistance entre installations** : les données du compte survivent à la désinstallation et à la réinstallation de l'application
- **Stockage sécurisé** : utilise des mécanismes de stockage sécurisé spécifiques à la plate-forme
- **Flexibilité de type** : stockez toute structure de données de compte sérialisable
- **Prise en charge multiplateforme** : implémentation native pour iOS et Android

## Exemple d'utilisation

```typescript
import { PersistentAccount } from '@capgo/capacitor-persistent-account';

// Define your account data structure
interface UserAccount {
  userId: string;
  username: string;
  email: string;
  preferences: {
    theme: string;
    notifications: boolean;
  };
}

// Save account data
const accountData: UserAccount = {
  userId: '12345',
  username: 'john_doe',
  email: 'john@example.com',
  preferences: {
    theme: 'dark',
    notifications: true
  }
};

await PersistentAccount.saveAccount({ data: accountData });

// Read account data
const result = await PersistentAccount.readAccount();
if (result.data) {
  const account = result.data as UserAccount;
  console.log('Restored account:', account.username);
} else {
  console.log('No account data found');
}
```

## Cas d'utilisation

- **Intégration des utilisateurs** : préservez la progression des utilisateurs grâce aux réinstallations d'applications
- **Récupération de compte** : restaurez les sessions utilisateur après les mises à jour de l'application
- **Stockage des préférences** : conserver les paramètres et configurations utilisateur
- **Applications hors ligne** : stockez les données utilisateur essentielles localement

## Implémentation de la plateforme

### iOS
- Utilise les services de trousseau iOS pour un stockage sécurisé et persistant
- Les données survivent à la suppression de l'application et aux restaurations de l'appareil

### Android
- Utilise le gestionnaire de compte Android ou les préférences partagées avec sauvegarde
- Conserve les données lors des réinstallations d'applications et des migrations d'appareils

## Considérations de sécurité

- Les données du compte sont stockées à l'aide d'un stockage sécurisé spécifique à la plateforme
- Envisagez le cryptage des données pour les informations sensibles
- Mettre en œuvre une validation appropriée des données lors de la lecture des comptes stockés
- Suivre les directives de la plateforme pour le traitement des données utilisateur
