## Prérequis

Ce guide vous aidera à intégrer la connexion Apple avec l'authentification Supabase sur Android. On suppose que vous avez déjà complété :
- le [Supabase Apple Connexion - Configuration générale](./general/).

:::note[Important]
Apple La connexion sur Android nécessite un serveur principal, car Apple ne fournit pas de prise en charge native de Android. Nous utiliserons une fonction Edge Supabase comme backend.
:::

## Étape 1 : Déployer la fonction Backend Edge

Tout d’abord, nous devons déployer la fonction Edge Supabase qui gérera le rappel Apple OAuth.

<Steps>

1. ** Accédez au répertoire de votre projet Supabase **

   ```bash
   cd your-project/supabase
   ```

2. **Créez la fonction Edge** (si elle n'existe pas)

   ```bash
   supabase functions new apple-signin-callback
   ```

3. **Copiez le code de la fonction Edge**

   L'implémentation complète de la fonction Edge est disponible dans [exemple d'application] (https://github.com/Cap-go/capacitor-social-login/tree/main/example-app/supabase/functions/apple-signin-callback).

   Copiez les fichiers suivants dans votre projet :
   - `supabase/functions/apple-signin-callback/index.ts` - Code fonction bord principal
   - `supabase/functions/apple-signin-callback/deno.json` - Importer la carte pour les dépendances (inclut la bibliothèque `jose` pour la signature JWT)

4. **Configurer la vérification JWT**

   Le point de terminaison de rappel Apple OAuth doit être public (aucune authentification requise) car Apple y sera redirigé. Mettez à jour votre fichier `supabase/config.toml` :

   ```toml
   [functions.apple-signin-callback]
   enabled = true
   verify_jwt = false  # Important: Set to false for public OAuth callback
   import_map = "./functions/apple-signin-callback/deno.json"
   entrypoint = "./functions/apple-signin-callback/index.ts"
   ```

   :::caution[Important]
   Le paramètre `verify_jwt = false` rend ce point de terminaison public. Ceci est requis car la redirection OAuth de Apple n'inclut pas les en-têtes d'authentification Supabase. Le point de terminaison valide lui-même le flux OAuth.
   :::

5. **Déployer la fonction**

   ```bash
   supabase functions deploy apple-signin-callback
   ```

6. **Obtenez l'URL de votre fonction**

   Après le déploiement, vous obtiendrez une URL telle que :
   ```
   https://your-project-ref.supabase.co/functions/v1/apple-signin-callback
   ```

   Si vous ne le trouvez pas, vous pouvez procéder comme suit :

   1. Open `https://supabase.com/dashboard/project/YOUR_PROJECT_REF/functions`
   2. Cliquez sur l'URL de la fonction `apple-signin-callback` pour la copier.
      <img src="/social-login-assets/supabase_functions_apple_signin_callback.webp" alt="Supabase Fonctions Apple Rappel de connexion" />

   :::note[Save this URL]
   Vous aurez besoin de cette URL à l'étape suivante lors de la configuration du portail de développeur Apple.
   :::

</Steps>

## Étape 2 : Configurer le portail de développeur Apple

Nous devons maintenant configurer le portail de développeur Apple avec l'URL du backend et obtenir toutes les valeurs requises.

:::note[URL de la fonction]
Assurez-vous d'avoir votre URL de fonction Edge Supabase de l'étape 1 avant de continuer. Vous en aurez besoin pour configurer l'URL de retour dans le portail des développeurs Apple.
:::

<Steps>

1. **Suivez le Apple Connexion Android Guide de configuration**

   Complétez le [Apple Connexion Android Guide de configuration](/docs/plugins/social-login/apple/android/) pour :
   - Create a Service ID
   - Générer une clé privée (fichier .p8)
   - Obtenez votre identifiant d'équipe et votre identifiant de clé
   - Configurer l'URL de retour

2. **Définissez l'URL de retour dans le portail des développeurs Apple**

   Lors de la configuration de l'URL de retour dans le portail de développeur Apple (étape 6.9 du guide Apple), utilisez votre URL de fonction Edge Supabase :

   ```
   https://your-project-ref.supabase.co/functions/v1/apple-signin-callback
   ```:::avertissement[Important : utilisez l'URL de la fonction Edge Supabase]
   **N'utilisez PAS** l'URL de redirection du [Apple Connexion Android Guide de configuration](/docs/plugins/social-login/apple/android/). Ce guide utilise une URL de serveur backend personnalisée. Pour l'intégration Supabase, vous **devez** utiliser votre URL de fonction Edge Supabase à la place.
   :::

   :::caution[Correspondance exacte requise]
   L'URL de retour doit correspondre **exactement** à ce que vous configurez ici. Apple est très strict sur la correspondance des URI de redirection.
   :::

3. **Collectez toutes les valeurs requises**

   Après avoir terminé le guide de configuration Apple, vous devriez avoir :
   - **APPLE_TEAM_ID** : votre identifiant d'équipe de développeurs Apple
   - **APPLE_KEY_ID** : ID de clé du portail de développeur Apple
   - **APPLE_PRIVATE_KEY** : votre fichier de clé privée .p8 (doit être encodé en base64)
   - **ANDROID_SERVICE_ID** : votre identifiant de service Apple (par exemple, `com.example.app.service`)
   - **BASE_REDIRECT_URL** : l'URL de votre lien profond (par exemple, `capgo-demo-app://path`)

   :::note[URL du lien profond]
   Le `BASE_REDIRECT_URL` est le schéma de lien profond configuré dans votre `AndroidManifest.xml`. C'est là que le backend redirigera après l'authentification.

   La valeur de votre lien profond dépend du code `android:scheme="capgo-demo-app"`. Si vous avez défini `android:scheme="capgo-demo-app"` dans votre `AndroidManifest.xml`, alors votre lien profond sera `capgo-demo-app://path`.
   :::

</Steps>

## Étape 3 : Définir les secrets Supabase

Nous devons maintenant configurer les variables d'environnement (secrets) pour la fonction Edge Supabase.

<Steps>

1. **Encodez votre clé privée**

   Tout d’abord, encodez votre clé privée Apple (fichier .p8) en base64 :

   ```bash
   base64 -i AuthKey_XXXXX.p8
   ```

   Copiez l'intégralité de la sortie (c'est une seule longue chaîne).

2. **Définissez les secrets en utilisant Supabase CLI**

   ```bash
   supabase secrets set APPLE_TEAM_ID=your_team_id
   supabase secrets set APPLE_KEY_ID=your_key_id
   supabase secrets set APPLE_PRIVATE_KEY=your_base64_encoded_key
   supabase secrets set ANDROID_SERVICE_ID=your.service.id
   supabase secrets set BASE_REDIRECT_URL=your-app://path
   supabase secrets set APPLE_REDIRECT_URI=https://your-project-ref.supabase.co/functions/v1/apple-signin-callback
   ```

   :::note[Remplacer les espaces réservés]
   Remplacez toutes les valeurs d'espace réservé par vos valeurs réelles de l'étape 2.
   :::

3. **Alternative : définissez des secrets dans le tableau de bord Supabase**

   Si vous préférez utiliser le tableau de bord :
   1. Accédez au tableau de bord de votre projet Supabase
   2. Accédez à **Fonctions Edge** → **Paramètres** → **Secrets**.
   3. Ajoutez chaque variable secrète avec sa valeur

</Steps>

:::note[Android Configuration de l'application]
Le [Apple Login Android Guide de configuration](/docs/plugins/social-login/apple/android/) couvre déjà la configuration de votre application Android :
- Ajout du filtre d'intention de lien profond à `AndroidManifest.xml`
- Ajout du gestionnaire `onNewIntent` à `MainActivity.java`

Assurez-vous d'avoir terminé ces étapes avant de continuer. Le schéma de liens profonds que vous y configurez sera votre `BASE_REDIRECT_URL` à l'étape 3.
:::

## Étape 4 : Utilisez l'assistant d'authentification

Vous pouvez désormais utiliser l'assistant d'authentification dans le code de votre application.

### Implémentation

L’implémentation complète est disponible dans le fichier [`supabaseAuthUtils.ts`](https://github.com/Cap-go/capacitor-social-login/blob/main/example-app/src/supabaseAuthUtils.ts) de l’application exemple.

### Utilisation de l'assistant d'authentification

```typescript
import { authenticateWithAppleSupabase } from './supabaseAuthUtils';

const result = await authenticateWithAppleSupabase();
if (result.success) {
  console.log('Signed in:', result.user);
  // Navigate to your authenticated area
} else {
  console.error('Error:', result.error);
}
```

### Mettre à jour la fonction d'assistance

Lorsque vous utilisez la fonction d'assistance `authenticateWithAppleSupabase`, vous **devez** mettre à jour les valeurs suivantes pour qu'elles correspondent à votre configuration :

1. **Mise à jour `redirectUrl`** - Définissez ceci sur l'URL de votre fonction Edge Supabase :

   ```typescript
   const redirectUrl = platform === 'android' 
     ? 'https://your-project-ref.supabase.co/functions/v1/apple-signin-callback'
     : undefined;
   ```2. **Mise à jour `clientId`** - Définissez ceci sur votre ID de service Apple :

   ```typescript
   await SocialLogin.initialize({
     apple: {
       clientId: isIOS
         ? undefined // iOS uses bundle ID automatically
         : 'your.service.id.here', // Your Apple Service ID for Android
       redirectUrl: redirectUrl,
     },
   });
   ```

   :::attention[Important]
   Remplacez `'your.service.id.here'` par votre ID de service Apple réel (la même valeur que vous avez utilisée pour `ANDROID_SERVICE_ID` à l'étape 3).
   :::

Voir la [implémentation complète](https://github.com/Cap-go/capacitor-social-login/blob/main/example-app/src/supabaseAuthUtils.ts) pour référence.

## Étape 5 : Tester l'intégration

<Steps>

1. **Créez et exécutez votre application Android**

   ```bash
   npx cap sync android
   npx cap run android
   ```

2. **Testez le flux d'authentification**

   - Appuyez sur le bouton "Connectez-vous avec Apple".
   - Vous devriez voir la page Apple OAuth dans un navigateur.
   - Après vous être authentifié, vous devriez être redirigé vers votre application
   - Vérifiez les journaux de la console pour toute erreur

3. **Vérifiez le flux**

   Le flux complet doit être :
   1. L'utilisateur appuie sur « Connectez-vous avec Apple »
   2. L'application ouvre le navigateur avec Apple OAuth
   3. L'utilisateur s'authentifie avec Apple
   4. Apple redirige vers : `https://your-project-ref.supabase.co/functions/v1/apple-signin-callback`
   5. La fonction Edge échange du code contre des jetons
   6. La fonction Edge redirige vers : `your-app://path?id_token=...&access_token=...`
   7. L'application Android reçoit le lien profond et traite le jeton d'identité
   8. L'application se connecte à Supabase avec le jeton d'identité

</Steps>

## Dépannage

Si l'authentification échoue :

- **Incompatibilité d'URI de redirection** : vérifiez que l'URL de retour dans le portail de développeur Apple correspond exactement au secret `APPLE_REDIRECT_URI`.
- **Le lien profond ne fonctionne pas** : vérifiez que le filtre d'intention `AndroidManifest.xml` correspond à votre `BASE_REDIRECT_URL`
- **Secrets manquants** : Vérifiez que tous les secrets sont correctement définis à l'aide de la « liste des secrets de la supabase »
- **Échec de l'échange de jetons** : vérifiez les journaux des fonctions Edge dans le tableau de bord Supabase pour connaître les messages d'erreur détaillés.
- **L'application ne reçoit pas de rappel** : assurez-vous que `onNewIntent` est correctement implémenté dans MainActivity
- Consultez l'[exemple de code d'application](https://github.com/Cap-go/capacitor-social-login/blob/main/example-app/src/supabaseAuthUtils.ts) pour référence.

## Comment ça marche

Sur Android, la connexion Apple utilise un flux de redirection OAuth :

1. **Initialisation** : Le plugin est initialisé avec votre ID de service et l'URL de redirection backend
2. **OAuth Flow** : ouvre un onglet personnalisé de navigateur/Chrome avec la page OAuth de Apple.
3. **Backend Callback** : Apple redirige vers votre fonction Edge Supabase avec un code d'autorisation
4. **Token Exchange** : la fonction Edge échange le code contre des jetons à l'aide du point de terminaison de jeton de Apple.
5. **Deep Link Redirect** : la fonction Edge redirige vers votre application avec le jeton d'identité
6. **Supabase Authentification** : l'application reçoit le jeton et se connecte à Supabase

Ce flux est nécessaire car Apple ne fournit pas de prise en charge native de Android pour la connexion avec Apple.
