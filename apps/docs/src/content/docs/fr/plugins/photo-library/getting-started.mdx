## Autorisations de la plateforme

- **iOS** : mettez à jour `Info.plist` avec `NSPhotoLibraryUsageDescription` (et `NSPhotoLibraryAddUsageDescription` si vous envisagez de sauvegarder des médias).
- **Android** : Pour Android 13+, déclarez les autorisations photo/vidéo (`READ_MEDIA_IMAGES`, `READ_MEDIA_VIDEO`). Capacitor 5 gère cela automatiquement, mais revérifiez les manifestes personnalisés.

## Demander l'accès

```typescript
import { PhotoLibrary } from '@capgo/capacitor-photo-library';

const { state } = await PhotoLibrary.requestAuthorization();
if (state !== 'authorized' && state !== 'limited') {
  throw new Error('Photo access denied');
}
```

## Charger la galerie

```typescript
import { Capacitor } from '@capacitor/core';

const { assets, hasMore, nextPage } = await PhotoLibrary.getLibrary({
  limit: 100,
  includeAlbums: false,
  thumbnailWidth: 256,
  thumbnailHeight: 256,
});

const gallery = assets.map((asset) => ({
  id: asset.id,
  fileName: asset.fileName,
  created: asset.creationDate,
  thumbnailUrl: asset.thumbnail
    ? asset.thumbnail.webPath ?? Capacitor.convertFileSrc(asset.thumbnail.path)
    : undefined,
  photoUrl: asset.file
    ? asset.file.webPath ?? Capacitor.convertFileSrc(asset.file.path)
    : undefined,
}));
```

Pour récupérer des pages supplémentaires, appelez `PhotoLibrary.getLibrary({ cursor: nextPage })`.

## Laisser les utilisateurs choisir un média

```typescript
const selection = await PhotoLibrary.pickMedia({
  selectionLimit: 5,
  includeVideos: true,
});

console.log('User selected', selection.assets.length, 'items');
```

## Nettoyer les fichiers en cache

Les miniatures et exportations générées sont stockées dans le répertoire de cache de l'application sous `photoLibrary`. Supprimez-les avec votre fichier préféré API lorsque vous devez récupérer de l'espace de stockage.
